<task type="auto" tdd="true">
  <name>Task 2: Crypto penetration testing</name>
  <files>Metadata/pentest/crypto-pentest-report.md, DataWarehouse.Hardening.Tests/Security/CryptoPentestTests.cs</files>
  <behavior>
    - Two consecutive encryptions of same plaintext produce different ciphertexts (no IV reuse)
    - Manipulating ciphertext padding does not leak information via timing or error differences
    - Secret comparison (HMAC verify, token check) takes constant time regardless of input
    - FIPS mode rejects non-FIPS algorithms (MD5, DES, RC4)
    - Post-quantum key exchange produces valid shared secret
  </behavior>
  <action>
    1. Create `CryptoPentestTests.cs`:
       - **IV reuse detection**: Encrypt same plaintext 1,000 times with same key. Assert all ciphertexts are unique (proving unique IV/nonce per encryption).
       - **Padding oracle**: Send 256 variations of corrupted ciphertext (last byte = 0x00..0xFF). Assert all produce same error type and similar timing (no distinguishable padding vs. content error).
       - **Constant-time comparison**: Use `Stopwatch` to measure HMAC verification with correct tag vs. wrong-first-byte vs. wrong-last-byte vs. all-wrong. Assert timing variance < 1ms (no early-exit on mismatch).
       - **FIPS enforcement**: Enable FIPS mode, attempt to use MD5/DES/RC4. Assert `CryptographicException` or equivalent rejection.
       - **Key derivation**: PBKDF2/Argon2 with minimum iteration count. Assert iteration count >= 100,000 for PBKDF2 or memory >= 64MB for Argon2.
       - **Post-quantum**: If PQ key exchange implemented, test ML-KEM (Kyber) encapsulate/decapsulate round-trip.
    2. Generate `Metadata/pentest/crypto-pentest-report.md`:
       ```
       ## Crypto Penetration Test Results

       | Test | Target | Result | Evidence |
       |------|--------|--------|----------|
       | IV uniqueness | AES-GCM | PASS | 1000/1000 unique |
       | Padding oracle | AES-CBC | PASS | Timing variance < 0.5ms |
       | Constant-time HMAC | HMAC-SHA256 | PASS | Max timing delta 0.3ms |
       | FIPS enforcement | MD5/DES/RC4 | PASS | All rejected |
       ```
    3. Test all encryption strategies in `UltimateEncryption` plugin, not just one algorithm.
  </action>
  <verify>
    <automated>dotnet test DataWarehouse.Hardening.Tests/DataWarehouse.Hardening.Tests.csproj --filter "FullyQualifiedName~CryptoPentest" --no-build 2>&1 | tail -5</automated>
  </verify>
