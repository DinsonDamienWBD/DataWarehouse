  <behavior>
    - SQL injection payloads ('; DROP TABLE; --, UNION SELECT, etc.) rejected by parser
    - Path traversal sequences (../, ..\, %2e%2e) in S3 keys rejected
    - Zip bombs (42.zip pattern, nested compression) detected by size ratio check
    - VDE header with block_count = Int64.MaxValue rejected (prevents OOM allocation)
    - Assembly without valid plugin signature rejected by loader
    - Null byte injection in filenames rejected
  </behavior>
  <action>
    1. Create `DataPlanePentestTests.cs`:
       - **SQL injection** (target: PostgreSQL wire protocol parser):
         ```csharp
         [Theory]
         [InlineData("'; DROP TABLE users; --")]
         [InlineData("1 UNION SELECT * FROM information_schema.tables")]
         [InlineData("1; EXEC xp_cmdshell('whoami')")]
         [InlineData("1' OR '1'='1")]
         public void SqlInjectionBlocked(string payload) { ... }
         ```
       - **Path traversal** (target: S3 key handler):
         ```csharp
         [Theory]
         [InlineData("../../../etc/passwd")]
         [InlineData("..\\..\\windows\\system32\\config\\sam")]
         [InlineData("%2e%2e%2f%2e%2e%2fetc%2fpasswd")]
         [InlineData("....//....//etc/passwd")]
         public void PathTraversalBlocked(string key) { ... }
         ```
       - **Zip bombs** (target: compression strategies):
         - Create a test zip with 1KB compressed / 1GB decompressed ratio
         - Assert decompression is rejected when ratio exceeds threshold (e.g., 100:1)
       - **Oversized allocation** (target: VDE parser):
         - Craft VDE header bytes with `block_count = long.MaxValue`
         - Assert parser rejects with size validation error, not OOM
       - **Malicious plugin** (target: plugin loader):
         - Create a mock assembly without valid plugin attributes
         - Assert loader rejects it during validation
         - Create an assembly with a valid interface but malicious Initialize() (attempts file access outside sandbox)
         - Assert sandbox prevents file access
       - **Null byte injection**: Embed `\0` in filenames, S3 keys, SQL strings. Assert rejection.
    2. Generate `Metadata/pentest/data-plane-report.md` with results table:
       ```

       | Attack Vector | Target | Payloads Tested | Blocked | Status |
       |---------------|--------|-----------------|---------|--------|
       | SQL Injection | PostgreSQL parser | 15 | 15/15 | PASS |
       | Path Traversal | S3 key handler | 10 | 10/10 | PASS |
       ```
    3. If any attack succeeds, fix the vulnerable code path (add input validation, sanitization, or size checks) and retest.
  </action>
