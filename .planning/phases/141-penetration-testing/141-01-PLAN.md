  <action>
    1. Start the Dashboard API and S3-compatible API on localhost (if possible via `dotnet run` or test host).
    2. Run OWASP ZAP automated scan:
       - Target Dashboard REST API endpoints
       - Target S3-compatible API endpoints

       - `nmap -sV -p- localhost` — identify all open ports and services

       - **Authentication bypass**: Send requests without auth token, assert 401
       - **Privilege escalation**: Use low-privilege token to access admin endpoints, assert 403
       - **IDOR**: Access resource with wrong user ID, assert 403/404
       - **CSRF**: Send state-changing request without CSRF token (if applicable)
       - **SSRF**: Inject internal URLs in webhook/callback parameters, assert rejection
    5. Document all findings in `Metadata/pentest/network-api-report.md`:

       ```
       ### Manual Tests
       | Test | Result | Evidence |
       |------|--------|----------|
       ```
    6. Remediate any high-severity findings and retest.
  </action>
