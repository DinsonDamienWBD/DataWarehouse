1. Create `DataWarehouse.Hardening.Tests/Soak/SoakTestConfiguration.cs`:
   ```csharp
   public sealed class SoakTestConfiguration
   {
       public TimeSpan Duration { get; init; } = TimeSpan.FromMinutes(10); // CI default
       public int ConcurrentWorkers { get; init; } = 4;
       public int ChunkSizeBytes { get; init; } = 1_048_576; // 1MB
       public TimeSpan GcSampleInterval { get; init; } = TimeSpan.FromSeconds(5);
       public int Gen2ThresholdPerMinute { get; init; } = 2;
       public double MaxWorkingSetGrowthPercent { get; init; } = 10.0; // max 10% growth over duration

       public static SoakTestConfiguration CI => new() { Duration = TimeSpan.FromMinutes(10) };
       public static SoakTestConfiguration Manual24Hr => new() { Duration = TimeSpan.FromHours(24) };
       public static SoakTestConfiguration Manual72Hr => new() { Duration = TimeSpan.FromHours(72) };
   }
   ```

2. Create `DataWarehouse.Hardening.Tests/Soak/GcEventMonitor.cs`:
   - Extends `System.Diagnostics.Tracing.EventListener`
   - Listens to `System.Runtime` event source for GC-related counters:
     - `gc-heap-size` (Gen0/1/2 heap sizes)
     - `gen-0-gc-count`, `gen-1-gc-count`, `gen-2-gc-count`
     - `alloc-rate` (allocation rate bytes/sec)
     - `gc-fragmentation` (if available)
   - Collects samples at configurable interval into a `ConcurrentQueue<GcSample>`
   - `GcSample` record: `DateTime Timestamp, long Gen0Count, long Gen1Count, long Gen2Count, long HeapSizeBytes, long WorkingSetBytes, long AllocRateBytes`
   - Implements `IDisposable` for cleanup
   - Provides analysis methods:
     - `GetGen2RatePerMinute()` — Gen2 collections per minute over entire duration
     - `IsWorkingSetMonotonic()` — linear regression on working set samples, returns true if slope significantly positive
     - `GetWorkingSetGrowthPercent()` — (final - initial) / initial * 100
     - `GetSamples()` — return all collected samples
   - Monitor also tracks `Process.GetCurrentProcess().WorkingSet64` at each sample point
  </action>

1. Create `DataWarehouse.Hardening.Tests/Soak/SoakTestHarness.cs`:

   - Uses `GcEventMonitor` for continuous GC tracking
   - Uses `SoakTestConfiguration` for parameters

2. Workload pattern — moderate continuous read/write:
   - N concurrent worker tasks (default 4) each performing:
     a. Generate random chunk (1MB)
     b. Write chunk through VDE pipeline
     c. Read back and verify checksum
     d. Small delay (10-50ms) to simulate realistic access pattern
     e. Repeat until duration expires
   - Use `CancellationTokenSource` with timeout for clean shutdown

3. Test methods:
   a. `[Fact]` `Test_SoakCI_10Minutes()`:
      - Uses `SoakTestConfiguration.CI`
      - Runs 10-minute soak with GC monitoring
      - Asserts at end:
        - Gen2 rate < threshold per minute
        - Working set growth < max percent
        - No monotonic working set growth
        - No exceptions during workload
        - No deadlocks (all workers completed before timeout + grace)
      - Tagged `[Trait("Category", "Soak")]` and `[Trait("Duration", "CI")]`

   b. `[Fact(Skip = "Manual 24hr soak - run explicitly")]` `Test_Soak_24Hours()`:
      - Uses `SoakTestConfiguration.Manual24Hr`
      - Same assertions
      - Tagged `[Trait("Category", "Soak")]` and `[Trait("Duration", "Manual")]`

4. After workload completes, generate in-test summary to `ITestOutputHelper`:
   - Total operations performed
   - Total data written/read
   - GC summary (Gen0/1/2 counts, rates)
   - Working set trend (initial, peak, final, growth%)
   - Pass/fail per criterion

5. The harness must handle graceful shutdown:
   - CancellationToken propagated to all workers
   - Workers finish current operation before stopping
   - GcEventMonitor disposed after final sample collected
