 10    /// <summary>
 11    /// Time-Lock Puzzle Strategy - RSA-based time-lock encryption.
 12    ///
 13    /// Based on the seminal paper "Time-lock puzzles and timed-release crypto"
 14    /// by Rivest, Shamir, and Wagner (1996).
 15    ///
 16    /// Algorithm:
 17    /// The key is encrypted such that decryption requires T sequential squaring operations
 18    /// modulo N (RSA modulus). This creates a compute-bound delay that cannot be parallelized.
 19    ///
 20    /// To create a time-lock puzzle:
 21    /// 1. Generate RSA modulus N = p * q
 22    /// 2. Generate random a
 23    /// 3. Compute b = a^(2^T) mod N (efficiently using phi(N) = (p-1)(q-1))
 24    /// 4. Encrypt key K as: C = K XOR H(b)
 25    ///
 26    /// To solve the puzzle (without knowing p, q):
 27    /// 1. Compute b = a^(2^T) mod N by performing T sequential squarings
 28    /// 2. Decrypt key as: K = C XOR H(b)
 29    ///
 30    /// Security Properties:
 31    /// - Cannot be parallelized (inherent sequential computation)
 32    /// - Puzzle creation is O(log T), solving is O(T)
 33    /// - Based on RSA assumption (factoring is hard)
 34    ///
 35    /// Features:
 36    /// - Configurable unlock time (compute-bound, not wall-clock)
 37    /// - Puzzle verification without solving
 38    /// - Parallel puzzle chains for different time horizons
 39    /// - Progress checkpointing for long solves
