  <behavior>
    - EraseBlockAsync(blockAddress, blockSize, policy) overwrites block per policy
    - Zero mode: single pass of 0x00 bytes
    - Crypto mode: fill with RandomNumberGenerator.Fill, then zero
    - DoD mode: pass 1 (0x00), pass 2 (0xFF), pass 3 (CSPRNG random), optional verify
    - NvmeTrim mode: issues TRIM command (or logs advisory if no TRIM support)
    - CryptoThenTrim: crypto-erase followed by TRIM
    - Verify pass re-reads and confirms expected content
  </behavior>
  <action>
    1. Create `DataWarehouse.SDK/Security/SecureErase/SecureEraseManager.cs`:
       - `ISecureEraseManager` interface:
         - `Task<EraseResult> EraseBlockAsync(long blockAddress, int blockSizeBytes, ErasePolicy policy, CancellationToken ct = default)`
         - `Task<EraseResult> EraseBatchAsync(IReadOnlyList<long> blockAddresses, int blockSizeBytes, ErasePolicy policy, CancellationToken ct = default)`
         - `Task<EraseResult> EraseRangeAsync(long startBlock, long blockCount, int blockSizeBytes, ErasePolicy policy, CancellationToken ct = default)`
       - `SecureEraseManager` implementation:
         - Constructor takes `Func<long, int, Memory<byte>, CancellationToken, Task>` writeCallback (abstracted block write)
         - Constructor takes `Func<long, int, CancellationToken, Task<Memory<byte>>>?` readCallback (for verify)
         - Constructor takes `Func<long, long, CancellationToken, Task>?` trimCallback (for NVMe TRIM)
         - Zero pass: `buffer.Span.Clear()` then write
         - Crypto pass: `RandomNumberGenerator.Fill(buffer.Span)` then write, then `CryptographicOperations.ZeroMemory(buffer.Span)`

         - Batch: process blocks in parallel up to `Environment.ProcessorCount` concurrency

       - Test: CryptoThenTrim calls both crypto and trim
       - Test: ErasePolicy.None does nothing (returns success with 0 bytes)
