    - Test: DetectAsync returns VulkanDevice when only vulkaninfo succeeds
    - Test: DetectAsync returns GpuCapability.None when all probes fail (no exception)
    - Test: Device info includes name, memory, driver version, compute capability
    - Test: Probe timeout (2s) prevents hanging on unresponsive drivers
  </behavior>
  <action>
1. Create `GpuCapabilityDetector`:
   - `DetectAsync(CancellationToken ct) -> GpuCapabilityReport`
   - Probe CUDA: run `nvidia-smi --query-gpu=name,memory.total,driver_version,compute_cap --format=csv,noheader` with 2s timeout
   - Probe OpenCL: run `clinfo --raw` with 2s timeout, parse platform/device info
   - Probe Vulkan: run `vulkaninfo --summary` with 2s timeout, parse device info
   - Each probe wrapped in try/catch — failure means API not available, not error
   - `GpuCapabilityReport` record: `CudaDevices`, `OpenClDevices`, `VulkanDevices`, `BestAvailable` (nullable)
   - `GpuDeviceInfo` record: `Name`, `MemoryBytes`, `DriverVersion`, `ComputeCapability`, `Api` (enum: Cuda/OpenCl/Vulkan)
   - Cache result — detection runs once at startup, not per-call

2. Mark `[SdkCompatibility("8.0.0", Notes = "Phase 131: GPU capability detection")]`.

     - Compression: CUDA > OpenCL > Vulkan (CUDA nvcomp library)
     - Query filter: CUDA > OpenCL > Vulkan (all similar for simple kernels)
   - `GpuOperationType` enum: General, Crypto, Compression, Filter, Aggregation
