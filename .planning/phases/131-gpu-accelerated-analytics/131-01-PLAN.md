<objective>
Bridge UltimateCompute GPU strategies to VDE SQL OLAP query pipeline for GPU-accelerated analytics.

Purpose: ADV-02 requires GPU-accelerated analytics. UltimateCompute already has CUDA, OpenCL, Vulkan strategies but they are not wired to the query pipeline. This plan creates the bridge layer: predicate pushdown (WHERE filters on columnar data via GPU kernel) and aggregation (SUM/COUNT/AVG via parallel reduction).

Output: IGpuQueryBridge contract, GpuQueryBridge implementation, GpuPredicateCompiler, tests.
</objective>

    - Test: AggregateAsync(Avg) returns correct average

  <action>
1. Implement `GpuQueryBridge : IGpuQueryBridge`:
   - Constructor accepts `IReadOnlyList<ComputeRuntimeStrategyBase> gpuStrategies` (injected by plugin)
   - On construction, probe each GPU strategy via `InitializeAsync` — first success wins
   - Store detected device info from nvidia-smi / clinfo / vulkaninfo output
   - `IsGpuAvailable` returns true if any GPU strategy initialized

2. `FilterAsync`:
   - Use `GpuPredicateCompiler` to validate params
   - Build a CUDA/OpenCL/Vulkan compute task with filter kernel code embedded as string
   - Kernel: each thread checks one row against predicate, outputs bitmap
   - Dispatch via selected GPU strategy's `ExecuteAsync`
   - Parse output bitmap from stdout
   - CPU fallback: if no GPU, iterate columnar data with System.Numerics SIMD Vector<T> comparison

3. `AggregateAsync`:
   - Build compute task with parallel reduction kernel
   - For Sum: tree reduction with __shfl_down_sync (CUDA) or equivalent
   - For Min/Max: tournament reduction
   - For Avg: Sum + Count then divide
   - CPU fallback: LINQ aggregate with SIMD vectorization where applicable

4. Thread safety: `GpuQueryBridge` is stateless per-call, device selection cached at construction.
