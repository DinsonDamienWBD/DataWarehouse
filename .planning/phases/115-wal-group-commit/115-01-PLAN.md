Create the GroupCommitAccumulator that collects WAL entries for a configurable interval and flushes them with a single fsync per group, replacing the current per-write fsync pattern in WalBlockWriterStage.

Purpose: Gap A.9 — each write currently goes through the full WAL pipeline individually, causing excessive fsync overhead. Group commit batches multiple WAL entries and issues a single fsync, targeting 10x throughput for small writes.

Output: GroupCommitConfig record, GroupCommitAccumulator class, and unit tests proving batching behavior.
