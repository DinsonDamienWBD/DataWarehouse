<objective>
Add adaptive group sizing to the GroupCommitAccumulator so it flushes immediately under low load (latency priority) and accumulates larger groups under high load (throughput priority), with BackpressureAware integration.

Purpose: Static group commit intervals are suboptimal. Low-frequency writes should not wait 10ms unnecessarily, while high-frequency bursts should batch aggressively. This plan implements the adaptive policy and integrates with the existing backpressure infrastructure.

Output: AdaptiveGroupCommitPolicy class, updated GroupCommitAccumulator with policy hook, and tests proving adaptive behavior.
