<task type="auto">
  <name>Task 1: Run dotCover coverage analysis on hardening tests</name>
  <files>Metadata/dotcover-hardening-report/</files>
  <action>
1. Create output directory: `mkdir -p Metadata/dotcover-hardening-report/`

2. Run dotCover coverage analysis using the CLI. The `dotCover` tool is available at `~/.dotnet/tools/dotCover`. Run coverage on the hardening tests with assembly filters to include only production code:
   ```
   dotCover cover \
     --targetExecutable="dotnet" \
     --targetArguments="test DataWarehouse.Hardening.Tests/DataWarehouse.Hardening.Tests.csproj -c Release --no-build" \
     --filters="+:DataWarehouse.*;-:*.Tests;-:*.Benchmarks;-:xunit.*;-:Microsoft.*" \
     --output="Metadata/dotcover-hardening-report/coverage.dcvr" \
     --reportType="HTML" \
     --attributeFilters="System.Diagnostics.CodeAnalysis.ExcludeFromCodeCoverageAttribute"
   ```

   If the above single-command approach is not supported by this version of dotCover, use the two-step approach:
   ```
   # Step 1: Collect coverage snapshot
   dotCover cover \
     --targetExecutable="dotnet" \
     --targetArguments="test DataWarehouse.Hardening.Tests/DataWarehouse.Hardening.Tests.csproj -c Release --no-build" \
     --filters="+:DataWarehouse.*;-:*.Tests;-:*.Benchmarks" \
     --output="Metadata/dotcover-hardening-report/coverage.dcvr"

   # Step 2: Generate HTML report from snapshot
   dotCover report \
     --source="Metadata/dotcover-hardening-report/coverage.dcvr" \
     --output="Metadata/dotcover-hardening-report/index.html" \
     --reportType="HTML"
   ```

   Alternative: If `dotCover` CLI uses different syntax (e.g., `dotCover.exe cover /TargetExecutable=...`), adapt accordingly. Check `dotCover help cover` for exact syntax.

3. Also generate an XML report for programmatic analysis:
   ```
   dotCover report \
     --source="Metadata/dotcover-hardening-report/coverage.dcvr" \
     --output="Metadata/dotcover-hardening-report/coverage-detail.xml" \
     --reportType="DetailedXML"
   ```

4. Verify the HTML report was generated successfully by checking `Metadata/dotcover-hardening-report/index.html` exists.
