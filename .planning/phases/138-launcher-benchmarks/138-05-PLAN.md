  <action>
BenchmarkCIRunner.cs:
1. CI-optimized runner: shorter iterations (Quick job), fewer warmup cycles
2. Run all benchmarks, collect results as JSON
3. Output: structured JSON with benchmark name, mean, median, P50/P99/P999, allocated bytes
4. Save results to Metadata/benchmarks/{date}-{commit-sha}.json
5. Entry point: `dotnet run -- --ci` to run in CI mode

BaselineComparer.cs:
1. Load baseline from Metadata/benchmarks/baseline.json
2. Compare current results against baseline
3. Per-benchmark comparison: calculate % change for mean, P50, P99
4. Output: ComparisonReport with per-benchmark status (improved/stable/regressed)
5. Update baseline: `dotnet run -- --update-baseline` writes current results as new baseline

RegressionDetector.cs:
1. Accept ComparisonReport, flag benchmarks with >10% regression
2. Configurable threshold (default 10%, per-benchmark override in baseline.json)
3. Output: markdown-formatted report suitable for PR comment
4. Return exit code 1 if any regression exceeds threshold (CI gate)
5. Exclude known-noisy benchmarks from gating (configurable allowlist)
6. Report format:
   ```
   ## Benchmark Results
   | Benchmark | Baseline | Current | Change | Status |
   |-----------|----------|---------|--------|--------|
   | VDE Write 1MB | 5.2ms | 5.1ms | -1.9% | PASS |
   | Federation Join | 12ms | 15ms | +25% | FAIL |
   ```
  </action>
