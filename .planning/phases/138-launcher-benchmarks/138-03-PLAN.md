StartupPolicySchema.cs:
Define C# records for startup-policy.json:
```
StartupPolicy:
  RequiredPlugins: string[] (boot fails if any missing)
  OptionalPlugins: string[] (log warning if missing)
  PluginLoadOrder: string[] (load in this sequence)
  ResourceLimits:
    MaxMemoryMB: int (default: no limit)
    MaxCpuPercent: int (default: 100)
    MaxThreads: int (default: Environment.ProcessorCount * 4)
    MaxPlugins: int (default: 52)
  StartupTimeout: TimeSpan (default: 120s)
  HealthCheckDelay: TimeSpan (default: 10s)
  FailurePolicy: "halt" | "degrade" | "retry"
  RetryAttempts: int (default: 3)
  Environment: Dictionary<string, string> (set env vars)
```

StartupPolicyEngine.cs:
1. LoadPolicyAsync(string path) — read startup-policy.json, deserialize, validate
2. ValidatePolicy() — check all required plugins exist in plugin directory
3. EnforceResourceLimits() — set GC memory limit, thread pool limits, CPU affinity
4. OrchestrateBoot(DataWarehouseKernel kernel) — load plugins in policy order, check required, warn optional
5. HandleFailure(PluginLoadFailure failure) — halt (throw), degrade (continue without), retry (reload)
6. Generate default policy: if no startup-policy.json exists, create one with defaults
7. PolicyEngine integration: subscribe to runtime policy changes for dynamic reconfiguration
