  <action>
LivenessProbe.cs:
- Returns 200 OK always (process alive = liveness OK)
- Response: `{"status": "Healthy", "timestamp": "ISO8601", "uptime": "HH:MM:SS"}`
- Lightweight: no kernel dependency, no I/O

ReadinessProbe.cs:
- Checks: kernel initialized, plugin registry loaded, message bus running, storage accessible
- Returns 200 OK only when ALL checks pass
- Returns 503 Service Unavailable with failed checks when not ready
- Response: `{"status": "Healthy|Degraded|Unhealthy", "checks": [{"name": "kernel", "status": "Healthy", "duration": "5ms"}, ...], "totalDuration": "15ms"}`
- Each check has configurable timeout (default 5s)

StartupProbe.cs:
- Tracks boot sequence progress: 0% (starting) -> 20% (kernel init) -> 40% (plugins loading) -> 60% (plugins initialized) -> 80% (message bus started) -> 100% (ready)
- Returns 200 when boot complete, 503 during boot
- Response: `{"status": "Starting|Ready", "progress": 60, "phase": "plugins_initializing", "elapsed": "3.2s", "plugins_loaded": 30, "plugins_total": 52}`
- Subscribe to kernel lifecycle events to track progress
