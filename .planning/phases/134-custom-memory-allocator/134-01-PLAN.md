<objective>
Implement a native memory slab allocator for zero-GC I/O hot paths with four slab sizes aligned to VDE block sizes.

Purpose: ADV-05 requires custom allocators for zero-GC I/O. The existing AlignedMemoryPool handles single buffer sizes. SlabAllocator extends this to multiple slab sizes (4KB, 64KB, 1MB, 4MB) covering all VDE block operations, with a free-list pool per size that completely avoids GC pressure on the hot path.

Output: ISlabAllocator contract, SlabAllocator, SlabPool, tests.
</objective>

  <action>
1. Create `ISlabAllocator` in `DataWarehouse.SDK/Contracts/Memory/`:
   ```
   public interface ISlabAllocator : IDisposable
   {
       SlabHandle Rent(SlabSize size);
       void Return(SlabHandle handle);
       SlabAllocatorStats GetStats();
   }
   ```
   `SlabSize` enum: `Size4KB = 4096, Size64KB = 65536, Size1MB = 1048576, Size4MB = 4194304`
   `SlabHandle`: wrapper around `unsafe byte*` with length, IDisposable (calls Return on dispose)
   `SlabAllocatorStats`: TotalAllocated, TotalPooled, TotalFreed per slab size

2. Create `SlabPool` in `Infrastructure/Memory/`:
   - One pool per slab size — generic over size parameter
   - Free list: `ConcurrentQueue<IntPtr>` (not ConcurrentBag — FIFO for cache friendliness)
   - `Rent()`: TryDequeue from free list. If empty, `NativeMemory.AlignedAlloc(size, 4096)`. Increment TotalAllocated counter.
   - `Return(IntPtr)`: If pool count < maxPooled, Enqueue to free list. Else `NativeMemory.AlignedFree`. Increment TotalFreed counter.
   - `maxPooled` per size: 4KB=1024, 64KB=256, 1MB=64, 4MB=16 (scales inversely with size)
   - Thread-safe via ConcurrentQueue (lock-free)
   - `Dispose()`: free all pooled buffers via NativeMemory.AlignedFree

3. Mark `[SdkCompatibility("8.0.0", Notes = "Phase 134: Slab allocator")]`.
  </action>

  </verify>
