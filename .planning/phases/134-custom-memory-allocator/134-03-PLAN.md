<objective>
Implement GC pause mitigation for latency-critical code paths targeting <1ms max GC pause on I/O hot path.

Purpose: ADV-05 requires <1ms GC pause on hot paths. Even with slab and arena allocators reducing GC pressure, background GC can still pause threads. This plan uses `GC.TryStartNoGCRegion` to suppress GC during critical sections and `MemoryPressureMonitor` to schedule GC during idle windows.

Output: GcPauseMitigator, MemoryPressureMonitor, tests.
</objective>

   - Do NOT use around network I/O (unpredictable duration)

    - Test: Monitor tracks memory pressure via GC.GetGCMemoryInfo()
    - Test: High pressure (>80% committed) triggers immediate Gen2 during next idle
    - Test: Low pressure (<50%) skips scheduled GC
    - Test: StartMonitoring / StopMonitoring lifecycle works correctly
  </behavior>
  <action>
1. **MemoryPressureMonitor** (`Infrastructure/Memory/MemoryPressureMonitor.cs`):
   - Background `PeriodicTimer` (250ms interval) checks I/O activity and memory pressure
   - `ReportIoActivity()`: called by I/O path on each operation, resets idle timer
   - `_lastIoActivity` volatile DateTime field
   - **Idle detection**: if `DateTime.UtcNow - _lastIoActivity > idleThreshold` (default 500ms), system is idle
   - **Pressure assessment**:
     - `GC.GetGCMemoryInfo()` to get `HeapSizeBytes`, `TotalCommittedBytes`, `HighMemoryLoadThresholdBytes`
     - High pressure: TotalCommittedBytes > 80% of HighMemoryLoadThreshold
     - Normal: 50-80%
     - Low: <50%
   - **Scheduling logic**:
     - Idle + High pressure: `GC.Collect(2, GCCollectionMode.Optimized, blocking: false)`
     - Idle + Normal pressure: `GC.Collect(1, GCCollectionMode.Optimized, blocking: false)`
     - Idle + Low pressure: skip
     - Active I/O (not idle): never trigger GC, regardless of pressure
   - **Full GC notification** (optional): `GC.RegisterForFullGCNotification(10, 10)` to get early warning, then schedule work completion before GC hits
   - `MemoryPressureLevel` enum: Low, Normal, High, Critical
   - `StartMonitoring(CancellationToken)` / `StopMonitoring()`
