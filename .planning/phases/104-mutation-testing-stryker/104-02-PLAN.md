<task type="auto">
  <name>Task 1: Tighten test assertions for surviving mutants</name>
  <files>DataWarehouse.Hardening.Tests/</files>
  <action>
1. Read `Metadata/stryker-hardening-report/stryker-initial-run.md` to get the list of surviving mutants.

2. For each surviving mutant, analyze why it survived:
   a. **Missing assertion** — test exercises the code but doesn't check the specific value/behavior the mutation changes. Fix: add targeted assertion.
   b. **Missing test coverage** — no test exercises that code path at all. Fix: add new test method.
   c. **Equivalent mutant** — the mutation produces functionally identical behavior (e.g., replacing `x >= 0` with `x > -1`). Document but do not fix.
   d. **Timeout mutant** — mutation causes infinite loop/hang. These count as killed. No action needed.

3. For each non-equivalent surviving mutant, add or tighten assertions in existing test files:
   - Add precise value checks (`Assert.Equal(expected, actual)` not just `Assert.NotNull`)
   - Add boundary condition tests that distinguish original from mutated behavior
   - Add negative tests that verify error/exception paths
   - Each new assertion should target a specific mutant ID from the initial report

4. Organize new assertions logically — add to existing test classes where they fit, or create new test classes only if a code area had zero coverage.

5. Run `dotnet test DataWarehouse.Hardening.Tests/` to verify all new tests pass before re-running Stryker.
  </action>
