**ArcBlockCache.cs** — Adaptive Replacement Cache (ARC) for block data:
- Namespace: `DataWarehouse.SDK.VirtualDiskEngine.IO.ObjectStorage`
- Two LRU lists: T1 (recently used once), T2 (used at least twice)
- Two ghost lists: B1, B2 (evicted keys for frequency tracking)
- Adaptive parameter `p` shifts balance between T1 and T2
- Thread-safe via ReaderWriterLockSlim
- Methods: `bool TryGet(long blockNumber, Memory<byte> buffer)`, `void Put(long blockNumber, ReadOnlyMemory<byte> data)`, `void Invalidate(long blockNumber)`, `void Clear()`
- Constructor: `ArcBlockCache(int maxBlocks, int blockSize)`
- Internal storage: `Dictionary<long, byte[]>` for actual data, linked lists for LRU tracking
- Apply `[SdkCompatibility("8.0.0")]`

**S3EpochFlushBlockDevice.cs** — Core IBlockDevice implementation:
- Namespace: `DataWarehouse.SDK.VirtualDiskEngine.IO.ObjectStorage`
- Constructor: `S3EpochFlushBlockDevice(IObjectStorageClient objectStorage, S3BlockDeviceOptions options, ILogger<S3EpochFlushBlockDevice>? logger = null)`
- Implements `IBlockDevice` (BlockSize, BlockCount, ReadBlockAsync, WriteBlockAsync, FlushAsync, DisposeAsync)
- Internal state:
  - `ArcBlockCache _cache` — L1 read cache
  - `ConcurrentDictionary<long, byte[]> _localWal` — in-memory WAL (block number -> latest data). This is the write buffer, not the VDE WAL.
  - `long _currentEpoch` — current epoch number, incremented on flush
  - `SemaphoreSlim _flushLock` — guards epoch flush
  - `Timer _backgroundFlushTimer` — periodic epoch flush based on S3FlushIntervalMs
  - `volatile bool _disposed`
- **WriteBlockAsync**: Validate block number range. Copy data into `_localWal[blockNumber]`. Update `_cache` with new data. If `_localWal` byte size exceeds `LocalWalSizeMb`, trigger epoch flush.
- **ReadBlockAsync**: 1) Check `_cache.TryGet` -> return if hit. 2) Check `_localWal.TryGetValue` -> copy to buffer, cache, return. 3) Fall back to `_objectStorage.GetObjectAsync` using epoch manifest to locate block. Cache result. Copy to buffer.
- **FlushAsync**: Package all entries in `_localWal` into an epoch object. Call `_objectStorage.PutObjectAsync` with key `{EpochObjectPrefix}epoch-{VdeUuid}-{epoch}.dwvd-wal`. Update manifest. Clear `_localWal`. Increment `_currentEpoch`.
- **DisposeAsync**: Stop timer, flush remaining WAL entries, dispose objectStorage if needed.
- For the read-from-S3 path, maintain an `_epochBlockIndex` (`Dictionary<long, (long epochNumber, int offsetInEpoch)>`) that maps block numbers to their epoch location. Updated when manifest is loaded or epoch is flushed.
- Apply `[SdkCompatibility("8.0.0")]`
- Use `ObjectDisposedException.ThrowIf(_disposed, this)` guard on all public methods.
- Use `ArgumentOutOfRangeException.ThrowIfNegative/ThrowIfGreaterThanOrEqual` for block number validation (same pattern as FileBlockDevice).
