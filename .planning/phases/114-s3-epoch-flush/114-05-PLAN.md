**EpochTieringPolicy.cs** — Intelligent storage tiering:
- Namespace: `DataWarehouse.SDK.VirtualDiskEngine.IO.ObjectStorage`
- `StorageTier` enum: `Standard = 0, InfrequentAccess = 1, Glacier = 2, Archive = 3`

- Sealed class `EpochTieringPolicy`:
  - Constructor: `EpochTieringPolicy(IObjectStorageClient objectStorage, S3BlockDeviceOptions options, ILogger<EpochTieringPolicy>? logger = null)`
  - **EvaluateAsync method** — `Task<TieringReport> EvaluateAsync(EpochManifest manifest, CancellationToken ct)`:
    1. For each epoch in manifest.Epochs:
       - Calculate age = `DateTimeOffset.UtcNow - epoch.CreatedUtc`
       - Determine target tier: age > ArchiveAfterDays -> Archive, age > TierAfterDays -> InfrequentAccess, else Standard
       - Compare with current tier (stored in manifest entry metadata or default Standard)
    2. Return `TieringReport` with list of `TieringAction(long EpochNumber, string ObjectKey, StorageTier CurrentTier, StorageTier TargetTier)`
  - **ApplyAsync method** — `Task<int> ApplyAsync(TieringReport report, CancellationToken ct)`:
    1. For each action where CurrentTier != TargetTier:
       - Use `_objectStorage.CopyObjectAsync` with metadata indicating new storage class (provider-specific metadata key: `x-amz-storage-class` for S3, `AccessTier` for Azure, `storageClass` for GCS)
       - Log tier change
    2. Return count of objects tiered
    NOTE: Since IObjectStorageClient.CopyObjectAsync doesn't support storage class metadata, add an optional `IDictionary<string, string>? destinationMetadata` parameter to CopyObjectAsync, OR use PutObjectAsync with the storage class in metadata. Choose the simpler approach: store a `StorageTier` field on `EpochManifestEntry` and let the actual tiering be a no-op at the IObjectStorageClient level (real tiering requires provider-specific lifecycle policies configured externally). The policy TRACKS tier state and RECOMMENDS actions; actual S3 lifecycle rules handle the physical tier changes. Log recommended tier changes.

**ReadAheadPrefetcher.cs** — Sequential read-ahead:
- Namespace: `DataWarehouse.SDK.VirtualDiskEngine.IO.ObjectStorage`
- Sealed class `ReadAheadPrefetcher`:
  - Constructor: `ReadAheadPrefetcher(IObjectStorageClient objectStorage, ArcBlockCache cache, S3BlockDeviceOptions options, Dictionary<long, (long EpochNumber, int IndexInEpoch)> blockIndex, ILogger<ReadAheadPrefetcher>? logger = null)`
  - **OnBlockRead method** — `void OnBlockRead(long blockNumber)`:
    1. Track last N block reads (ring buffer of 8 entries)
    2. Detect sequential pattern: if last 3+ reads are consecutive ascending block numbers
    3. If sequential detected and `EnableReadAheadPrefetch` is true: fire-and-forget `PrefetchRangeAsync` for next `ReadAheadBlockCount` blocks
  - **PrefetchRangeAsync method** — `Task PrefetchRangeAsync(long startBlock, int count, CancellationToken ct)`:
    1. For each block in range that is NOT already in `_cache`:
       - Look up in `blockIndex` to find epoch
       - Download epoch object if not already cached (cache entire epoch's blocks)
       - Put each block from epoch into `_cache`
    2. Use `SemaphoreSlim` to limit concurrent prefetch operations to `MaxConcurrentS3Operations`
  - Internal: Track which epochs have been prefetch-loaded to avoid redundant downloads

  ```csharp
  private static IBlockDevice CreateS3BlockDevice(BlockDeviceOptions options, ILogger? logger)
  {
      if (options.S3Options is null)
          throw new InvalidOperationException(
              "S3BlockDeviceOptions must be provided when using S3EpochFlush implementation. " +
              "Set BlockDeviceOptions.S3Options before calling Create.");

      // S3 block device requires async initialization (mount + recovery).
      // Use synchronous wrapper since BlockDeviceFactory.Create is synchronous.
      // For full async support, use S3BlockDeviceFactory.CreateAsync directly.
      var inMemory = new InMemoryObjectStorageClient();
      var device = S3BlockDeviceFactory.CreateInMemoryAsync(options.S3Options, logger)
          .GetAwaiter().GetResult();
      logger?.LogInformation("BlockDeviceFactory: created S3EpochFlushBlockDevice " +
          "(bucket={Bucket}, blockSize={BlockSize}, blockCount={BlockCount})",
          options.S3Options.BucketName, options.S3Options.BlockSize, options.S3Options.BlockCount);
      return device;
  }
  ```
