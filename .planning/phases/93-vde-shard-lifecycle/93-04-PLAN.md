Create `ShardMigrationEngine.cs` in namespace `DataWarehouse.SDK.VirtualDiskEngine.Federation.Lifecycle`:

1. `sealed class ShardMigrationEngine : IAsyncDisposable` with constructor taking:
   - `IShardVdeAccessor shardAccessor` (for reading source shard data and writing to destination)
   - `RoutingTable routingTable` (for atomic slot reassignment on completion)
   - `MigrationRedirectTable redirectTable` (for registering in-flight redirects)
   - `IDistributedLockService lockService` (for distributed coordination)
   - `FederationOptions options` (for timeouts)

2. `async Task<ShardMigrationState> MigrateShardAsync(Guid sourceShardId, Guid destinationShardId, int startSlot, int endSlot, CancellationToken ct)`:
   - Create ShardMigrationState, register in redirect table
   - Transition Pending -> Preparing: acquire distributed lock `$"shard-migration:{sourceShardId}"` with lease 10 minutes. If lock fails, throw `InvalidOperationException("Shard already being migrated")`
   - Transition Preparing -> CopyingBlocks: enumerate all objects from source via `shardAccessor.ListShardAsync`, for each object track key in a `List<string>`. Update progress after each batch of 100 objects.
   - Transition CopyingBlocks -> CatchingUp: register redirect so new reads go to destination. In a real system this is where delta-sync happens; for SDK layer, mark as caught up immediately (the VDE layer above handles CoW semantics).
   - Transition CatchingUp -> Switching: reassign slots in RoutingTable from sourceShardId to destinationShardId via `routingTable.AssignSlot(slot, destinationShardId)` for each slot in [startSlot, endSlot).
   - Transition Switching -> Completed: unregister redirect, release lock, record completion time.
   - Wrap entire flow in try/catch: on any exception, call `RollbackAsync` (transition to RolledBack, reassign slots back to source, unregister redirect, release lock, set FailureReason).

3. `async Task<MigrationProgress> GetProgressAsync(Guid migrationId, CancellationToken ct)`:
   - Look up migration state from redirect table by migration ID, return progress snapshot.

4. `async Task CancelMigrationAsync(Guid sourceShardId, CancellationToken ct)`:
   - Find active migration, call RollbackAsync if in non-terminal phase.

5. Private `async Task RollbackAsync(ShardMigrationState state, CancellationToken ct)`:
   - Transition to RolledBack
   - Restore routing table slots to source shard ID
   - Unregister redirect
   - Release distributed lock (ignore failures during cleanup)

6. `IAsyncDisposable`: cancel all active migrations on dispose.

Mark `[SdkCompatibility("6.0.0")]`. Lock owner string: `$"migration-engine-{Environment.MachineName}"`. Use `ArgumentNullException.ThrowIfNull` for all constructor params.
