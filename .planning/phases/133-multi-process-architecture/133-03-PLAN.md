  <behavior>
    - Test: Worker crash triggers restart within 100ms (measured via Stopwatch)
    - Test: Restarted worker receives reconstructed state and resumes processing
    - Test: Circuit breaker trips after 5 restarts in 60 seconds — worker stays down, degradation mode
    - Test: GPU worker crash triggers CPU fallback for in-flight operations
    - Test: Supervisor handles simultaneous crash of multiple workers
    - Test: Supervisor ignores expected shutdowns (StopWorkerAsync)
  </behavior>
  <action>
1. **KernelSupervisor** (`Infrastructure/Process/KernelSupervisor.cs`):
   - Constructor accepts `IProcessIsolation`, `IoProcessManager`, `WalStateReconstructor`
   - Subscribe to `ProcessIsolationManager.WorkerCrashed` event
   - On crash event:
     a. Log crash with worker ID, plugin ID, exit code
     b. Check circuit breaker: if >5 restarts for this worker in last 60s, mark as permanently down and enter degradation mode
     c. Start timer (Stopwatch) for restart latency measurement
     d. Call `WalStateReconstructor.ReconstructAsync(workerId)` to get recovery payload
     e. Call `IProcessIsolation.SpawnWorkerAsync(pluginId)` to create new worker
     f. Send recovery payload via IPC to new worker
     g. Update routing tables (replace old worker ID with new)
     h. Stop timer — assert <100ms in tests, log actual latency
   - Graceful degradation: When worker marked permanently down:
     - GPU worker: route operations to CPU fallback strategies
     - I/O worker: route to standard filesystem I/O
     - Compute worker: route to in-process execution (degraded performance, maintained correctness)
   - `SupervisorStatus` property: `Running`, `Degraded(list of down workers)`, `Stopped`

2. **Circuit breaker** implementation:
   - Per-worker `CircularBuffer<DateTime>` of last N restart timestamps
   - Trip condition: >5 entries in 60-second window
   - Reset: manual `ResetCircuitBreaker(workerId)` or automatic after 5 minutes cooldown
   - On trip: fire `WorkerPermanentlyDown` event, switch to degradation mode

3. Mark `[SdkCompatibility("8.0.0", Notes = "Phase 133: Kernel supervisor")]`.

   - Handle corrupt entries: skip with warning log, don't abort reconstruction
   - Performance: stream entries without loading full WAL into memory
