<objective>
Implement process isolation for CPU-intensive plugins with shared-nothing worker processes and IPC.

Purpose: ADV-04 requires multi-process shared-nothing architecture. Currently the kernel runs all 52 plugins in a single process. CPU-intensive plugins (Compression, Encryption, Compute, Intelligence) should run in isolated worker processes to prevent CPU starvation and enable per-process crash isolation.

Output: IProcessIsolation contract, ProcessIsolationManager, IpcChannel, WorkerProcessHost, tests.
</objective>

1. Create `IProcessIsolation` contract in `DataWarehouse.SDK/Contracts/Process/`:
   ```
   public interface IProcessIsolation
   {
       Task<WorkerProcessInfo> SpawnWorkerAsync(string pluginId, CancellationToken ct);
       Task<IpcMessage> SendAsync(string workerId, IpcMessage message, CancellationToken ct);
       Task StopWorkerAsync(string workerId, CancellationToken ct);
       IReadOnlyDictionary<string, WorkerProcessInfo> ActiveWorkers { get; }
   }
   ```
   Plus records: `WorkerProcessInfo` (Id, ProcessId, PluginId, StartTime, Status), `IpcMessage` (Type, Payload as byte[], CorrelationId), `WorkerStatus` enum

2. Create `IpcChannel` in `DataWarehouse.SDK/Infrastructure/Process/`:
   - Cross-platform: uses `NamedPipeServerStream`/`NamedPipeClientStream` on Windows, Unix domain `Socket` on Linux/macOS
   - `RuntimeInformation.IsOSPlatform(OSPlatform.Windows)` for platform detection
   - Framing protocol: 4-byte length prefix + payload (handles partial reads)
   - `SendAsync(IpcMessage)` / `ReceiveAsync()` with `CancellationToken`
   - Connection timeout: 5 seconds
   - Auto-reconnect: on `IOException`, close and reopen with backoff (100ms, 200ms, 400ms, max 3 retries)
   - Max message size: 4MB (configurable)
   - Thread-safe: uses `SemaphoreSlim` for send serialization

  <action>
1. **ProcessIsolationManager** (`Infrastructure/Process/ProcessIsolationManager.cs`):
   - Implements `IProcessIsolation`
   - `SpawnWorkerAsync`: Start `dotnet DataWarehouse.WorkerHost.dll --plugin {pluginId} --ipc-endpoint {pipeName}` as child process. Create `IpcChannel` for the connection. Register in `ActiveWorkers` dictionary.
   - `SendAsync`: Route message to correct worker's IpcChannel, await response with timeout (30s default)
   - `StopWorkerAsync`: Send `IpcMessage(Type="Shutdown")`, wait 5s, then `Process.Kill()` if still alive
   - Monitor worker processes: background task polls `Process.HasExited` every 250ms, fires `WorkerCrashed` event on unexpected exit
   - Shared-nothing: each worker gets unique pipe name, no shared state between workers
   - `ConcurrentDictionary<string, WorkerState>` for worker tracking

2. **WorkerProcessHost** (`Infrastructure/Process/WorkerProcessHost.cs`):
   - Entry point for worker process (loaded by dotnet command)
   - Parses `--plugin` and `--ipc-endpoint` args
   - Creates worker-side `IpcChannel` (client mode)
   - Loads specified plugin DLL via `Assembly.LoadFrom` or plugin-specific factory
   - Creates independent `BoundedCache` instance (not shared with kernel)
   - Message loop: receive IpcMessage, dispatch to plugin strategy, send result back
   - Graceful shutdown on `Shutdown` message type
   - Heartbeat: sends ping every 5s so parent knows worker is alive

3. Shared-nothing verification: worker process must NOT reference kernel state, must NOT share memory.
