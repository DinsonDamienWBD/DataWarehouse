Implement process-per-core I/O with dedicated I/O process per NVMe namespace and CPU affinity pinning.

Purpose: ADV-04 requires process-per-core I/O. SPDK naturally runs single-threaded per queue pair. By dedicating one process per NVMe namespace and pinning each to a CPU core, we achieve maximum I/O throughput with no lock contention.

Output: IoProcessManager, NvmeNamespaceMapper, tests.
</objective>

1. **NvmeNamespaceMapper** (`Infrastructure/Process/NvmeNamespaceMapper.cs`):
   - `Discover() -> IReadOnlyList<NvmeNamespace>`: Detect NVMe namespaces
     - Linux: parse `/sys/class/nvme/*/` and `/dev/nvme*n*`
     - Windows: use `StorageDeviceProperty` via P/Invoke or WMI `Win32_DiskDrive WHERE MediaType='Fixed hard disk media'`
     - Fallback: return empty list if no NVMe detected
   - `MapToCore(NvmeNamespace ns, int totalCores) -> int`: Round-robin core assignment
   - `NvmeNamespace` record: DevicePath, NamespaceId, SizeBytes, SectorSize

2. **IoProcessManager** (`Infrastructure/Process/IoProcessManager.cs`):
   - Constructor accepts `IProcessIsolation` (from 133-01) and `NvmeNamespaceMapper`
   - `StartAsync(CancellationToken ct)`:
     - Discover NVMe namespaces
     - For each namespace: spawn worker process via `IProcessIsolation.SpawnWorkerAsync` with plugin="io-worker"
     - Set CPU affinity on spawned process:
       - Linux: `sched_setaffinity` via P/Invoke (`[DllImport("libc")]`)
       - Windows: `Process.ProcessorAffinity = (IntPtr)(1L << coreIndex)`
     - Configure worker as single-threaded: `Environment.SetEnvironmentVariable("DOTNET_ThreadPool_ForceMinWorkerThreads", "1")`
   - `RouteIoAsync(string namespacePath, IpcMessage ioRequest, CancellationToken ct)`:
     - Find I/O process for target namespace
     - Forward via `IProcessIsolation.SendAsync`
   - `StopAsync`: graceful shutdown of all I/O processes

3. Mark `[SdkCompatibility("8.0.0", Notes = "Phase 133: Process-per-core I/O")]`.
