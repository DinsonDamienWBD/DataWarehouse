   a. `Test_NoNewLOHAllocations_OnZeroAllocPaths()`:
      - Identify zero-alloc hot paths (buffer pool operations, message bus dispatch, strategy resolution).
      - Force GC, capture LOH size before.
      - Execute the hot path code under test.
      - Force GC, capture LOH size after.
      - Assert: LOH delta == 0 (or within epsilon for measurement overhead).

   b. `Test_GCPressure_NoRegression()`:
      - Record Gen0/Gen1/Gen2 collection counts before test batch.
      - Run all non-Coyote hardening tests programmatically (using xUnit runner API or direct method invocation).
      - Record Gen0/Gen1/Gen2 collection counts after.
      - Assert: Gen2 collections did not increase by more than a defined threshold (e.g., max 2 Gen2 collections for the entire test suite run).
      - Report per-generation GC rates.

   c. `Test_WorkingSetBounded()`:
      - Monitor `Process.GetCurrentProcess().WorkingSet64` at intervals during test execution.
      - Assert: working set does not grow monotonically (allow spikes but require stabilization).

4. IMPORTANT: Exclude Coyote test classes from profiling — Coyote's scheduler creates synthetic allocations.

5. Tests runnable via `dotnet test --filter "FullyQualifiedName~DotMemoryAllocationTests"`.
