- `FipsEnforcementMode` enum: `Disabled = 0`, `Audit = 1`, `Strict = 2`

Create `DataWarehouse.SDK/Security/FipsModeEnforcer.cs`:
- `IFipsModeEnforcer` interface with: `FipsEnforcementMode CurrentMode { get; }`, `FipsValidationResult ValidateAlgorithm(string algorithmId)`, `void EnforceAlgorithm(string algorithmId)` (throws in Strict), `FipsValidationResult ValidateCipherSuite(string storageCipher, string transitCipher, string kdf, string hashAlgorithm)`
- `FipsModeEnforcer` implementation: Constructor takes `FipsEnforcementMode mode`. `ValidateAlgorithm` checks against hardcoded FIPS-approved set (AES-128, AES-256, AES-128-GCM, AES-256-GCM, AES-256-XTS, SHA-256, SHA-384, SHA-512, HMAC-SHA-256, HMAC-SHA-384, HMAC-SHA-512, ECDSA-P256, ECDSA-P384, RSA-2048, RSA-3072, RSA-4096, PBKDF2-SHA256, PBKDF2-SHA384, PBKDF2-SHA512). - Non-FIPS algorithms to explicitly reject: ChaCha20, ChaCha20-Poly1305, XChaCha20, BLAKE2, BLAKE3, Serpent, Twofish, Camellia, Argon2id, SHA3-256, SHA3-512, ML-KEM, Kyber, custom.
