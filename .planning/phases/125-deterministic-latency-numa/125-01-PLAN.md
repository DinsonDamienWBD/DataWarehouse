    **P/Invoke declarations (add to NumaAllocator or separate NativeNuma static class):**

    Windows (kernel32.dll):
    ```csharp
    [DllImport("kernel32.dll", SetLastError = true)]
    static extern bool GetNumaHighestNodeNumber(out uint highestNodeNumber);

    [DllImport("kernel32.dll", SetLastError = true)]
    static extern bool GetNumaNodeProcessorMask(byte node, out ulong processorMask);

    [DllImport("kernel32.dll", SetLastError = true)]
    static extern IntPtr VirtualAllocExNuma(IntPtr hProcess, IntPtr lpAddress, UIntPtr dwSize, uint flAllocationType, uint flProtect, uint nndPreferred);

    [DllImport("kernel32.dll", SetLastError = true)]
    static extern bool VirtualFree(IntPtr lpAddress, UIntPtr dwSize, uint dwFreeType);
    ```

    Linux (libnuma.so.1):
    ```csharp
    [DllImport("libnuma.so.1")]
    static extern int numa_available();

    [DllImport("libnuma.so.1")]
    static extern IntPtr numa_alloc_onnode(UIntPtr size, int node);

    [DllImport("libnuma.so.1")]
    static extern void numa_free(IntPtr start, UIntPtr size);

    [DllImport("libnuma.so.1")]
    static extern int numa_max_node();
    ```

    **Enhanced DetectWindowsNumaTopology:**
    - Call GetNumaHighestNodeNumber → get node count
    - For each node: GetNumaNodeProcessorMask → populate CpuCores
    - Query GlobalMemoryStatusEx for total memory, distribute by node count (or use GetNumaAvailableMemoryNode if available)
    - Build DistanceMatrix: Windows doesn't expose NUMA distances directly — use heuristic (10 for local, 20 for remote) or skip

    **Enhanced DetectLinuxNumaTopology:**
    - Parse /sys/devices/system/node/node*/cpulist → populate CpuCores (format "0-31,64-95")
    - Parse /sys/devices/system/node/distance → populate DistanceMatrix
    - Parse /sys/devices/system/node/node*/meminfo MemTotal → TotalMemoryBytes (already done)
    - List /sys/devices/system/node/node*/device/pci* → populate LocalDevices

    **Upgrade Allocate(sizeInBytes, preferredNodeId):**
    - If not NUMA-aware: return new byte[sizeInBytes] (unchanged)
    - If NUMA-aware on Windows:
      1. Try VirtualAllocExNuma for preferred node
      2. If fails (returns IntPtr.Zero): try adjacent node from DistanceMatrix (lowest distance != 10)
      3. If fails: VirtualAlloc without NUMA preference (any node)
      4. Wrap native pointer in a managed byte[] using Span/unsafe or track allocation for Dispose
    - If NUMA-aware on Linux:
      1. Try numa_alloc_onnode for preferred node
      2. If fails (returns IntPtr.Zero): try adjacent node
      3. If fails: numa_alloc_local (any local node)
      4. Same wrapping strategy

    IMPORTANT: Native NUMA allocations return IntPtr, not managed byte[]. Either:
    - Return byte[] via Marshal.Copy (simpler, but copies data) — acceptable for initial implementation
    - Or use UnmanagedMemoryManager<byte> to wrap native pointer as Memory<byte> — preferred for zero-copy
    - Document the approach chosen. For this phase, Marshal.Copy is acceptable since the allocation itself is NUMA-pinned even if the managed copy is not. The key benefit is that the data written to the buffer stays on the correct NUMA node.

    **Upgrade GetDeviceNumaNode on Windows:**
    - Use SetupDiGetDeviceProperty with DEVPKEY_Device_Numa_Proximity_Domain
    - Or simpler: parse device path to PCI bus, map bus to NUMA node via registry

    **Memory tracking for Dispose:**
    - Track all native allocations in a ConcurrentDictionary<IntPtr, (UIntPtr size, bool isWindows)>
    - On Dispose: iterate and call VirtualFree (Windows) or numa_free (Linux) for each

    Tests should use conditional skips for platform-specific tests: `[SkipOnPlatform("Windows")]` for libnuma tests and vice versa. Also test that on single-socket systems (most CI), the fallback path works correctly.
