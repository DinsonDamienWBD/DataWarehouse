</objective>

    **ProcessorAffinityManager.cs** in `DataWarehouse.SDK/Hardware/Cpu/`:

    Constructor: INumaAllocator numaAllocator, ILogger logger

    **SetThreadAffinity(Thread thread, int[] coreIds):**
    - Windows: Set thread.ProcessorAffinity = bitmask of coreIds (e.g., cores [0,1,2,3] → 0xF)
    - Linux: P/Invoke sched_setaffinity(pid_t pid, size_t cpusetsize, cpu_set_t* mask) from libc
    - Validate all coreIds are within [0, Environment.ProcessorCount)

    **PinToNumaNode(Thread thread, int numaNodeId):**
    - Get CpuCores for numaNodeId from INumaAllocator.Topology
    - Call SetThreadAffinity with those cores
    - If non-NUMA: log warning and return (no-op)

    **PinIoThreadToDevice(Thread thread, string devicePath):**
    - Get NUMA node via INumaAllocator.GetDeviceNumaNode(devicePath)
    - Call PinToNumaNode with that node
    - Log: "Pinned I/O thread {threadId} to NUMA node {nodeId} (device: {devicePath})"

    **GetCurrentAffinity(Thread thread) -> int[]:**
    - Windows: Read thread.ProcessorAffinity
    - Linux: sched_getaffinity

    **GenerateIsolcpusRecommendation(int[] dedicatedCores) -> string:**
    - Generate Linux kernel parameter string: "isolcpus=X,Y,Z" or "isolcpus=X-Y,Z-W" (range format)
    - These cores will be excluded from the general scheduler, dedicated to DataWarehouse I/O

    **P/Invoke for Linux:**
    ```csharp
    [DllImport("libc", SetLastError = true)]
    static extern int sched_setaffinity(int pid, UIntPtr cpusetsize, ref ulong mask);

    [DllImport("libc", SetLastError = true)]
    static extern int sched_getaffinity(int pid, UIntPtr cpusetsize, ref ulong mask);
    ```

    Note: pid=0 means current thread. cpu_set_t is a bitmask — for up to 64 cores use ulong, for >64 use ulong[] array.

  <action>
    **NumaAwareTaskScheduler.cs** — Custom TaskScheduler (extends System.Threading.Tasks.TaskScheduler):

    Constructor: INumaAllocator numaAllocator, ProcessorAffinityManager affinityManager, NumaSchedulerOptions options, ILogger logger

    **Design:**
    - Create dedicated thread pool per NUMA node: `Dictionary<int, BlockingCollection<Task>>`
    - Each NUMA node gets `options.ThreadsPerNode` threads (default: CpuCores.Count on that node)
    - Threads are pinned to their NUMA node via ProcessorAffinityManager.PinToNumaNode at creation
    - Tasks are tagged with preferred NUMA node via AsyncLocal<int> or custom Task state

    **QueueTask(Task task):**
    - Check task for NUMA node preference (via TaskCreationOptions or custom state)
    - If preference specified: queue to that node's BlockingCollection
    - If no preference: round-robin across nodes for load balancing

    **TryExecuteTaskInline(Task task, bool taskWasPreviouslyQueued):**
    - Only inline if current thread is on the correct NUMA node
    - Otherwise return false (force queuing)

    **GetScheduledTasks():**
    - Return snapshot of all pending tasks across all node queues

    **NumaSchedulerOptions:**
    ```
    NumaSchedulerOptions {
        ThreadsPerNode = 0 (0 = auto: use CpuCores.Count per node)
        MaxQueueDepthPerNode = 10000
        ThreadPriority = ThreadPriority.AboveNormal (for I/O threads)
    }
    ```

    **Non-NUMA fallback:** If numaAllocator.IsNumaAware is false, queue all tasks to ThreadPool.UnsafeQueueUserWorkItem (standard behavior, no pinning).

    **Dispose:** Signal all BlockingCollections to complete, join all threads with timeout.
