    **2. Achieving Deterministic Latency — Configuration Guide**

    Section A: NUMA Pinning
    - Detect NUMA topology: `dw hardware numa-info`
    - Configure NUMA-aware allocation in dw-config.json: `"numa": { "enabled": true, "pin_io_to_device_node": true }`
    - Verify: check logs for "NUMA topology detected: {N} nodes"

    Section B: CPU Isolation
    - Linux: add `isolcpus=X-Y` to GRUB_CMDLINE_LINUX
    - Configure DataWarehouse to use isolated cores: `"cpu_isolation": { "io_cores": [4,5,6,7], "compute_cores": [8,9,10,11] }`
    - Verify: `cat /sys/devices/system/cpu/isolated` shows expected cores

    Section C: Block Device Selection
    - Decision matrix: use case → recommended block device
    - SPDK for lowest latency (requires vfio-pci, root access)
    - NVMe direct for good latency without SPDK complexity
    - FileBlockDevice for portability

    Section D: OS Tuning
    - Linux: `vm.swappiness=0`, `transparent_hugepages=never`, `nmi_watchdog=0`
    - Set thread priority: `"thread_priority": "AboveNormal"`
    - Disable frequency scaling: `cpupower frequency-set -g performance`

    **3. SLA Template**
    - "DataWarehouse guarantees P99 read latency <= {X} us for 4KB random reads on {BlockDevice} with {Tier} deployment, when configured per this guide."
    - Conditions: NUMA pinning enabled, CPU isolation configured, OS tuned per Section D
    - Measurement methodology: sustained 10-minute test, exclude first 30s warmup

    - `dw benchmark latency --device {path} --duration 600` command
