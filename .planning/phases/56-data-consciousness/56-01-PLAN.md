Create `DataWarehouse.SDK/Contracts/Consciousness/ConsciousnessScore.cs` in namespace `DataWarehouse.SDK.Contracts.Consciousness`.

Define the following types:

1. `ConsciousnessGrade` enum: Enlightened (>=90), Aware (>=75), Dormant (>=50), Dark (>=25), Unknown (<25) — mirrors QualityGrade pattern from UltimateDataQuality.

2. `ConsciousnessAction` enum: Retain, Archive, Review, Purge, Quarantine — recommended lifecycle action.

3. `ValueDimension` enum: AccessFrequency, Lineage (how many downstream dependents), Uniqueness (is data duplicated elsewhere), Freshness (age vs staleness), BusinessCriticality, ComplianceValue.

4. `LiabilityDimension` enum: PIIPresence, PHIPresence, PCIPresence, ClassificationLevel, RetentionObligation, RegulatoryExposure, BreachRisk.

5. `ValueScore` record: double OverallScore (0-100), Dictionary<ValueDimension, double> DimensionScores, List<string> ValueDrivers, DateTime ScoredAt.

6. `LiabilityScore` record: double OverallScore (0-100), Dictionary<LiabilityDimension, double> DimensionScores, List<string> LiabilityFactors, string[] DetectedPIITypes, string[] ApplicableRegulations, DateTime ScoredAt.

7. `ConsciousnessScore` record: string ObjectId, double CompositeScore (0-100), ConsciousnessGrade Grade (computed from CompositeScore), ConsciousnessAction RecommendedAction (computed from value+liability quadrant), ValueScore Value, LiabilityScore Liability, string ScoringStrategy, DateTime ScoredAt, Dictionary<string, object> Metadata.

The RecommendedAction logic: high-value+low-liability=Retain, high-value+high-liability=Review, low-value+low-liability=Archive, low-value+high-liability=Purge. Threshold: 50 for high/low boundary.

8. `ConsciousnessScoringConfig` record: Dictionary<ValueDimension, double> ValueWeights (must sum to 1.0, default even distribution), Dictionary<LiabilityDimension, double> LiabilityWeights (must sum to 1.0), double ValueLiabilityRatio (default 0.6 — value contributes 60% to composite), double ArchiveThreshold (default 30), double PurgeThreshold (default 20), double ReviewThreshold (default 70), int ScoringBatchSize (default 1000).

Use `sealed record` for score types (immutable). Use XML doc comments on everything. Follow the pattern from RecordQualityScore in UltimateDataQuality.
