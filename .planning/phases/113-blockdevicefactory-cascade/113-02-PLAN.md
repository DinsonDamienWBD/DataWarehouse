    1. Add a static `TryCreate` method that the factory cascade can call:
       ```csharp
       /// <summary>
       /// Attempts to create an IoRing-backed block device. Returns null if IoRing is not
       /// available on this Windows version (pre-Win11).
       /// </summary>
       public static IoRingBlockDevice? TryCreate(
           string path, int blockSize, long blockCount, bool createNew,
           BlockDeviceOptions? options = null)
       {
           if (!IsSupported) return null;

           var fileOptions = FileOptions.Asynchronous | FileOptions.RandomAccess
               | FileOptions.WriteThrough | (FileOptions)0x20000000; // FILE_FLAG_NO_BUFFERING
           var mode = createNew ? FileMode.CreateNew : FileMode.Open;
           var handle = File.OpenHandle(path, mode, FileAccess.ReadWrite, FileShare.None, fileOptions);

           if (createNew)
           {
               RandomAccess.SetLength(handle, (long)blockSize * blockCount);
           }

           try
           {
               int ringSize = options?.QueueDepth ?? 64;
               return new IoRingBlockDevice(handle, blockSize, blockCount, ringSize);
           }
           catch (PlatformNotSupportedException)
           {
               handle.Dispose();
               return null;
           }
       }
       ```
       This opens the file handle internally (same flags as WindowsOverlappedBlockDevice) so the factory doesn't need to manage handle lifecycle separately.

    2. Add note that `TryCreate` manages the file handle -- the IoRingBlockDevice must dispose it. Currently the class does NOT own the handle. Add a `_ownsHandle` bool field:
       - Set `_ownsHandle = false` in existing constructor (caller owns handle).
       - In `TryCreate`, after constructing, set `_ownsHandle = true` via an internal setter or second constructor overload.
       - In `DisposeAsync`, if `_ownsHandle`, dispose `_fileHandle` after releasing the DangerousRef.
       - Use a private constructor overload: `private IoRingBlockDevice(SafeFileHandle handle, int blockSize, long blockCount, int ringSize, bool ownsHandle)` called by TryCreate.

    **WindowsOverlappedBlockDevice changes:**

    1. Add a static `TryCreate` method (simpler since it always works on Windows):
       ```csharp
       /// <summary>
       /// Creates a WindowsOverlapped-backed block device. Always succeeds on Windows 10+.
       /// Returns null on non-Windows platforms.
       /// </summary>
       public static WindowsOverlappedBlockDevice? TryCreate(
           string path, int blockSize, long blockCount, bool createNew,
           BlockDeviceOptions? options = null)
       {
           if (!OperatingSystem.IsWindows()) return null;
           int concurrency = options?.QueueDepth ?? 64;
           return new WindowsOverlappedBlockDevice(path, blockSize, blockCount, createNew, concurrency);
       }
       ```

    Do NOT change any existing read/write/batch/flush/dispose logic. Only add factory entry points.
