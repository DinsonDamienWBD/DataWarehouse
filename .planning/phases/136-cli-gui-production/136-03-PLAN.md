  <name>Task 1: Create DynamicCommandBridge</name>
  <files>DataWarehouse.CLI/Integration/DynamicCommandBridge.cs</files>
  <action>
Create DynamicCommandBridge class that:
1. Takes DynamicCommandRegistry and the CLI RootCommand in constructor
2. Subscribes to OnPluginLoaded — for each new capability, creates a System.CommandLine Command and adds to root
3. Subscribes to OnPluginUnloaded — removes commands for that plugin's capabilities
4. Subscribes to OnCapabilityChanged — enables/disables specific commands
5. Uses IsCommandAvailable() to gate execution — if user runs a command for an unloaded plugin, show helpful message: "Plugin {X} is not loaded. Install with: dw plugin install {X}"
6. Updates tab completion list on every change (invalidate completion cache)
7. Thread-safe — registry events may fire from background threads

Pattern: The bridge is a long-lived service started in Program.cs after kernel init. It runs for the lifetime of the CLI process (important for InteractiveMode).
