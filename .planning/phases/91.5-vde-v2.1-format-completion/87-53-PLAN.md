**OnlineResizeManager.cs** -- class with `[SdkCompatibility("6.0.0")]`:
- Constructor: `OnlineResizeManager(OperationJournalRegion journal, int blockSize)`
- `public const int CheckpointIntervalBlocks = 4096` -- checkpoint OPJR every 4096 blocks per spec

Core methods:
- `ResizeOperation BeginResize(ulong currentTotalBlocks, ulong newTotalBlocks)`:
  1. Validates newTotalBlocks != currentTotalBlocks
  2. Creates OPJR entry (type=RESIZE, totalUnits=abs(newTotalBlocks-currentTotalBlocks), payload stores newTotalBlocks as 8 LE bytes)
  3. Returns `ResizeOperation` with OperationId, PendingTotalBlocks, ResizeJournalOperationId
  4. Caller must atomically write PendingTotalBlocks + ResizeJournalOperationId to Superblock

- `ResizeCheckpoint ProcessBlocks(ulong operationId, ulong blocksProcessedSoFar)`:
  1. If (blocksProcessedSoFar % CheckpointIntervalBlocks == 0), checkpoint to OPJR
  2. Calculate ProgressPercent = (blocksProcessedSoFar * 1000000) / TotalUnits
  3. Return ResizeCheckpoint indicating if checkpoint was written

- `void CompleteResize(ulong operationId)`:
  1. Transition OPJR state to COMPLETED
  2. Caller must clear PendingTotalBlocks + ResizeJournalOperationId in Superblock

- `ResizeRecoveryInfo? CheckForPendingResize()`:
  1. Scans OPJR for RESIZE operations in INPROGRESS state
  2. Returns recovery info (operationId, completedUnits, newTotalBlocks from payload) or null
  3. Caller resumes from CompletedUnits on crash recovery

Supporting types:
- `ResizeOperation` readonly struct: `ulong OperationId, ulong PendingTotalBlocks, ulong ResizeJournalOperationId`
- `ResizeCheckpoint` readonly struct: `bool WasCheckpointed, ulong CompletedUnits, uint ProgressPercent`
- `ResizeRecoveryInfo` readonly struct: `ulong OperationId, ulong CompletedUnits, ulong NewTotalBlocks`
