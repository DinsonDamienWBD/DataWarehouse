Shannon entropy calculation:
- `double ComputeShannonEntropy(ReadOnlySpan<byte> data)`:
  1. Build byte frequency histogram (256 buckets)
  2. Compute Shannon entropy: H = -sum(p_i * log2(p_i)) for all p_i > 0
  3. Result range: 0.0 (uniform) to 8.0 (maximum randomness)

Baseline tracking:
- `EntropyBaseline` struct: `double Mean, double StandardDeviation, long SampleCount`
- Exponential moving average for mean and variance
- `void UpdateBaseline(double entropy)` -- update running statistics
- `bool IsBaselineTrusted` -- true when SampleCount >= MinWritesBeforeBaseline

Anomaly detection:
- `AnomalyResult AnalyzeWrite(ReadOnlySpan<byte> writeData)`:
  1. Compute Shannon entropy of write data
  2. If not baseline trusted: compare against static thresholds (EntropyThreshold)
  3. If baseline trusted: compare against baseline.Mean + AnomalyDeviationFactor * baseline.StdDev
  4. Also check if entropy > EntropyThreshold (absolute ceiling)
  5. Return AnomalyResult
- `AnomalyResult` readonly struct:
  - `double Entropy, double BaselineMean, double BaselineStdDev`
  - `bool IsAnomaly, string? AnomalyReason`
  - `double DeviationsFromBaseline` -- how many stddevs above baseline

Sliding window:
- `CircularBuffer<double> _entropyWindow` of WindowSizeBlocks entries
- Smooths entropy readings to avoid single-block false positives

Panic fork trigger:
- `Task<PanicForkResult?> EvaluateAndForkAsync(ReadOnlySpan<byte> writeData, Func<string, bool, Task<long>> createSnapshotFunc, CancellationToken ct)`:
  1. Compute entropy, update baseline
  2. Add to sliding window
  3. If anomaly detected AND rate limit not exceeded:
     a. Call createSnapshotFunc(snapshotName, isReadOnly=true) to create immutable snapshot
     b. Record fork timestamp for rate limiting
     c. Fire OnPanicFork event
     d. Return PanicForkResult
  4. If no anomaly: return null

- `PanicForkResult` readonly struct:
  - `long SnapshotId, double TriggeringEntropy, double BaselineEntropy`
  - `string Reason, DateTimeOffset Timestamp`
  - `bool WriteBlocked` -- true if write was rejected (aggressive mode)

Events and monitoring:
- `event Action<PanicForkResult>? OnPanicFork` -- alert callback
- `EntropyMonitorStats GetStats()`:
  - `EntropyMonitorStats` readonly struct: `long WritesAnalyzed, int PanicForksTriggered, double CurrentBaselineMean, double CurrentBaselineStdDev, double LastEntropy`
