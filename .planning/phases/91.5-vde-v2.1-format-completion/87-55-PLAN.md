Serialization:
- `void Serialize(Span<byte> buffer, int blockSize)` -- writes single block at block 14 with RCVR trailer. Writes RecoveryMagic at +0x00, then all fields. HMAC covers bytes [0x00..0x29B]. Use UniversalBlockTrailer with BlockTypeTag RCVR.
- `static EmergencyRecoveryBlock Deserialize(ReadOnlySpan<byte> buffer, int blockSize)` -- reads and validates magic
- `byte[] ComputeHmac(byte[] hmacKey)` -- HMAC-BLAKE3 over bytes [0x00..0x29B]. Use System.Security.Cryptography.HMACSHA256 as stand-in (BLAKE3 not in BCL; document that production should use BLAKE3 when available). Key derived from hardware_serial + recovery_passphrase.
- `bool VerifyHmac(byte[] hmacKey)` -- recomputes and compares

Helper for UTF-8 string read/write into fixed-size null-padded fields:
- `static void WriteFixedString(Span<byte> dest, string value, int maxBytes)` -- UTF-8 encode, null-pad remainder
- `static string ReadFixedString(ReadOnlySpan<byte> source, int maxBytes)` -- read until null or end

Critical: This block is NEVER encrypted. Contains only identification and contact info.

**RecoveryFooter.cs** -- class with `[SdkCompatibility("6.0.0")]`:
- `public const int FooterSizeBlocks = 1` -- always 1 block at EOF

Properties:
- `byte[] SuperblockMirror` -- raw bytes of Superblock Block 0 (up to blockSize)
- `uint Generation` -- generation number for the footer

Methods:
- `static long GetFooterOffset(long fileSizeBytes, int blockSize)` -- returns `fileSizeBytes - blockSize` (footer is last block of VDE file)
- `void Serialize(Span<byte> buffer, int blockSize)` -- copies SuperblockMirror into buffer, writes SUPB trailer (same block type as Superblock)
- `static RecoveryFooter Deserialize(ReadOnlySpan<byte> buffer, int blockSize)` -- reads footer, verifies SUPB trailer
- `bool ShouldUpdate(RecoveryFooterUpdateReason reason)` -- returns true for CleanUnmount or EpochBoundary, false for NormalWrite

`RecoveryFooterUpdateReason` enum: `CleanUnmount, EpochBoundary, VdeGrow, VdeShrink, NormalWrite`

Document the update protocol per spec:
- On clean unmount: footer updated to match current Superblock Block 0
- On epoch boundary: footer updated asynchronously
- On VDE grow: write new footer at new EOF, mark old footer block RESERVED, update Superblock, then free old
- On VDE shrink: footer moves first, then free space released

**FailoverState.cs** -- readonly struct with `[SdkCompatibility("6.0.0")]`, IEquatable:
- `public const int Size = 36` (per spec, Superblock +0x188 to +0x1AB)

`FailoverRole` enum (byte):
- `Standalone = 0x00, Secondary = 0x01, Promoting = 0x02, Primary = 0x03, Demoting = 0x04`

`FailoverFlags` flags enum (byte):
- `None = 0, FencingActive = 1 << 0, SplitBrainDetected = 1 << 1, WitnessRequired = 1 << 2`

Properties matching spec:
- `FailoverRole Role` (+0x00)
- `FailoverFlags Flags` (+0x01)
- `ushort PeerCount` (+0x02)
- `ulong FailoverEpoch` (+0x04, monotonic, higher = more recent)
- `Guid PrimaryNodeUuid` (+0x0C, 16 bytes)
- `ulong LastFailoverUtcTicks` (+0x1C, UTC nanoseconds)

State machine validation:
- `static bool IsValidTransition(FailoverRole from, FailoverRole to)`:
  - Standalone -> Secondary (join cluster)
  - Secondary -> Promoting (begin promotion)
  - Promoting -> Primary (promotion complete)
  - Primary -> Demoting (begin demotion)
  - Demoting -> Secondary (demotion complete, re-sync)

Split-brain resolution:
- `static FailoverRole ResolveSplitBrain(ulong localEpoch, ulong remoteEpoch)` -- higher epoch wins (returns Primary), lower demotes (returns Promoting for hold-off). Equal epochs: both set SplitBrainDetected, neither promotes without witness.

Serialization:
- `void WriteTo(Span<byte> buffer)` -- writes 36 bytes
- `static FailoverState ReadFrom(ReadOnlySpan<byte> buffer)` -- reads 36 bytes
