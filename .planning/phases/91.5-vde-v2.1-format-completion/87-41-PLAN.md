Export pipeline:
- `Task<ExportResult> ExportAsync(Stream outputStream, CancellationToken ct)`:
  1. Selection phase: scan source inode table, apply predicates to select inodes
  2. Layout planning: calculate target VDE size (metadata + selected data blocks)
  3. Write Superblock Group (4 blocks): new VolumeUUID, correct TotalBlocks/FreeBlocks, FormatMajor=2/Minor=1
  4. Write Mirror Superblock Group (4 blocks)
  5. Write Region Directory (2 blocks): region pointers for metadata + data regions
  6. Write Allocation Bitmap: mark exported blocks as allocated
  7. Write Inode Table: copy selected inodes with remapped block addresses
     - Old extent StartBlock -> new sequential StartBlock in exported data region
     - Build block address remap table: Dictionary<long, long> (old -> new)
  8. Write Integrity Tree: recompute Merkle tree for exported subset
  9. Write Data Region: copy data blocks sequentially (compacted layout)
     - For each selected inode's extents: read source blocks, write to new sequential position
  10. Write final block trailers (TRLR blocks every 256 blocks in data region)
  11. Flush and finalize

Block address remapping:
- `Dictionary<long, long> _blockRemap` -- maps source block address to destination address
- Sequential allocation: first selected extent -> block 0 of data region, etc.
- Update all InodeExtent.StartBlock references to new addresses

- `ExportResult` readonly struct:
  - `long TotalInodesExported, long TotalBlocksExported, long ExportSizeBytes`
  - `Guid ExportedVolumeUuid` -- new UUID for the exported VDE
  - `TimeSpan Duration`
  - `bool SizeLimitReached`

Validation:
- `Task<bool> ValidateExportAsync(Stream exportedStream, CancellationToken ct)`:
  - Read Superblock, verify magic and version
  - Verify Region Directory points to valid regions
  - Spot-check a few data blocks via TRLR checksums
