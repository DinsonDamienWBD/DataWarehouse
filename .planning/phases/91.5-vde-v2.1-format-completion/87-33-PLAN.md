Heat tracking:
- `ConcurrentDictionary<long, HeatRecord> _heatMap` -- keyed by StartBlock
- `HeatRecord` readonly struct: `long AccessCount, DateTimeOffset LastAccessed, HeatTier CurrentTier`
- `void RecordAccess(long startBlock)` -- increment access count, update last accessed
- `HeatTier ClassifyExtent(long startBlock)` -- apply thresholds to determine tier

Background migration:
- `Task<MigrationResult> RunMigrationCycleAsync(CancellationToken ct)`:
  1. Scan heat map for misplaced extents (current shard != target shard for tier)
  2. Sort candidates by priority (hottest misplaced-cold first, coldest misplaced-hot first)
  3. For each candidate (within budget):
     a. Allocate blocks in target shard's allocation group
     b. Copy extent data from source to target blocks
     c. Update extent pointer with new StartBlock
     d. Free source blocks
  4. Return MigrationResult

- `MigrationResult` readonly struct:
  - `int ExtentsMigrated, long BlocksMoved, long BytesMoved`
  - `int HotToCorrect, int ColdToCorrect` -- breakdown by direction
  - `TimeSpan Duration, bool BudgetExhausted`

Continuous operation:
- `Task RunContinuousAsync(CancellationToken ct)` -- periodic scan + migrate loop

Allocation integration:
- `long AllocateWithHeatHint(int blockCount, HeatTier hint)` -- allocate in shard matching hint
