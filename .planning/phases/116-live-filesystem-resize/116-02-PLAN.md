<objective>
Create VdeShrinkEngine for offline volume shrink operations.

Purpose: Enable VDE volumes to shrink offline. The engine first identifies the used extent range, runs the OnlineDefragmenter to compact all used blocks toward low addresses, verifies the compaction succeeded, then truncates the allocation bitmap and removes allocation groups beyond the new boundary. Refuses shrink if used blocks exceed target size. All steps journaled via OPJR.

Output: VdeShrinkEngine class with ShrinkAsync method, plus comprehensive TDD tests.
</objective>

    1. **ShrinkRefusalException**: Custom exception with `UsedBlocks`, `TargetTotalBlocks`, and descriptive message. Thrown when shrink is impossible because used blocks exceed target.

    2. **ShrinkResult** record: `(long PreviousTotalBlocks, long NewTotalBlocks, long BlocksRemoved, int AllocationGroupsRemoved, long BlocksDefragmented, TimeSpan Duration)`.

    3. **VdeShrinkEngine** class:
       - Constructor: `(OnlineResizeManager resizeManager, BitmapAllocator bitmap, AllocationGroupDescriptorTable agTable, OnlineDefragmenter defragmenter, IBlockDevice device, int blockSize)`
       - `ShrinkAsync(long currentTotalBlocks, long newTotalBlocks, CancellationToken ct)`:
         a. Validate newTotalBlocks < currentTotalBlocks and newTotalBlocks > 0
         b. Scan bitmap to find highest used block index. If highestUsed >= newTotalBlocks, calculate total used blocks. If totalUsed > newTotalBlocks, throw ShrinkRefusalException.
         c. Call `resizeManager.BeginResize(current, new)` to start OPJR journal
         d. If highestUsed >= newTotalBlocks: run defragmenter to compact all extents below newTotalBlocks. The defragmenter relocates blocks from high addresses to free space at low addresses.
         e. Post-defrag safety check: rescan bitmap to confirm no used blocks at or above newTotalBlocks. If any remain, throw InvalidOperationException("Defragmentation did not compact all blocks below target").
         f. Truncate bitmap: reduce internal byte[] to `(newTotalBlocks + 7) / 8` bytes. Use `TruncateBitmap(long newTotalBlocks)` helper.
         g. Remove allocation groups: iterate AGs and dispose/remove any where StartBlock >= newTotalBlocks. For AGs that partially overlap the new boundary, shrink their BlockCount.
         h. Update superblock: new TotalBlocks, recalculate FreeBlocks = newTotalBlocks - usedBlocks, update ExpectedFileSize, CheckpointSequence++
         i. Serialize and write superblock to device
         j. Call `resizeManager.ProcessBlocks` in batches for progress
         k. Call `resizeManager.CompleteResize`
         l. Return ShrinkResult
