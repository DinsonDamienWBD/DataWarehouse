  <action>
    Create `ResizableBlockDeviceAdapter.cs` in `DataWarehouse.SDK/VirtualDiskEngine/Operations/`:

    1. **IResizableBlockDevice** interface (extends IBlockDevice):
       ```csharp
       public interface IResizableBlockDevice : IBlockDevice
       {
           bool CanGrow { get; }
           bool CanShrink { get; }
           Task GrowUnderlyingAsync(long newBlockCount, CancellationToken ct = default);
           Task ShrinkUnderlyingAsync(long newBlockCount, CancellationToken ct = default);
           Task<long> DetectCurrentSizeAsync(CancellationToken ct = default);
       }
       ```

    2. **FileResizableBlockDevice** class:
       - Wraps an existing `FileBlockDevice` or takes a `SafeFileHandle` + blockSize
       - `GrowUnderlyingAsync`: calls `RandomAccess.SetLength(_handle, newBlockCount * blockSize)`, updates internal `_blockCount` field
       - `ShrinkUnderlyingAsync`: calls `RandomAccess.SetLength(_handle, newBlockCount * blockSize)` to truncate, updates `_blockCount`
       - `DetectCurrentSizeAsync`: calls `RandomAccess.GetLength(_handle) / blockSize`
       - Delegates all IBlockDevice read/write/flush to underlying device
       - `CanGrow = true`, `CanShrink = true`

    3. **RawPartitionResizableBlockDevice** class:
       - Wraps `RawPartitionBlockDevice`
       - `GrowUnderlyingAsync`: calls `DetectCurrentSizeAsync` and verifies new size matches (user must have resized partition externally). If new detected size >= newBlockCount, update BlockCount. Otherwise throw InvalidOperationException with message "Partition has not been resized externally. Current size: {detected} blocks, requested: {newBlockCount} blocks."
       - `ShrinkUnderlyingAsync`: throws `NotSupportedException("Raw partition shrink must be performed externally using OS partition tools.")`
       - `DetectCurrentSizeAsync`: re-opens device handle, reads device size via platform API
       - `CanGrow = true`, `CanShrink = false`

    4. Mark all with `[SdkCompatibility("8.0.0", Notes = "Phase 116: Resizable block device adapter (SIO-05)")]`.

    For tests, use temp files for FileResizableBlockDevice tests. RawPartitionBlockDevice tests use mocked interfaces since raw partition access requires root. Test the interface contracts thoroughly.

    Note: FileBlockDevice has `BlockCount` as a get-only property. The resizable adapter must either:
    - Use a mutable wrapper field for BlockCount (preferred approach)
    - Or create a new FileBlockDevice after resize

    Choose the mutable wrapper approach: FileResizableBlockDevice has its own `_blockCount` field that it updates, and delegates to the underlying handle for I/O.
