  <files>DataWarehouse.Hardening.Tests/Scale/HyperscaleSimulator.cs</files>
  <action>
Create HyperscaleSimulator.cs that simulates PB-scale VDE operations using compressed state:

SimulatedAllocationGroup: Tracks { totalBlocks, usedBlocks, fragmentationRatio } per group (not actual blocks). 8 bytes per group instead of TB of real data.

SimulatedExtentTree: Tracks { nodeCount, depth, avgFanout } without actual B-tree. Models tree growth: each insert increments nodeCount, depth = log(nodeCount, avgFanout).

SimulatedShardCatalog: Tracks { shardId, logicalSize, inodeCount, hotness } per shard. Shard placement algorithm: consistent hashing with virtual nodes.

HyperscaleSimulator class:
- Constructor(long totalCapacityBytes, int shardCount, int allocationGroupsPerShard)
- AllocateFile(long sizeBytes): deterministically allocates to best-fit group, updates compressed state
- DeleteFile(long fileId): marks as freed, tracks fragmentation
- RunGC(): simulates garbage collection, compacts fragmented groups
- RebalanceShards(): moves hottest shard's data to coldest shard
- GetMetrics(): returns { totalAllocated, fragmentationPct, avgTreeDepth, shardImbalance }
- Deterministic: seeded Random, same seed = same results

Memory budget: entire 1PB simulation in <100MB RAM (compressed state only).
