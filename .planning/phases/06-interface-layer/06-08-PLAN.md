**B8: Innovation Strategies (Strategies/Innovation/)**

**UnifiedApiStrategy** (StrategyId: "unified-api"):
- Protocol: InterfaceProtocol.REST
- Single API endpoint that accepts requests in ANY protocol format and auto-detects
- HandleRequestAsync: Inspect Content-Type, path pattern, and body structure to detect protocol (REST, GraphQL, gRPC, JSON-RPC), delegate to appropriate internal handler, return response in detected format

**ProtocolMorphingStrategy** (StrategyId: "protocol-morphing"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Accept request in one protocol, transform to another via X-Target-Protocol header
- Maintain transformation rules for each protocol pair

**NaturalLanguageApiStrategy** (StrategyId: "natural-language-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Accept natural language queries in body text, route to Intelligence plugin via message bus (intelligence.nlp.intent) for intent extraction, map to DataWarehouse operations
- Graceful degradation: keyword-based pattern matching when Intelligence unavailable

**VoiceFirstApiStrategy** (StrategyId: "voice-first-api"):
- Protocol: InterfaceProtocol.Custom
- HandleRequestAsync: Process voice-oriented requests (text transcription in body), route to NLP for intent extraction, return SSML-formatted speech response + structured data
- Graceful degradation: text-based responses when voice synthesis unavailable

**IntentBasedApiStrategy** (StrategyId: "intent-based-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Accept high-level intent declarations { "intent": "...", "constraints": {...} }, route to Intelligence for resolution, execute resolved operations
- Graceful degradation: map common intents to predefined operations

**AdaptiveApiStrategy** (StrategyId: "adaptive-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Detect client capabilities from User-Agent, Accept, X-Client-Capabilities headers, adjust response detail level and format accordingly

**SelfDocumentingApiStrategy** (StrategyId: "self-documenting-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Every response includes X-Api-Help header, support ?explain=true for docs, ?example=true for examples, root path returns navigable API map

**PredictiveApiStrategy** (StrategyId: "predictive-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Track per-client request sequences, use Intelligence plugin to predict next query, include X-Prefetch-Available header
- Graceful degradation: LRU cache of popular queries when AI unavailable

**VersionlessApiStrategy** (StrategyId: "versionless-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: No explicit API versioning required, detect client expectations from request shape, apply backward-compatible transformations automatically

**ZeroConfigApiStrategy** (StrategyId: "zero-config-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Auto-discover available operations from registered strategies, return self-describing root document, support ?discover=true for guided exploration

**B9: Security Strategies (Strategies/Security/)**

**ZeroTrustApiStrategy** (StrategyId: "zero-trust-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Validate JWT/mTLS/API key on every request, check device posture, enforce least-privilege, route auth to Access Control via message bus

**QuantumSafeApiStrategy** (StrategyId: "quantum-safe-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Post-quantum cryptographic primitives, support ML-KEM key encapsulation, ML-DSA signatures via X-PQ-Algorithm header, route key ops to Encryption/KeyManagement via message bus

**EdgeCachedApiStrategy** (StrategyId: "edge-cached-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Set Cache-Control, ETag, Last-Modified headers, support conditional requests (If-None-Match, If-Modified-Since), return 304 Not Modified when appropriate

**SmartRateLimitStrategy** (StrategyId: "smart-rate-limit"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Track per-client request rates with sliding window, use Intelligence for abuse detection, return 429 + Retry-After when exceeded, support tiered limits
- Graceful degradation: fixed-window rate limiting when AI unavailable

**CostAwareApiStrategy** (StrategyId: "cost-aware-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Compute operation cost, return X-Api-Cost/X-Api-Budget-Remaining headers, support cost quotas, reject over-budget requests (402), support ?estimate=true

**AnomalyDetectionApiStrategy** (StrategyId: "anomaly-detection-api"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Track request patterns, use Intelligence for ML-based anomaly scoring, flag via X-Anomaly-Score header, route high-anomaly to Access Control
- Graceful degradation: statistical threshold detection when AI unavailable

All AI-dependent strategies MUST delegate to Intelligence plugin via message bus and provide rule-based fallbacks. All are internal sealed classes with XML docs.

**B10: Developer Experience Strategies (Strategies/DeveloperExperience/)**

**InstantSdkGenerationStrategy** (StrategyId: "instant-sdk-generation"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /sdk/{language} generates client SDK code for specified language (csharp, python, typescript, java, go, rust), introspect registered strategies to build operation catalog

**InteractivePlaygroundStrategy** (StrategyId: "interactive-playground"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /playground returns HTML page with interactive API explorer, POST /playground/execute executes request from playground

**MockServerStrategy** (StrategyId: "mock-server"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: POST /mocks registers mock response, GET/POST/PUT/DELETE /mock/* returns configured mock, support response delay simulation and recording mode

**ApiVersioningStrategy** (StrategyId: "api-versioning"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Route requests to version handler based on URL path (/v1/...), header (X-Api-Version), query param (?api-version=), or Accept header versioning

**ChangelogGenerationStrategy** (StrategyId: "changelog-generation"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /changelog returns auto-generated changelog, GET /changelog/{version} returns changes for specific version, format as structured JSON

**BreakingChangeDetectionStrategy** (StrategyId: "breaking-change-detection"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: POST /compatibility-check accepts OpenAPI spec and compares against current, detect removed endpoints, changed types, removed fields

**B11: Air-Gap Convergence UI Strategies (Strategies/Convergence/)**

These strategies provide the interface layer for T123/T124 air-gap convergence workflows.

**InstanceArrivalNotificationStrategy** (StrategyId: "instance-arrival-notification"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: POST /convergence/notify receives air-gapped instance arrival, GET /convergence/pending lists pending arrivals

**ConvergenceChoiceDialogStrategy** (StrategyId: "convergence-choice-dialog"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /convergence/{instanceId}/choice returns dialog data (keep separate vs merge), POST records decision

**MergeStrategySelectionStrategy** (StrategyId: "merge-strategy-selection"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /convergence/{instanceId}/strategies lists merge strategies (last-write-wins, manual, field-level), POST selects strategy

**MasterInstanceSelectionStrategy** (StrategyId: "master-instance-selection"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /convergence/{instanceId}/instances lists eligible masters, POST selects master instance

**SchemaConflictResolutionUIStrategy** (StrategyId: "schema-conflict-resolution"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /convergence/{instanceId}/conflicts lists schema conflicts with per-field diff, POST resolves individual or batch conflicts

**MergePreviewStrategy** (StrategyId: "merge-preview"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /convergence/{instanceId}/preview returns merge preview (records to add/update/delete, conflicts remaining), dry-run mode

**MergeProgressTrackingStrategy** (StrategyId: "merge-progress"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: POST /convergence/{instanceId}/execute starts merge, GET /convergence/{instanceId}/progress returns progress percentage

**MergeResultsSummaryStrategy** (StrategyId: "merge-results-summary"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /convergence/{instanceId}/results returns post-merge summary (records merged, conflicts resolved, time taken)

All strategies include XML docs, are internal sealed classes, use message bus for DataWarehouse operations.
