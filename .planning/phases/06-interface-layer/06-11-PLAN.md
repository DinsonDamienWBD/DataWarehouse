Create 6 strategy files in the DeveloperExperience/ subdirectory. Each extends InterfaceStrategyBase and implements IPluginInterfaceStrategy.

**InstantSdkGenerationStrategy** (StrategyId: "instant-sdk-generation"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /sdk/{language} generates client SDK code for specified language (csharp, python, typescript, java, go, rust), introspect registered strategies to build operation catalog

**InteractivePlaygroundStrategy** (StrategyId: "interactive-playground"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /playground returns HTML page with interactive API explorer, POST /playground/execute executes request from playground

**MockServerStrategy** (StrategyId: "mock-server"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: POST /mocks registers mock response, GET/POST/PUT/DELETE /mock/* returns configured mock, support response delay simulation and recording mode

**ApiVersioningStrategy** (StrategyId: "api-versioning"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: Route requests to version handler based on URL path (/v1/...), header (X-Api-Version), query param (?api-version=), or Accept header versioning

**ChangelogGenerationStrategy** (StrategyId: "changelog-generation"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: GET /changelog returns auto-generated changelog, GET /changelog/{version} returns changes for specific version, format as structured JSON

**BreakingChangeDetectionStrategy** (StrategyId: "breaking-change-detection"):
- Protocol: InterfaceProtocol.REST
- HandleRequestAsync: POST /compatibility-check accepts OpenAPI spec and compares against current, detect removed endpoints, changed types, removed fields

All are internal sealed classes with XML docs, using message bus for DataWarehouse operations.
