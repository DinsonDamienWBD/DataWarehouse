<objective>
Build the VdeFederationRouter: the core namespace resolution engine that maps any key/path to a specific (shardVdeId, inodeNumber) address. Implements path-embedded routing (hash each path segment to a catalog slot, then resolve the slot to a downstream VDE ID), an LRU warm cache for O(1) repeated lookups, and cold-path resolution that traverses the catalog hierarchy in at most 5 hops.

Purpose: This is the routing brain of VDE 2.0B federation. Without it, no operation can find which shard holds a given key.
Output: 5 files in DataWarehouse.SDK/VirtualDiskEngine/Federation/ providing the complete routing infrastructure.
</objective>

**PathHashRouter.cs** -- deterministic path-to-slot mapping:
- `[SdkCompatibility("6.0.0", Notes = "Phase 92: VDE 2.0B Federation Router (VFED-04)")]`
- `public sealed class PathHashRouter`
- Constructor: `PathHashRouter(int slotCount = FederationConstants.DefaultDomainSlotCount)`
- `public int ComputeSlot(ReadOnlySpan<char> pathSegment)` -- hash path segment using `System.IO.Hashing.XxHash64` with `FederationConstants.HashSeed`, then modulo slotCount. Convert chars to UTF8 bytes on stack (stackalloc for segments <= 256 chars, rent from ArrayPool otherwise).
- `public int ComputeSlot(ReadOnlySpan<byte> utf8PathSegment)` -- direct UTF8 overload (avoids allocation for callers that already have bytes)
- `public int[] ComputeSlots(string fullPath)` -- splits path by '/' separator, returns array of slots for each segment. First segment determines domain slot, subsequent segments for deeper routing.
- Path normalization: trim leading/trailing '/', collapse consecutive '/', case-preserved (no lowering -- binary exact)

**VdeFederationRouter.cs** -- the main router:
- `[SdkCompatibility("6.0.0", Notes = "Phase 92: VDE 2.0B Federation Router (VFED-05)")]`
- `public sealed class VdeFederationRouter : IAsyncDisposable`
- Private fields:
  - `_pathHashRouter` (PathHashRouter)
  - `_routingTable` (RoutingTable)
  - `_warmCache` -- `BoundedDictionary<string, (ShardAddress Address, long ExpiryTicks)>` with capacity `FederationConstants.MaxWarmCacheEntries`. Use existing `BoundedDictionary` from SDK.
  - `_catalogResolver` -- `IShardCatalogResolver?` (interface defined here, implemented by ShardCatalog in plan 92-02/03). Null means single-VDE mode.
  - `_singleVdeId` -- `Guid?` for passthrough mode
- Constructor: `VdeFederationRouter(RoutingTable routingTable, PathHashRouter pathHashRouter, IShardCatalogResolver? catalogResolver = null)`
- Static factory: `public static VdeFederationRouter CreateSingleVde(Guid vdeId)` -- creates a passthrough router that returns the single VDE for all keys (zero overhead: no hashing, no table lookup, just returns the ID)
- **Core resolve method**: `public async ValueTask<ShardAddress> ResolveAsync(string key, CancellationToken ct = default)`
  1. If `_singleVdeId` is set, return `new ShardAddress(_singleVdeId.Value, 0, ShardLevel.Data)` immediately (zero-overhead passthrough)
  2. Check warm cache: if key found and not expired, return cached address (1 hop)
  3. Cold path: call `_pathHashRouter.ComputeSlots(key)`, use first slot to look up domain VDE in `_routingTable`, then call `_catalogResolver!.ResolveAsync(key, slots, ct)` to walk the hierarchy (up to MaxColdPathHops)
  4. Cache the result in warm cache with TTL
  5. Return ShardAddress

- `public void ClearCache()` -- full cache flush
- `public FederationRouterStats GetStats()` -- return record with CacheHits, CacheMisses, ColdPathResolutions, AverageHops
