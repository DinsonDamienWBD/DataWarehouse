    - Decode: lose shard 0 and shard 4 -> still reconstruct
    - Decode: lose both parity shards -> data shards alone suffice
    - Verify: correct shards return true, corrupted shard returns false
    - Encode then Decode roundtrip: output == original input byte-for-byte
  </behavior>
  <action>
Create ReedSolomonErasureCoder.cs:
- Implements IErasureCoder with ErasureCodingScheme.ReedSolomon
- Constructor(int defaultK = 4, int defaultM = 2) -- default (4,2) coding
- Encode: Build Vandermonde encoding matrix (k+m rows x k cols). Multiply matrix by data shards to produce k+m output shards. First k output shards = original data (systematic coding). Last m shards = parity.
- Decode: Given available shards bitmap, select any k shards. Build sub-matrix from encoding matrix rows. Invert sub-matrix in GF(2^8). Multiply inverse by available shards to reconstruct all k data shards.
- Verify: Re-encode from data shards, compare parity shards.
- SIMD path: Use System.Runtime.Intrinsics.X86.Avx2 (256-bit) for bulk XOR and GF multiply-accumulate when available. Fallback to scalar path. Check Avx2.IsSupported at construction time, cache result.
- Thread-safe: encoding matrix computed once at construction, immutable. No shared mutable state.
- [SdkCompatibility("8.0.0", Notes = "Phase 128: Reed-Solomon erasure coder")]
