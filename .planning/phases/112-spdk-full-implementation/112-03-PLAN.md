<objective>
Enhance SpdkBlockDevice with queue-per-thread lock-free I/O, zero-copy DMA path, and proper NVMe flush command support.

Purpose: The existing SpdkBlockDevice uses a single queue pair with SemaphoreSlim, creating contention under concurrent workloads. Queue-per-thread eliminates locking. Zero-copy avoids the DMA-to-caller buffer copy when the caller already provides aligned memory. These are critical for achieving the 2M+ IOPS target.

Output: Lock-free SpdkBlockDevice with per-thread queue pairs, zero-copy detection, and NVMe flush.
</objective>
