   - `spdk_nvme_zns_zone_mgmt_send` (zone open/close/reset/finish) — takes namespace, qpair, zone SLBA, action enum, completion callback
   - `spdk_nvme_zns_report_zones` — takes namespace, qpair, DMA buffer, buffer size, start SLBA, report type, completion callback
   - `spdk_nvme_zns_ns_get_zone_size` — returns zone size in LBAs
   - `spdk_nvme_zns_ns_get_num_zones` — returns total zone count
   - `spdk_nvme_ns_get_csi` — returns command set identifier (NVM=0, ZNS=2)

2. Supporting structs in the same file:
   - `SpdkNvmeZnsZoneAction` enum: Open=0, Close=1, Finish=2, Reset=3
   - `SpdkNvmeZnsReportType` enum: All=0, Empty=1, ImplicitlyOpened=2, ExplicitlyOpened=3, Closed=4, Full=5, ReadOnly=6, Offline=7
   - `SpdkNvmeZnsZoneDescriptor` struct (Sequential layout): SLBA, write pointer, zone state, zone type, capacity
   - `SpdkNvmeZnsReportHeader` struct: nr_zones, followed by array of ZoneDescriptor

3. In the existing `SpdkNativeMethods.cs`, add to the I/O Commands region:
   - `spdk_nvme_ns_cmd_flush` — submit NVMe flush command with completion callback

   - `spdk_nvme_ctrlr_cmd_get_log_page` — for SMART/Health log, Error log
   - `spdk_nvme_ctrlr_get_num_ns` — get number of namespaces
   - `spdk_nvme_ns_get_max_io_xfer_size` — max transfer size for a namespace
   - `spdk_nvme_ns_get_flags` — namespace feature flags (thin provisioning, etc.)

All bindings use `[LibraryImport(LibName, ...)]` with `internal static partial` methods matching existing style. Use `unsafe partial` where function pointers are needed. Use same `LibName = "spdk"` constant. Add full XML documentation matching existing style.
