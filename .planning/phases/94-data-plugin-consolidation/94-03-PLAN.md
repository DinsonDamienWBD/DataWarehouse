<objective>
Clarify the UniversalFabric vs UltimateDataFabric naming confusion with documentation and code comments.

Purpose: The codebase has two "fabric" concepts that cause confusion: (1) `UniversalFabric` plugin (`com.datawarehouse.storage.universal-fabric`) which is a physical storage I/O routing layer implementing `IStorageFabric` -- it routes Store/Retrieve/Delete operations to backend storage strategies via placement rules, S3 server, and live migration. (2) `UltimateDataFabric` which was a logical data virtualization layer (unified views across heterogeneous data sources) that has been merged into `UltimateDataManagement` as 13 fabric strategies. UniversalFabric is NOT in PLUGIN-CATALOG at all, creating further confusion. The roadmap suggests renaming to UltimateStorageFabric, but renaming a plugin directory/namespace is high-risk (breaks csproj references, solution files, namespaces, and all consumers). Instead, we document the distinction clearly.

Output: Updated PLUGIN-CATALOG with UniversalFabric entry, updated code/AI-map documentation.
</objective>
