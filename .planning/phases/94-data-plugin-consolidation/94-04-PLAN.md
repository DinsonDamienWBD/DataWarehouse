<objective>
Add graceful degradation to all message bus delegation paths introduced by plans 94-01 and 94-02. When the authoritative plugin (UltimateDataLineage for lineage, UltimateDataCatalog for catalog) is unavailable or slow, delegating plugins must fall back to a local BoundedCache of recently-seen results rather than failing or blocking.

Purpose: Production resilience -- plugins must not hard-fail when a peer plugin is down. The microkernel architecture means plugins start/stop independently, so delegation must tolerate partial availability.

Output: A `MessageBusDelegationHelper` class in each delegating plugin that wraps SendMessageAsync with ICircuitBreaker (from SDK) + local BoundedCache fallback. Each plugin's message handlers updated to use the helper instead of raw SendMessageAsync.
</objective>

  <action>
Create a `MessageBusDelegationHelper` class in each delegating plugin's `Delegation/` folder. Each instance wraps message bus calls to one authoritative plugin topic. The class must:

1. **Constructor** accepts: `PluginBase ownerPlugin` (to access SendMessageAsync), `string targetTopic` (e.g. "lineage.track"), `CircuitBreakerOptions? options` (defaults: FailureThreshold=5, FailureWindow=60s, BreakDuration=30s).
2. **Internal state**: `InMemoryCircuitBreaker` from SDK (NOT a custom implementation), `BoundedCache<string, object>` for local fallback (max 500 entries, 5-minute TTL).
3. **`DelegateAsync(Dictionary<string, object> payload, CancellationToken ct)`** method:
   - Wraps the message bus call in `_circuitBreaker.ExecuteAsync(...)` with a 2-second timeout via `CancellationTokenSource.CreateLinkedTokenSource`.
   - On success: cache the response keyed by a deterministic hash of the payload (use topic + action + key IDs concatenated).
   - On failure (circuit open OR timeout OR exception): look up cached result. If cache hit, return cached result with `payload["_delegationFallback"] = true`. If cache miss, set `payload["_delegationUnavailable"] = true` and return without throwing.
4. **`GetStatistics()`** exposes the circuit breaker stats for observability.
5. Use `IDisposable` pattern to dispose the cache.

Each plugin gets its own copy (plugin isolation rule -- plugins do NOT share code). The implementations will be near-identical but exist in separate assemblies. Use the SDK's existing `InMemoryCircuitBreaker` and `BoundedCache` types -- do NOT hand-roll circuit breaker logic.

For UltimateDataCatalog: helper targets `lineage.*` topics (delegating its `_relationships` to UltimateDataLineage).
For UltimateDataLake: TWO helpers -- one targeting `lineage.*` topics, one targeting `catalog.*` topics.
For UltimateDataManagement: helper targets `catalog.*` topics (for fabric catalog delegation if applicable from 94-02).
