JwtRefreshService.cs:
1. Issue refresh tokens (opaque, stored server-side in ConcurrentDictionary or IDistributedCache)
2. Refresh endpoint: POST /api/v1/auth/refresh — accepts expired access token + valid refresh token, returns new pair
3. Refresh token rotation: each use invalidates old refresh token and issues new one
4. Refresh token expiry: 7 days (configurable)
5. Access token expiry: 15 minutes (configurable)
6. Revocation: POST /api/v1/auth/revoke — invalidates all refresh tokens for a user
7. Family tracking: detect refresh token reuse (indicates theft) — revoke entire family

AuthenticationConfig.cs updates:
1. Configure JWT bearer with short-lived access tokens
2. Add refresh token validation
3. Session idle timeout: 30 minutes (configurable)
4. Concurrent session limit: configurable (default 3)

AuthController.cs updates:
1. Login returns access token + refresh token
2. Add /refresh endpoint
3. Add /revoke endpoint
4. Add /sessions endpoint (list active sessions)
5. Add /sessions/{id}/logout endpoint (force-logout specific session)

MfaIntegration.cs:
1. Integrate with UltimateAccessControl MFA providers
2. Support TOTP (authenticator app), SMS, email as second factors
3. MFA challenge on login: after password verification, require MFA if enabled for user
4. MFA enrollment endpoint: POST /api/v1/auth/mfa/enroll
5. MFA verify endpoint: POST /api/v1/auth/mfa/verify
6. Remember device: skip MFA for trusted devices (30 days)

Program.cs:
