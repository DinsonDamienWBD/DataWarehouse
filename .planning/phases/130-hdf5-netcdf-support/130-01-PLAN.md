  <action>
1. Add `HDF5-CSharp` NuGet package to `DataWarehouse.Plugins.UltimateDataFormat.csproj`. Use latest stable version. This package wraps HDF.PInvoke with a C#-friendly API.

2. Replace stub `ParseAsync` in Hdf5Strategy.cs:
   - Save stream to temp file (HDF5 requires file-based access via native library)
   - Open with `Hdf5File.OpenRead(tempPath)`
   - Enumerate root group children (groups + datasets)
   - For each dataset: read name, datatype, dimensions, chunk layout, compression filter
   - Read dataset data into typed arrays based on datatype (int, float, double, string)
   - Read attributes from groups and datasets
   - Return DataFormatResult.Success with structured output containing groups/datasets/attributes
   - Clean up temp file in finally block

3. Replace stub `SerializeAsync`:
   - Create temp file, open with `Hdf5File.Create(tempPath)`
   - Accept input as Dictionary<string, object> representing group/dataset hierarchy
   - Create groups for nested dictionaries
   - Write datasets with chunking enabled (chunk size from DataFormatContext or default 64KB)
   - Set compression filter (deflate level 6 default)
   - Attach attributes from metadata
   - Copy temp file to output stream
   - Clean up temp file

4. Replace stub `ExtractSchemaCoreAsync`:
   - Open file, walk group hierarchy
   - Build FormatSchema with SchemaField per dataset (name=path, dataType=HDF5 type, dimensions as description)

5. Implement VDE extent mapping helper: `MapDatasetToExtents(string datasetPath, long offset, long length)` that converts HDF5 dataset byte ranges to VDE-compatible extent descriptors for the storage engine.

6. Keep existing DetectFormatCoreAsync (signature check) and ValidateCoreAsync (size check) — they are already production-quality. Enhance ValidateCoreAsync to also verify superblock version field at offset 8.

7. Increment counters: `hdf5.parse` on successful parse, `hdf5.serialize` on successful serialize.
  </action>
