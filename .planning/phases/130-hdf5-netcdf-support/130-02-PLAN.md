  <action>
1. **NetCDF-3 Classic Parser** (pure C#, no external dependency):
   - Read header: magic "CDF\x01", numrecs, dim_list, att_list, var_list
   - Parse dimensions: name, length, isUnlimited
   - Parse variables: name, dimids[], type (byte/char/short/int/float/double), vsize, begin offset, attributes
   - Parse global attributes: name, type, values
   - Read variable data by computing offset from `begin` field + record stride for unlimited dims
   - Support hyperslab selection: caller specifies start[]/count[] per dimension

2. **NetCDF-4 Parser** (via HDF5-CSharp, already added by 130-01):
   - Detect HDF5 signature -> open as HDF5 file
   - Read NetCDF-4 conventions: `_NCProperties` attribute, dimension scales, coordinate variables
   - Map HDF5 groups to NetCDF-4 group hierarchy
   - Extract CF convention attributes: `units`, `long_name`, `standard_name`, `_FillValue`, `scale_factor`, `add_offset`
   - Apply `scale_factor` and `add_offset` unpacking for packed data

3. **SerializeAsync** (NetCDF-4 output via HDF5):
   - Create HDF5 file with NetCDF-4 conventions
   - Write `_NCProperties` attribute on root group
   - Create dimensions as HDF5 dimension scales
   - Write variables as HDF5 datasets with CF attributes
   - Support unlimited dimension via HDF5 chunked storage with extensible maxdims

4. **ExtractSchemaCoreAsync**:
   - Return FormatSchema with fields: one per variable, dataType = NetCDF type, description includes dimensions and units

5. Keep existing DetectFormatCoreAsync — it already handles CDF and HDF5 signatures correctly.

6. Increment counters: `netcdf.parse` / `netcdf.serialize`.
