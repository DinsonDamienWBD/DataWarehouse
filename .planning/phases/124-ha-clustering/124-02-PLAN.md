  <action>
    **ClusterJoinProtocol.cs:**
    - Constructor: IClusterMembership membership, ILogger logger, ClusterJoinOptions options
    - `JoinAsync(string seedAddress, int seedPort, string certPath, string keyPath, CancellationToken)`:
      1. Load node certificate from certPath/keyPath
      2. Connect to seed node at seedAddress:seedPort with mTLS (SslStream with client certificate)
      3. Request cluster topology from seed (simple JSON protocol over TLS stream):
         - Send: `{"type":"join_request","node_id":"...","address":"...","port":...}`
         - Receive: `{"type":"join_response","cluster_id":"...","members":[...],"ca_cert":"..."}`
      4. Validate received CA cert matches local CA cert (prevent impersonation)
      5. Start SWIM gossip with all discovered members (use SwimClusterMembership)
      6. Create ClusterJoinRequest with Role=Follower
      7. Call IClusterMembership.JoinAsync to register with Raft
      8. Wait for health check: poll IsHealthyAsync(self.NodeId) with timeout (30s default)
      9. If healthy: log success, node is now a Raft voter (promotion happens via Raft configuration change)
      10. If unhealthy after timeout: LeaveAsync and throw

    **ClusterJoinOptions:**
    ```
    ClusterJoinOptions {
        MaxRetries = 3
        RetryDelayMs = 2000
        HealthCheckTimeoutMs = 30000
        HealthCheckIntervalMs = 1000
    }
    ```

    Join protocol must be idempotent — if node is already a member, verify membership and return without disruption.

    IMPORTANT: The join protocol does NOT restart or reconfigure existing members. It only adds the new node to the Raft configuration via a configuration change entry appended to the Raft log.
