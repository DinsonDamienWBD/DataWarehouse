    **ClusterConfig.cs** — Cluster-wide configuration model:
    ```
    ClusterConfig {
        ClusterId: string (auto-generated GUID)
        ClusterName: string (user-provided or "dw-cluster-{short-guid}")
        ReplicationFactor: int
        Nodes: List<NodeConfig>
        RaftConfig: RaftConfiguration { ElectionTimeoutMs=5000, HeartbeatIntervalMs=1000, SnapshotIntervalEntries=10000 }
        TlsConfig: TlsConfiguration { CaCertPath, RequireMutualTls=true }
        CreatedAt: DateTimeOffset
    }

    NodeConfig {
        NodeId: string (dw-node-{short-guid})
        Address: string (configurable, default localhost for single-machine)
        Port: int (base 9400 + index)
        RaftPort: int (base 9500 + index)
        GossipPort: int (base 9600 + index)
        IsVoter: bool (true for initial nodes)
        CertificatePath: string
        KeyPath: string
    }
    ```

    JSON serializable with System.Text.Json. Include `Save(string path)` and `static Load(string path)` methods.

    **ClusterBootstrapper.cs** — Turnkey initialization:
    - Constructor: ILogger, optional base directory for config output
    - `InitializeAsync(int nodeCount, int replicationFactor, string? clusterName, CancellationToken)`:
      1. Validate: nodeCount >= 1, replicationFactor >= 1 and <= nodeCount
      2. Generate ClusterId (GUID)
      3. Generate NodeConfig for each node (sequential ports starting at 9400)
      4. Generate self-signed CA certificate (X509Certificate2, RSA 4096-bit, 10-year validity)
      5. For each node: generate node certificate signed by CA (RSA 2048-bit, 1-year validity, CN=dw-node-{id}, SAN=address)
      6. Write certificates to {baseDir}/certs/{nodeId}.pfx and CA to {baseDir}/certs/ca.crt
      7. Configure Raft: all initial nodes as voters
      8. Create ClusterConfig, save to {baseDir}/cluster.json
      9. Return ClusterConfig

    Use System.Security.Cryptography for certificate generation (X509Certificate2, RSA, CertificateRequest, X509BasicConstraintsExtension).

    All certificate operations must be production-quality: proper key usage extensions, subject alternative names, not placeholder/fake certs.
