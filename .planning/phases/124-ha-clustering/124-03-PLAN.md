    **SplitBrainProtection.cs:**
    - Constructor: IClusterMembership membership, ILogger logger
    - `int CalculateQuorum(int totalVoters)`: Returns floor(totalVoters / 2) + 1 (strict majority)
    - `bool HasQuorum()`: Count reachable voters via IsHealthyAsync, compare against quorum threshold
    - `bool IsWriteAllowed()`: Returns true only if HasQuorum() is true. Writes on minority partition MUST be rejected.
    - `EvaluateAsync(CancellationToken)`: Periodic quorum evaluation (called by failover monitor). Logs partition warnings.
    - Subscribe to OnMembershipChanged: on NodeDead/NodeSuspected, re-evaluate quorum.
    - On partition heal (NodeJoined with previously-Dead node): log "partition healed, node {id} rejoining from majority".

    Quorum is strictly majority-based: for N voters, quorum = floor(N/2) + 1. This prevents two partitions from both claiming write authority.

    **Heartbeat monitoring:**
    - Track last heartbeat from leader (via OnMembershipChanged events or periodic IsHealthyAsync)
    - If no heartbeat for HeartbeatTimeoutMs (default 5000ms from RaftConfig): declare leader suspected
