    Create ClusterCommands.cs following existing CLI command patterns (check existing commands like ServerCommands.cs or StorageCommands.cs for conventions).

    **dw cluster init:**
    - Options: --nodes (int, default 3), --replication-factor (int, default 2), --name (string, optional), --output-dir (string, default ./cluster)
    - Calls ClusterBootstrapper.InitializeAsync
    - Output: table of generated node configs (ID, address, port, cert path), cluster.json path

    **dw cluster join:**
    - Options: --seed (string, required, format ip:port), --cert (string, required), --key (string, required)
    - Calls ClusterJoinProtocol.JoinAsync
    - Output: "Successfully joined cluster {clusterId} as node {nodeId}"

    **dw cluster status:**
    - No required options (reads cluster.json from current dir or --config path)
    - Calls IClusterMembership.GetMembers() and GetLeader()
    - Output: table with columns [NodeId, Address, Port, Role, Status, IsLeader, LastSeen]
    - Highlight leader row, color-code status (Active=green, Suspected=yellow, Dead=red)

    **dw cluster add-node:**
    - Options: --address (string, required), --port (int, default 9400)
    - Generates new node certificate signed by cluster CA
    - Outputs join command for the new node: `dw cluster join --seed {leader}:{port} --cert {path} --key {path}`

    **dw cluster remove-node:**
    - Options: --node-id (string, required), --force (bool, default false)
    - Calls IClusterMembership.LeaveAsync for the specified node
    - With --force: removes even if node is unreachable
    - Output: "Node {nodeId} removed from cluster"

    **dw cluster rebalance:**
    - Triggers data redistribution across nodes (placeholder for actual data rebalance — log intent and list shard assignments)
    - Output: progress bar showing rebalance status
