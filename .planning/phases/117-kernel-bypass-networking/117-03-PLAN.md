public sealed class DpdkNetworkTransport : KernelBypassTransportBase
{

    public override string Name => "DPDK Network Transport";
    public override string Version => "8.0.0";

    public override KernelBypassCapability Capabilities =>
        KernelBypassCapability.ZeroCopy | KernelBypassCapability.PollMode | KernelBypassCapability.Batching;
    public override TransportPreference TransportType => TransportPreference.Dpdk;
}
```

Key implementation:
- Initialize EAL with `--proc-type=secondary` support for shared NIC
- PMD (Poll Mode Driver) thread: dedicated `Thread(IsBackground=true)` pinned to DPDK lcore, runs tight `rte_eth_rx_burst` loop
- TX batching: accumulate packets in `rte_mbuf[]` array, flush via `rte_eth_tx_burst` when batch full (32 packets) or timer expires (100us)
- Mbuf pool: pre-allocate `rte_pktmbuf_pool_create` with 8192 mbufs
- Memory registration: DPDK uses hugepage-backed memory; PlatformRegisterMemory allocates from mbuf pool
- ConnectAsync: configure port with 1 RX + 1 TX queue, start port
- SendAsync: allocate mbuf, copy data, queue for TX burst
- ReceiveAsync: dequeue from RX burst ring buffer
- IRdmaTransport bridge: map ReadRemoteAsync/WriteRemoteAsync to send/receive (DPDK is not true RDMA but provides the interface)
- Inject `IDpdkNative` interface (internal) for testability

Tests mock IDpdkNative. Verify PMD thread lifecycle, batch accumulation, graceful unavailable behavior, statistics.

    public override string Name => "TCP Fallback Transport";
    public override string Version => "8.0.0";

    public override KernelBypassCapability Capabilities => KernelBypassCapability.None;
    public override TransportPreference TransportType => TransportPreference.Tcp;
    public override bool IsAvailable => true; // TCP always available
}
```

Key implementation:
- Use `Socket` with `SocketAsyncEventArgs` pool (ObjectPool pattern) for minimal allocation
- Pool size: 64 pre-allocated SocketAsyncEventArgs
- ConnectAsync: `Socket.ConnectAsync` with TCP_NODELAY, buffer sizes 256KB
- SendAsync/ReceiveAsync: async socket operations via SocketAsyncEventArgs
- PlatformRegisterMemory: no-op (returns pinned GCHandle address -- TCP has no DMA)
- PlatformDeregisterMemory: unpin GCHandle
- IRdmaTransport bridge: ConnectAsync returns RdmaConnection with LocalKey=0/RemoteKey=0 (TCP has no RDMA keys), ReadRemoteAsync/WriteRemoteAsync map to socket send/receive
- SendRegisteredAsync/ReceiveRegisteredAsync: delegate to regular SendAsync/ReceiveAsync (no zero-copy benefit on TCP)

Tests use loopback socket pair (TcpListener + TcpClient on localhost). Verify full send/receive round-trip, event args pool reuse, statistics accumulation, dispose cleanup.
