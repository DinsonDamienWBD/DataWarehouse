<task type="auto">
  <name>Task 1: Run dotCover full-solution coverage analysis</name>
  <files>Metadata/coverage-baseline/coverage-report.xml, Metadata/coverage-baseline/per-project-summary.md</files>
  <action>
    1. Create directory `Metadata/coverage-baseline/`.
    2. Run dotCover CLI to cover all tests in the solution:
       ```
       dotnet dotcover cover-dotnet --output=Metadata/coverage-baseline/coverage-report.xml --report-type=DetailedXML -- test DataWarehouse.sln --no-build
       ```
       If `dotnet dotcover` is not available as a global tool, install it: `dotnet tool install --global JetBrains.dotCover.CommandLineTools`.
       If the full solution test run is too heavy, run per-test-project:
       - `dotnet test DataWarehouse.Tests/DataWarehouse.Tests.csproj`
       - `dotnet test DataWarehouse.Hardening.Tests/DataWarehouse.Hardening.Tests.csproj`
    3. Parse the XML output to extract per-project coverage percentages. For each project (SDK, Kernel, each of the 52 plugins, Shared, CLI, GUI, Dashboard, Launcher), extract: TotalStatements, CoveredStatements, CoveragePercent.
    4. Generate `Metadata/coverage-baseline/per-project-summary.md` as a markdown table:
       ```
       | Project | Statements | Covered | Coverage % | Status |
       |---------|-----------|---------|------------|--------|
       | DataWarehouse.SDK | ... | ... | ...% | OK / BELOW 80% |
       ```
    5. At the bottom of the summary, list all files with <80% coverage in a "Files Needing Coverage" section grouped by project.
    6. If dotCover fails due to build errors, first run `dotnet build DataWarehouse.sln` and fix any build issues before retrying.
  </action>
  <verify>
    <automated>test -f Metadata/coverage-baseline/coverage-report.xml && test -f Metadata/coverage-baseline/per-project-summary.md && grep -c "Coverage %" Metadata/coverage-baseline/per-project-summary.md</automated>
  </verify>
  <done>Coverage XML exists with per-project breakdown. Summary markdown lists all projects with coverage percentages. Files below 80% are identified.</done>
</task>
