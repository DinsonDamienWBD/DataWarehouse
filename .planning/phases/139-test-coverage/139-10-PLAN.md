  <name>Task 1: Create CI coverage gate workflow</name>
  <files>.github/workflows/coverage-gate.yml</files>
  <action>
    1. Check existing CI workflows in `.github/workflows/` — integrate with or extend `audit.yml` if it exists.
    2. Create `.github/workflows/coverage-gate.yml`:
       ```yaml
       name: Coverage Gate
       on:
         pull_request:
           branches: [master]
         push:
           branches: [master]
       jobs:
         coverage:
           runs-on: windows-latest
           steps:
             - uses: actions/checkout@v4
             - uses: actions/setup-dotnet@v4
               with:
                 dotnet-version: '10.0.x'
             - run: dotnet build DataWarehouse.sln
             - name: Run dotCover
               run: |
                 dotnet tool install --global JetBrains.dotCover.CommandLineTools
                 dotnet dotcover cover-dotnet --output=coverage.xml --report-type=DetailedXML -- test DataWarehouse.sln --no-build
             - name: Check coverage thresholds
               run: |
                 # Parse coverage.xml, assert >=95% overall, >=90% per-project
                 dotnet test DataWarehouse.Hardening.Tests --filter "FullyQualifiedName~CoverageGate" --no-build
             - name: Stryker mutation testing
               run: |
                 dotnet tool install --global dotnet-stryker
                 dotnet stryker --threshold-high 95 --threshold-low 90 --threshold-break 90
       ```
    3. Ensure the workflow fails (non-zero exit) if thresholds are not met.
    4. Add step for diff coverage on PRs: compare coverage of changed files against 100% threshold.
  </action>
  <verify>
    <automated>test -f .github/workflows/coverage-gate.yml && grep -c "threshold" .github/workflows/coverage-gate.yml</automated>
  </verify>
  <done>CI workflow exists with dotCover >=95% overall gate, >=90% per-project gate, and Stryker >=90% gate.</done>
