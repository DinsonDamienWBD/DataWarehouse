**ShardCapacityMonitor.cs** -- sealed class `ShardCapacityMonitor`:
- Constructor takes:
  - `Func<CancellationToken, ValueTask<IReadOnlyList<DataShardDescriptor>>> shardEnumerator` -- returns all current data shards
  - `double splitThresholdPercent = LifecycleConstants.DefaultSplitThresholdPercent` -- defaults to 80%
  - Validate threshold is in (0, 100) range.
- `ValueTask<IReadOnlyList<DataShardDescriptor>> FindSplitCandidatesAsync(CancellationToken ct = default)`:
  1. Call shardEnumerator to get all shards.
  2. Filter to shards where `UtilizationPercent >= splitThresholdPercent` AND `IsAvailable` (Healthy or Degraded).
  3. Sort descending by UtilizationPercent (fullest first -- highest priority).
  4. Return the filtered list.
- `ValueTask<IReadOnlyList<DataShardDescriptor>> FindMergeCandidatesAsync(double mergeThresholdPercent, CancellationToken ct = default)`:
  1. Call shardEnumerator.
  2. Filter to shards where `UtilizationPercent <= mergeThresholdPercent` AND `IsAvailable`.
  3. Sort ascending by StartSlot (for adjacency detection in merge engine).
  4. Return.
  Note: this method is here because the monitor scans for BOTH split and merge candidates. The merge engine (plan 93-03) consumes this output.

Thread safety: stateless per call, no locking needed.

Create `ShardSplitEngine` sealed class. `[SdkCompatibility("6.0.0", Notes = "Phase 93: VDE 2.0B Shard Lifecycle (VSHL-02)")]`.

Constructor takes:
- `PlacementPolicyEngine placementEngine`
- `Func<Guid, DataShardDescriptor, CancellationToken, ValueTask<Guid>> shardCreator` -- creates a new empty VDE data shard, returns its new VDE ID. Takes parent shard ID and descriptor template.
- `Func<Guid, Guid, int, int, CancellationToken, ValueTask<long>> dataRedistributor` -- copies blocks from source shard to target shard for slot range [startSlot, endSlot). Returns blocks copied. This is the CoW background copy.
- All parameters non-null (ArgumentNullException.ThrowIfNull).

**Core method:** `async ValueTask<ShardSplitResult> SplitAsync(DataShardDescriptor shard, IndexShardCatalog indexCatalog, RoutingTable routingTable, StorageTier newShardTier, CancellationToken ct = default)`:

Algorithm:
1. Validate shard has at least 2 slots (EndSlot - StartSlot >= 2). If not, return `ShardSplitResult.Skipped(shard, "Slot range too narrow to bisect")`.
2. Start a Stopwatch.
3. **Bisect**: `int midSlot = shard.StartSlot + (shard.EndSlot - shard.StartSlot) / 2`.
4. **Create lower half descriptor** (reuses original VDE ID):
   ```
   var lowerHalf = new DataShardDescriptor {
       DataVdeId = shard.DataVdeId,
       StartSlot = shard.StartSlot,
       EndSlot = midSlot,
       CapacityBlocks = shard.CapacityBlocks,  // capacity unchanged, blocks stay in-place
       UsedBlocks = 0,  // will be updated after redistribution
       Health = DataShardHealth.Healthy,
       LastHeartbeatUtcTicks = DateTime.UtcNow.Ticks
   };
   ```
5. **Compute placement** for the new upper shard: `var placement = await placementEngine.ComputePlacementAsync(shard, newShardTier, ct)`.
6. **Create upper half VDE**: `Guid newVdeId = await shardCreator(shard.DataVdeId, upperTemplate, ct)` where upperTemplate has StartSlot=midSlot, EndSlot=shard.EndSlot.
7. **Build upper half descriptor**:
   ```
   var upperHalf = new DataShardDescriptor {
       DataVdeId = newVdeId,
       StartSlot = midSlot,
       EndSlot = shard.EndSlot,
       CapacityBlocks = shard.CapacityBlocks,
       UsedBlocks = 0,
       Health = DataShardHealth.Healthy,
       LastHeartbeatUtcTicks = DateTime.UtcNow.Ticks
   };
   ```
8. **Atomic index update** (within try/catch for rollback):
   a. `indexCatalog.RemoveDataShard(shard.DataVdeId)` -- remove original
   b. `indexCatalog.AddDataShard(lowerHalf)` -- add lower
   c. `indexCatalog.AddDataShard(upperHalf)` -- add upper
   If any step throws, attempt rollback: remove both halves, re-add original. Return Failed result.
9. **Routing table update**:
   a. `routingTable.AssignRange(lowerHalf.StartSlot, lowerHalf.EndSlot, lowerHalf.DataVdeId)`
   b. `routingTable.AssignRange(upperHalf.StartSlot, upperHalf.EndSlot, upperHalf.DataVdeId)`
10. **Background data redistribution**: `long blocksMoved = await dataRedistributor(shard.DataVdeId, newVdeId, midSlot, shard.EndSlot, ct)`.
11. Update upper half UsedBlocks: `upperHalf = upperHalf with { UsedBlocks = blocksMoved }`. Update in index: `indexCatalog.UpdateDataShard(upperHalf)`.
12. Stop stopwatch. Return `ShardSplitResult` with Status=Completed, all descriptors, placement, blocksMoved, elapsed.

Wrap entire operation in try/catch. On any exception after step 8, return Failed with exception message.

Thread safety: the method is NOT concurrent-safe for the same shard. Callers must serialize splits per shard (e.g., via a ConcurrentDictionary of per-shard semaphores at the orchestrator level).
