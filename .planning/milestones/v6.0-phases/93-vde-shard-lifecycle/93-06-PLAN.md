   - Creates a RoutingTable with `shardCount * slotsPerShard` total slots
   - Creates InMemoryShardVdeAccessor
   - For each shard: AddShard, assign slot range, populate with test objects (`$"shard-{i}/obj-{j}"`)
   - Returns the topology

Tests to implement (each is a private static async Task method):

**Migration tests:**
- `TestMigrationCompletesSuccessfully`: Create 2-shard topology, migrate shard 0 to a new destination, verify routing table slots updated, migration state is Completed.
- `TestMigrationRollbackOnFailure`: Create topology, start migration, simulate failure (by disposing the engine mid-migration or cancelling), verify routing table restored to original, state is RolledBack or Failed.
- `TestMigrationRedirectDuringCatchUp`: Create topology, register a migration in CatchingUp phase in the redirect table, verify TryGetRedirect returns destination shard ID. For CopyingBlocks phase, verify TryGetRedirect returns false.
- `TestMigrationProgressTracking`: Create topology, start migration, verify progress updates (BlocksCopied increases, PercentComplete > 0 after some copying).

**Migration state machine tests:**
- `TestMigrationStateSerializationRoundTrip`: Create ShardMigrationState, serialize to 128 bytes, deserialize, verify all fields match.
- `TestMigrationStateIllegalTransition`: Verify transitioning from Completed to CopyingBlocks throws InvalidOperationException.

**Transaction tests:**
- `TestTwoPhaseCommitSuccess`: Create 3-shard topology, begin transaction with all 3 as participants, verify all vote commit, commit transaction, verify state is Committed, all locks released.
- `TestTwoPhaseCommitAbort`: Begin transaction, simulate one participant unable to acquire lock (pre-acquire the lock with a different owner), verify transaction aborts, all locks released.
- `TestTransactionTimeout`: Begin transaction with very short timeout (100ms), delay, call TimeoutExpiredTransactionsAsync, verify transaction is aborted.

**Saga tests:**
- `TestSagaForwardExecutionSuccess`: Create 3-step saga where all steps succeed, verify all steps Completed, saga status Completed.
- `TestSagaCompensationOnFailure`: Create 3-step saga where step 2 fails, verify steps 0 and 1 are Compensated (reverse order), saga status Compensated.
- `TestSagaCompensationFailureRetriesAndFails`: Create saga where compensation of step 0 always fails, verify it retries MaxCompensationRetries times, saga status Failed.

**Zero-overhead test:**
- `TestSingleShardNoLifecycleOverhead`: Create single-shard topology, verify MigrationRedirectTable has no active migrations, verify no routing table lookups needed (single shard passthrough from Phase 92).

Each test uses `Debug.Assert` for assertions (matching existing codebase pattern). Throw descriptive exceptions on assertion failure so the test runner captures them.
