Create `ShardTransactionCoordinator.cs` in namespace `DataWarehouse.SDK.VirtualDiskEngine.Federation.Lifecycle`:

1. `enum TransactionPhase : byte`: `Initiated=0`, `Preparing=1`, `Prepared=2`, `Committing=3`, `Committed=4`, `Aborting=5`, `Aborted=6`, `TimedOut=7`.

2. `enum ParticipantVote : byte`: `None=0`, `VoteCommit=1`, `VoteAbort=2`, `NoResponse=3`.

3. `sealed class ShardTransactionParticipant` (internal record):
   - `Guid ShardId`, `ParticipantVote Vote`, `string? AbortReason`, `bool Committed`, `DateTimeOffset? PreparedAtUtc`

4. `sealed class ShardTransaction`:
   - Properties: `Guid TransactionId` (auto-generated), `TransactionPhase Phase` (volatile backing), `IReadOnlyList<ShardTransactionParticipant> Participants`, `DateTimeOffset CreatedAtUtc`, `TimeSpan Timeout`, `string? AbortReason`
   - `bool IsTerminal` (Committed, Aborted, TimedOut)
   - `bool IsTimedOut` computed from CreatedAtUtc + Timeout vs UtcNow
   - Binary serialization: `const int HeaderSize = 64`. Layout: [0..15] TransactionId, [16] Phase, [17..23] reserved, [24..31] CreatedAtUtcTicks, [32..39] TimeoutTicks, [40..43] ParticipantCount, [44..63] reserved. Each participant: 32 bytes ([0..15] ShardId, [16] Vote, [17] Committed as byte, [18..31] reserved).

5. `sealed class ShardTransactionCoordinator : IAsyncDisposable`:
   - Constructor: `IShardVdeAccessor shardAccessor`, `IDistributedLockService lockService`, `FederationOptions options`
   - `BoundedDictionary<Guid, ShardTransaction>` for active transactions (max 4096)
   - `async Task<ShardTransaction> BeginTransactionAsync(IReadOnlyList<Guid> participantShardIds, TimeSpan? timeout, CancellationToken ct)`:
     - Create ShardTransaction with participants, default timeout from `options.ShardOperationTimeout * 2`
     - Transition to Preparing
     - For each participant shard: acquire distributed lock `$"shard-txn:{shardId}:{transactionId}"` with lease = timeout. Record vote as VoteCommit on success, VoteAbort on lock failure.
     - If ALL participants voted VoteCommit: transition to Prepared, return transaction
     - If ANY voted VoteAbort: call AbortTransactionAsync, throw `InvalidOperationException($"Transaction aborted: participant {shardId} voted abort: {reason}")`

   - `async Task CommitTransactionAsync(Guid transactionId, CancellationToken ct)`:
     - Validate transaction is in Prepared phase
     - Transition to Committing
     - For each participant: mark Committed=true (in production this signals the shard to finalize its prepared writes)
     - Transition to Committed
     - Release all distributed locks
     - Remove from active transactions

   - `async Task AbortTransactionAsync(Guid transactionId, CancellationToken ct)`:
     - Transition to Aborting
     - Release all distributed locks (ignore failures)
     - Transition to Aborted
     - Remove from active transactions

   - `ShardTransaction? GetTransaction(Guid transactionId)`: lookup from active transactions

   - `IReadOnlyList<ShardTransaction> GetActiveTransactions()`: return non-terminal transactions

   - Background timeout check: `async Task TimeoutExpiredTransactionsAsync(CancellationToken ct)` that iterates active transactions, aborts any where `IsTimedOut` is true.

   - `IAsyncDisposable`: abort all active transactions on dispose.

All classes `[SdkCompatibility("6.0.0")]`. Use `ArgumentNullException.ThrowIfNull` on all constructor params. Lock owner: `$"txn-coord-{Environment.MachineName}"`.

Create `ShardSagaOrchestrator.cs` in namespace `DataWarehouse.SDK.VirtualDiskEngine.Federation.Lifecycle`:

1. `enum SagaStepStatus : byte`: `Pending=0`, `Executing=1`, `Completed=2`, `CompensationNeeded=3`, `Compensating=4`, `Compensated=5`, `CompensationFailed=6`.

2. `enum SagaStatus : byte`: `Running=0`, `Completed=1`, `Compensating=2`, `Compensated=3`, `Failed=4` (compensation itself failed).

3. `sealed class SagaStep`:
   - `int StepIndex`, `Guid TargetShardId`, `string OperationKey` (identifies the write op), `SagaStepStatus Status`, `string? FailureReason`
   - `Func<CancellationToken, Task<bool>> ExecuteAsync` (the forward action, returns true on success)
   - `Func<CancellationToken, Task<bool>> CompensateAsync` (the rollback action, returns true on success)

4. `sealed class SagaDefinition`:
   - `Guid SagaId` (auto-generated), `IReadOnlyList<SagaStep> Steps`, `SagaStatus Status`, `int MaxCompensationRetries` (default 3), `TimeSpan StepTimeout`
   - `int CompletedStepCount` computed property
   - `int FailedStepIndex` — index of first failed step, or -1

5. `sealed class ShardSagaOrchestrator : IAsyncDisposable`:
   - Constructor: `IDistributedLockService lockService`, `FederationOptions options`
   - `BoundedDictionary<Guid, SagaDefinition>` for active sagas (max 4096)

   - `SagaDefinition CreateSaga(IReadOnlyList<SagaStep> steps, TimeSpan? stepTimeout)`:
     - Validate steps not empty, all have non-null Execute and Compensate delegates
     - Create SagaDefinition with Running status, default stepTimeout from options.ShardOperationTimeout

   - `async Task<SagaDefinition> ExecuteSagaAsync(SagaDefinition saga, CancellationToken ct)`:
     - Execute steps in order (index 0, 1, 2, ...)
     - For each step: transition to Executing, acquire lock `$"saga:{saga.SagaId}:step:{step.StepIndex}"`, call ExecuteAsync with a CancellationTokenSource linked to stepTimeout
     - On success: transition to Completed, release lock
     - On failure (returns false or throws): record FailureReason, transition to CompensationNeeded, set saga to Compensating, call `CompensateFromStepAsync(saga, stepIndex - 1, ct)` to undo all previously completed steps in REVERSE order
     - After all steps complete: set saga to Completed
     - Return updated saga definition

   - Private `async Task CompensateFromStepAsync(SagaDefinition saga, int fromStepIndex, CancellationToken ct)`:
     - Iterate from `fromStepIndex` down to 0 (reverse order)
     - For each Completed step: transition to Compensating, call CompensateAsync
     - On compensation success: transition to Compensated
     - On compensation failure: retry up to MaxCompensationRetries with exponential backoff (100ms * 2^attempt, max 5s). If all retries fail: transition to CompensationFailed, set saga status to Failed, throw `InvalidOperationException($"Saga {saga.SagaId} compensation failed at step {stepIndex}")`
     - After all compensations succeed: set saga to Compensated

   - `SagaDefinition? GetSaga(Guid sagaId)`: lookup
   - `IReadOnlyList<SagaDefinition> GetActiveSagas()`: non-terminal sagas

   - `IAsyncDisposable`: best-effort compensate all running sagas on dispose.

All classes `[SdkCompatibility("6.0.0")]`. Use `BoundedDictionary` from SDK.Utilities. Lock owner: `$"saga-orch-{Environment.MachineName}"`.
