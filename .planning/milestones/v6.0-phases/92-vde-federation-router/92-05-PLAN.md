public sealed class MergeSortStreamMerger
```

This class performs K-way merge-sort over multiple IAsyncEnumerable<StorageObjectMetadata> streams from different shards, producing a single unified sorted output.

Implementation:
- Constructor: `MergeSortStreamMerger(IComparer<StorageObjectMetadata>? comparer = null)` — defaults to sorting by Key (ordinal string comparison).
- Method: `IAsyncEnumerable<StorageObjectMetadata> MergeAsync(IReadOnlyList<ShardStream> sources, CancellationToken ct = default)` — performs K-way merge using a min-heap (PriorityQueue<(StorageObjectMetadata Item, int SourceIndex), StorageObjectMetadata>).
- `ShardStream` is a readonly record struct: `readonly record struct ShardStream(Guid ShardVdeId, IAsyncEnumerator<StorageObjectMetadata> Enumerator)`.
- The merge loop: initialize heap by advancing each enumerator once. Dequeue min, yield it, advance that enumerator and re-enqueue if not exhausted. Continue until heap is empty.
- Dispose all enumerators in finally block (even on cancellation).
- Handle empty streams gracefully (skip during initialization).
- For a single source (1 shard), yield directly from it without heap allocation (zero overhead path).

public sealed class FederatedPaginationCursor
```

Encodes per-shard positions into an opaque Base64 continuation token for deterministic cross-shard pagination.

Implementation:
- Internal state: `Dictionary<Guid, long> _shardOffsets` — maps ShardVdeId to the number of items already consumed from that shard.
- Static method: `FederatedPaginationCursor Create(IReadOnlyDictionary<Guid, long> shardOffsets)` — creates from shard positions.
- Method: `string Encode()` — serializes shard offsets to bytes using BinaryPrimitives (write int32 count, then for each entry write Guid bytes + int64 offset LE), then Convert.ToBase64String. Total size per entry = 24 bytes (16 guid + 8 long).
- Static method: `FederatedPaginationCursor Decode(string token)` — Convert.FromBase64String, read int32 count, then read entries. Throw FormatException on malformed input.
- Method: `long GetOffset(Guid shardVdeId)` — returns offset for shard, or 0 if shard not in cursor (new shard added after cursor was created).
- Method: `FederatedPaginationCursor WithAdvanced(Guid shardVdeId, long itemsConsumed)` — returns new cursor with updated offset for the given shard (immutable pattern).
- Property: `bool IsEmpty => _shardOffsets.Count == 0`
- Static field: `static readonly FederatedPaginationCursor Empty = new(new Dictionary<Guid, long>())`

Use BinaryPrimitives for all serialization (no BinaryReader/Writer). Validate cursor integrity on decode (check byte length matches declared count).

Methods:

1. `IAsyncEnumerable<StorageObjectMetadata> FanOutListAsync(string? prefix, int pageSize, FederatedPaginationCursor? cursor, CancellationToken ct = default)`:
   - Get all shard IDs from shardAccessor.
   - If prefix is provided, use router to determine which shards could contain matching keys (path-embedded routing narrows the target set). If router returns null or cannot narrow, fan out to ALL shards.
   - For each target shard, call `ListShardAsync(shardId, prefix)`, skipping items according to cursor offsets for that shard.
   - Feed all per-shard streams into MergeSortStreamMerger.MergeAsync.
   - Yield merged results, counting up to pageSize. Track per-shard consumed counts.
   - After yielding pageSize items (or all exhausted), encode a new FederatedPaginationCursor.
   - Use `[EnumeratorCancellation]` on ct parameter.

2. `IAsyncEnumerable<StorageObjectMetadata> FanOutSearchAsync(string query, int maxResults, CancellationToken ct = default)`:
   - Fan out SearchShardAsync to ALL shards in parallel (search cannot be narrowed by prefix routing).
   - Merge-sort results by key (or by a relevance score if SearchShardAsync provides one — default to key sort).
   - Yield up to maxResults items.
   - Use `Task.WhenAll` pattern to start all enumerators, then merge.

3. `Task FanOutDeleteAsync(string key, CancellationToken ct = default)`:
   - Use router to resolve which shard owns the key (single shard for point deletes).
   - Call `DeleteFromShardAsync(targetShardId, key)`.
   - If router returns no shard, attempt delete on all shards (defensive — eventual consistency scenario).

4. `Task<AggregatedMetadataStats> AggregateMetadataAsync(CancellationToken ct = default)`:
   - Get all shard IDs.
   - Fan out `GetShardStatsAsync` to all shards using `Task.WhenAll`.
   - Sum ObjectCount, TotalSizeBytes, UsedBlocks, FreeBlocks.
   - Return `AggregatedMetadataStats` (new readonly record struct defined here):
     ```csharp
     public readonly record struct AggregatedMetadataStats(int ShardCount, long TotalObjectCount, long TotalSizeBytes, long TotalUsedBlocks, long TotalFreeBlocks);
     ```

Single-shard optimization: If `GetAllShardIdsAsync` returns exactly 1 shard, skip merge overhead — delegate directly to that shard's ListAsync/SearchAsync without creating MergeSortStreamMerger or cursor machinery. This ensures zero overhead for single-VDE deployments.

Error handling: If a shard is unreachable during fan-out, log warning and continue with remaining shards (partial results are better than total failure). Track unreachable shards in cursor so retry can target them.

All methods use CancellationToken throughout. No blocking calls. Use ConfigureAwait(false) on all awaits.
