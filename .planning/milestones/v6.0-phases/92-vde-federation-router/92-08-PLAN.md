```csharp
[SdkCompatibility("6.0.0")]
public static class FederationIntegrationTests
```

Static test class with a `RunAllAsync` entry point and individual test methods. Each test returns (string testName, bool passed, string? errorMessage). Uses Assert helper (throw on failure).

Test categories and methods:

**Category 1: Routing Correctness**

1. `TestRouterResolvesCorrectShard` — Create VdeFederationRouter with 3 shards mapped to different namespace prefixes ("/users/", "/orders/", "/products/"). Verify Resolve("/users/alice") routes to shard 1, Resolve("/orders/12345") routes to shard 2, etc.

2. `TestRouterWarmCacheHit` — Resolve a path twice. First call is cold (>1 hop). Second call should be warm-cached (1 hop). Verify HopsUsed on second call is 1.

3. `TestRouterColdPathMaxHops` — In a multi-level setup (super-federation -> federation -> leaf), verify cold resolution takes at most 5 hops.

4. `TestRouterUnknownPathThrows` — Attempt to resolve a path that no shard handles. Verify InvalidOperationException is thrown.

**Category 2: Bloom Filter Integration**

5. `TestBloomFilterRejectsNonexistentKey` — Create ShardBloomFilterIndex with 3 shards. Add 10,000 keys to shard 1. Query MayContain for 10,000 keys NOT in shard 1. Verify false positive rate is under 1% (expect <0.1% with default config).

6. `TestBloomFilterAcceptsExistingKey` — Add key "test-key" to shard 1's filter. Verify MayContain returns true (zero false negatives).

7. `TestFilterCandidateShards` — With 3 shards, add a key only to shard 2. Call FilterCandidateShards for that key. Verify shard 2 is in result, shards 1 and 3 are filtered out.

8. `TestBloomFilterPersistenceRoundTrip` — Create filter, add keys, take snapshot via GetSnapshot, write via BloomFilterPersistence.WriteAsync to an InMemoryBlockDevice (reuse from Phase 91 tests or create minimal one), read back via ReadAsync, load snapshot, verify MayContain returns same results.

**Category 3: Cross-Shard Operations**

9. `TestCrossShardListMergesSorted` — Create InMemoryShardVdeAccessor with 3 shards, add objects with interleaved keys (shard1: "a", "d", "g"; shard2: "b", "e", "h"; shard3: "c", "f", "i"). FanOutListAsync with no prefix. Verify results come out in sorted order: a, b, c, d, e, f, g, h, i.

10. `TestCrossShardListWithPrefix` — Add objects with different prefixes to different shards. FanOutListAsync with prefix "users/". Verify only matching objects returned, still sorted.

11. `TestCrossShardPagination` — Add 30 objects across 3 shards. FanOutListAsync with pageSize=10. Collect first page (10 items), get cursor, collect second page with cursor (10 items), third page (10 items). Verify all 30 unique items returned, no duplicates, sorted order maintained across pages.

12. `TestCrossShardSearch` — Add objects to 3 shards. FanOutSearchAsync with query "user". Verify results from all shards containing "user" are returned, merge-sorted.

13. `TestCrossShardDelete` — Add object "orders/123" to shard 2. FanOutDeleteAsync("orders/123"). Verify object removed from shard 2 (ListShardAsync no longer returns it).

14. `TestCrossShardMetadataAggregation` — Add objects to 3 shards with known sizes. AggregateMetadataAsync. Verify TotalObjectCount and TotalSizeBytes equal sum across all shards.

**Category 4: Single-VDE Parity**

15. `TestSingleShardListMatchesDirect` — Create accessor with 1 shard, add objects. FanOutListAsync result should be identical to ListShardAsync result (same order, same items). Verifies zero overhead path.

16. `TestSingleShardDeleteMatchesDirect` — Same pattern: single shard delete via coordinator matches direct delete.

**Category 5: Recursive Composition**

17. `TestSuperFederationResolvesThreeLevels` — Create SuperFederationRouter containing two FederationNodes, each containing LeafVdeNodes. Resolve a path. Verify it reaches the correct leaf with HopsUsed reflecting traversal depth.

18. `TestSuperFederationTopology` — Build a 3-level topology. Call FederationTopology.BuildAsync and SummarizeAsync. Verify node counts, depths, and stats match expected values.

19. `TestSuperFederationMaxHopProtection` — Create circular reference (if possible through interface) or just verify that resolution with depth > 10 throws InvalidOperationException.
