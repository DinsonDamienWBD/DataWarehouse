<objective>
Build the transparent namespace layer: FederatedVirtualDiskEngine that exposes the same Store/Retrieve/Delete/List/Search API as VirtualDiskEngine but routes operations through the federation router. For single-VDE deployments, it delegates directly to the inner VDE with zero overhead. For federated deployments, it resolves shard addresses and dispatches operations. Cross-shard List/Search use fan-out with merge-sort and pagination.

Purpose: This is the user-facing facade that makes federation invisible. Users call the same API regardless of deployment scale.
Output: 3 files providing the federated VDE facade, cross-shard merger, and federation options.
</objective>

**FederationOptions.cs** -- configuration for federated VDE:
- `[SdkCompatibility("6.0.0", Notes = "Phase 92: VDE 2.0B Federation Options (VFED-14)")]`
- `public sealed record FederationOptions`
- Properties:
  - `FederationMode Mode { get; init; } = FederationMode.SingleVde` -- default to single-VDE (zero overhead)
  - `TimeSpan ShardOperationTimeout { get; init; } = TimeSpan.FromSeconds(30)` -- per-shard operation timeout
  - `int MaxConcurrentShardOperations { get; init; } = 16` -- max parallel fan-out for List/Search
  - `int ListPageSize { get; init; } = 1000` -- items per page in cross-shard List
  - `int SearchMaxResults { get; init; } = 10000` -- max total results for cross-shard Search
  - `bool EnableCrossShardPagination { get; init; } = true`
  - `TimeSpan CrossShardMergeTimeout { get; init; } = TimeSpan.FromSeconds(60)` -- total timeout for fan-out+merge
  - `int MaxRetryPerShard { get; init; } = 2` -- retry failed shard operations
- `public enum FederationMode : byte { SingleVde = 0, Federated = 1, SuperFederated = 2 }`
- `public void Validate()` -- throws on invalid combinations (e.g. MaxConcurrentShardOperations < 1)

**CrossShardMerger.cs** -- fan-out and merge for multi-shard operations:
- `[SdkCompatibility("6.0.0", Notes = "Phase 92: VDE 2.0B Cross-Shard Merger (VFED-15)")]`
- `public sealed class CrossShardMerger`
- Constructor: `CrossShardMerger(FederationOptions options)`

- `public async IAsyncEnumerable<StorageObjectMetadata> MergeListAsync(string? prefix, IReadOnlyList<ShardOperationTarget> targets, [EnumeratorCancellation] CancellationToken ct = default)`
  - `ShardOperationTarget` is a record: `public readonly record struct ShardOperationTarget(Guid ShardVdeId, Func<string?, CancellationToken, IAsyncEnumerable<StorageObjectMetadata>> ListFunc)`
  - Fan-out: start `IAsyncEnumerator` for each target in parallel (up to MaxConcurrentShardOperations)
  - Merge: k-way merge-sort by Key (lexicographic) using a priority queue (`PriorityQueue<(StorageObjectMetadata Item, int ShardIndex), string>`)
  - Each shard enumerator advances independently. When one shard is exhausted, it is removed from the queue.
  - Pagination: yield items one at a time. Caller can break the enumeration to implement page boundaries.
  - Error handling: if a shard fails, log warning and continue with remaining shards (partial results are better than total failure for List)

- `public async IAsyncEnumerable<StorageObjectMetadata> MergeSearchAsync(string query, IReadOnlyList<ShardSearchTarget> targets, [EnumeratorCancellation] CancellationToken ct = default)`
  - `ShardSearchTarget`: `public readonly record struct ShardSearchTarget(Guid ShardVdeId, Func<string, CancellationToken, IAsyncEnumerable<StorageObjectMetadata>> SearchFunc)`
  - Same k-way merge pattern but with result count limit (SearchMaxResults)
  - Yield at most SearchMaxResults items total across all shards

- `public async Task<long> FanOutDeleteAsync(string key, IReadOnlyList<ShardDeleteTarget> targets, CancellationToken ct = default)`
  - `ShardDeleteTarget`: `public readonly record struct ShardDeleteTarget(Guid ShardVdeId, Func<string, CancellationToken, Task> DeleteFunc)`
  - Send delete to resolved shard only (not fan-out -- key has a single home). This method exists for consistency but the federation router resolves the key first.
  - Returns count of shards that successfully processed the delete (should be 1)

- `public CrossShardMergeStats GetStats()` -- track fan-out count, merge latency, partial failures
- `public readonly record struct CrossShardMergeStats(long TotalFanOuts, long PartialFailures, long TotalItemsMerged);`

**FederatedVirtualDiskEngine.cs** -- transparent facade over single or federated VDE:
- `[SdkCompatibility("6.0.0", Notes = "Phase 92: VDE 2.0B Federated VDE (VFED-16)")]`
- `public sealed class FederatedVirtualDiskEngine : IAsyncDisposable`
- Constructor: `FederatedVirtualDiskEngine(FederationOptions options, VdeFederationRouter router, CrossShardMerger merger, Func<Guid, CancellationToken, ValueTask<VirtualDiskEngine?>> vdeResolver)`
  - `vdeResolver`: async factory that gets a VDE 2.0A instance by its GUID (from a VDE pool/registry)
- Private fields: `_options`, `_router`, `_merger`, `_vdeResolver`, `_singleVde` (nullable, set in single-VDE mode)

- **Static factory for single-VDE** (zero overhead path):
  `public static FederatedVirtualDiskEngine CreateSingleVde(VirtualDiskEngine vde, Guid vdeId)`
  - Sets `_singleVde = vde`, creates passthrough router via `VdeFederationRouter.CreateSingleVde(vdeId)`
  - All operations delegate DIRECTLY to `_singleVde` with no routing, no hashing, no cache lookup

- **StoreAsync**: `public async Task<StorageObjectMetadata> StoreAsync(string key, Stream data, IDictionary<string, string>? metadata, CancellationToken ct = default)`
  - If `_singleVde != null`: return `await _singleVde.StoreAsync(key, data, metadata, ct)` -- DIRECT passthrough
  - Else: resolve shard via `_router.ResolveAsync(key, ct)`, get VDE via `_vdeResolver(address.VdeId, ct)`, delegate `StoreAsync` to that VDE

- **RetrieveAsync**: `public async Task<Stream> RetrieveAsync(string key, CancellationToken ct = default)`
  - If `_singleVde != null`: return `await _singleVde.RetrieveAsync(key, ct)` -- DIRECT passthrough
  - Else: resolve shard, get VDE, delegate

- **DeleteAsync**: `public async Task DeleteAsync(string key, CancellationToken ct = default)`
  - If `_singleVde != null`: `await _singleVde.DeleteAsync(key, ct)` -- DIRECT passthrough
  - Else: resolve shard, get VDE, delegate

- **ExistsAsync**: `public async Task<bool> ExistsAsync(string key, CancellationToken ct = default)`
  - Same passthrough/resolve pattern

- **GetMetadataAsync**: `public async Task<StorageObjectMetadata> GetMetadataAsync(string key, CancellationToken ct = default)`
  - Same passthrough/resolve pattern

- **ListAsync**: `public IAsyncEnumerable<StorageObjectMetadata> ListAsync(string? prefix, CancellationToken ct = default)`
  - If `_singleVde != null`: return `_singleVde.ListAsync(prefix, ct)` -- DIRECT passthrough
  - Else: build `ShardOperationTarget` list from all known data shards (query root catalog for all domains, enumerate), delegate to `_merger.MergeListAsync(prefix, targets, ct)`
  - Helper method: `private async Task<IReadOnlyList<ShardOperationTarget>> BuildListTargetsAsync(string? prefix, CancellationToken ct)` -- uses routing table to identify which shards could contain the prefix (if prefix maps to a single domain slot, only that shard is queried; if null/empty prefix, all shards)

- **SearchAsync**: `public IAsyncEnumerable<StorageObjectMetadata> SearchAsync(string query, CancellationToken ct = default)`
  - If `_singleVde != null`: throw NotSupportedException("Search requires VDE 2.0B+ federation") -- or delegate if single VDE has search
  - Else: fan-out to all shards via merger

- Private helper: `private async ValueTask<VirtualDiskEngine> ResolveVdeForKeyAsync(string key, CancellationToken ct)`
  - Calls `_router.ResolveAsync(key, ct)`, validates result, calls `_vdeResolver`, throws `KeyNotFoundException` if shard not found or VDE unavailable

- **DisposeAsync**: dispose router, release VDE references

- **Stats**: `public FederatedVdeStats GetStats()` -- aggregate router stats + merger stats
- `public readonly record struct FederatedVdeStats(FederationMode Mode, FederationRouterStats RouterStats, CrossShardMergeStats MergerStats, int ActiveShardCount);`
