**WinFspMountHandle** class with `[SdkCompatibility("6.0.0")]`, `[SupportedOSPlatform("windows")]`:
- Implements `IMountHandle`
- Properties: `MountPoint`, `VdePath`, `IsActive`, `MountedAtUtc`, `Options`
- Internal fields: `VdeFilesystemAdapter _adapter`, `Thread? _mountThread`, `CancellationTokenSource _cts`
- `DisposeAsync()`: signals cancellation, waits for mount thread to exit, calls `_adapter.SyncAsync`, sets `IsActive = false`

**WinFspMountProvider** class with `[SdkCompatibility("6.0.0")]`, `[SupportedOSPlatform("windows")]`:
- Implements `IVdeMountProvider`
- `SupportedPlatforms => PlatformFlags.Windows`
- `IsAvailable`: Check at runtime if WinFsp is installed by probing the registry key `HKLM\SOFTWARE\WinFsp` or checking if `winfsp-x64.dll` exists in Program Files. Use `RuntimeInformation.IsOSPlatform(OSPlatform.Windows)` guard. Return false on non-Windows or if WinFsp is not installed.

**MountAsync implementation:**
1. Validate `mountPoint` is a valid drive letter (e.g., "Z:" or "Z:\") or UNC path
2. Create VdeFilesystemAdapter from the VDE file (open IBlockDevice, IInodeTable, etc.)
3. Create a WinFsp FileSystemHost-like abstraction. Since we cannot take a direct dependency on WinFsp.Net NuGet (SDK must remain dependency-free), implement the WinFsp binding via P/Invoke to `winfsp-x64.dll`:
   - Define internal static class `WinFspNative` with P/Invoke declarations:
     - `FspFileSystemCreate(string devicePath, ref FSP_FILE_SYSTEM_INTERFACE fsInterface, out IntPtr fileSystem)`
     - `FspFileSystemSetMountPoint(IntPtr fileSystem, string mountPoint)`
     - `FspFileSystemStartDispatcher(IntPtr fileSystem, int threadCount)`
     - `FspFileSystemStopDispatcher(IntPtr fileSystem)`
     - `FspFileSystemDelete(IntPtr fileSystem)`
   - Define `FSP_FILE_SYSTEM_INTERFACE` struct with function pointers for: `GetVolumeInfo`, `GetSecurityByName`, `Open`, `Close`, `Read`, `Write`, `Flush`, `GetFileInfo`, `SetBasicInfo`, `SetFileSize`, `Cleanup`, `ReadDirectory`, `Create`, `Overwrite`, `Rename`, `SetSecurity`, `GetSecurity`
4. Wire each WinFsp callback to VdeFilesystemAdapter:
   - `GetVolumeInfo` -> `adapter.StatfsAsync` (total size, free space, volume label "DWVD")
   - `Open` -> `adapter.LookupAsync` + `adapter.GetattrAsync`
   - `Close` -> `adapter.FlushAsync`
   - `Read` -> `adapter.ReadAsync`
   - `Write` -> `adapter.WriteAsync`
   - `GetFileInfo` -> `adapter.GetattrAsync`
   - `ReadDirectory` -> `adapter.ReaddirAsync`
   - `Create` -> `adapter.CreateAsync`
   - `Cleanup` -> `adapter.UnlinkAsync` (if delete-on-close flag set)
   - `Rename` -> `adapter.RenameAsync`
   - `Flush` -> `adapter.FlushAsync`
   - `SetBasicInfo` -> `adapter.SetattrAsync` (timestamps)
   - `SetFileSize` -> `adapter.SetattrAsync` (size/truncate)
5. Start the WinFsp dispatcher on a background thread
6. Return `WinFspMountHandle`

**UnmountAsync:**
- Stop the WinFsp dispatcher
- Flush via adapter.SyncAsync
- Delete the filesystem object
- Set handle IsActive = false

**ListMountsAsync:**
- Track active mounts in a `ConcurrentDictionary<string, WinFspMountHandle>`
- Return MountInfo for each active handle

**Error mapping:**
- Map VDE exceptions to WinFsp NTSTATUS codes:
  - FileNotFoundException -> STATUS_OBJECT_NAME_NOT_FOUND (0xC0000034)
  - DirectoryNotEmptyException -> STATUS_DIRECTORY_NOT_EMPTY (0xC0000101)
  - UnauthorizedAccessException -> STATUS_ACCESS_DENIED (0xC0000022)
  - IOException -> STATUS_UNEXPECTED_IO_ERROR (0xC00000E9)
  - OperationCanceledException -> STATUS_CANCELLED (0xC0000120)
  - Success -> STATUS_SUCCESS (0)

**Important:** All P/Invoke signatures are internal. The WinFsp DLL is loaded dynamically at runtime (not a compile-time dependency). If the DLL is not found, `IsAvailable` returns false and `MountAsync` throws `PlatformNotSupportedException`.
