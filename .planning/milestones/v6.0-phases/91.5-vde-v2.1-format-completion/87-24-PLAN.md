<objective>
Implement Smart Extents WASM Predicate Pushdown (CPSH, Module bit 19, VOPT-37): per-inode WASM predicate module enabling io_uring filtered reads and computational NVMe drive-side execution.

Purpose: Store WASM filter bytecode in the inode so the VDE engine can push predicates to the storage layer. On standard NVMe, WASM executes after DMA but before user-space copy (saves memory bandwidth). On computational NVMe (Samsung SmartSSD), the drive's ARM processor executes WASM directly on flash.
