Create `ModulePopulatorStage.cs` with `[SdkCompatibility("6.0.0")]`:
- Namespace: `DataWarehouse.SDK.VirtualDiskEngine.Pipeline.Stages`
- `class ModulePopulatorStage : IVdeWriteStage`:
  - Implements Write Stage 2 from VDE spec (Module Populator)
  - `StageName => "ModulePopulator"`
  - `ModuleGate => null` -- always runs (it internally gates per-module)
  - Constructor: `IReadOnlyDictionary<ModuleId, IModuleFieldHandler> handlers` -- registered per-module handlers
  - `async Task ExecuteAsync(VdePipelineContext context, CancellationToken ct)`:
    1. Get active modules: `var activeModules = ModuleRegistry.GetActiveModules(context.Manifest)`
    2. For each active module where a handler is registered (`_handlers.TryGetValue(module.Id, out var handler)`):
       a. Allocate `Memory<byte>` buffer of `handler.FieldSizeBytes` (skip if 0 bytes)
       b. Call `await handler.PopulateAsync(context, buffer, ct)`
       c. Store result in `context.ModuleFields[module.Id] = buffer` (as ReadOnlyMemory)
    3. For active modules with NO registered handler: store zero-filled bytes of `module.InodeFieldBytes` in `context.ModuleFields` (safe default -- modules not yet implemented get zeroed fields)
  - Property: `IReadOnlyDictionary<ModuleId, IModuleFieldHandler> Handlers => _handlers`

Create `ModuleExtractorStage.cs` with `[SdkCompatibility("6.0.0")]`:
- Namespace: `DataWarehouse.SDK.VirtualDiskEngine.Pipeline.Stages`
- `class ModuleExtractorStage : IVdeReadStage`:
  - Implements Read Stage 5 from VDE spec (Module Extractor)
  - `StageName => "ModuleExtractor"`
  - `ModuleGate => null` -- always runs (it internally gates per-module)
  - Constructor: `IReadOnlyDictionary<ModuleId, IModuleFieldHandler> handlers`
  - `async Task ExecuteAsync(VdePipelineContext context, CancellationToken ct)`:
    1. Get active modules from manifest
    2. Calculate cumulative field offsets: for each active module in bit order, offset += previous module's InodeFieldBytes. This determines where each module's bytes live in the inode extension area.
    3. For each active module:
       a. Slice the inode extension bytes: `ReadOnlyMemory<byte> fieldBytes = inodeExtensionArea.Slice(offset, module.InodeFieldBytes)`
       b. Store raw bytes: `context.ModuleFields[module.Id] = fieldBytes`
       c. If handler exists: `await handler.ExtractAsync(context, fieldBytes, ct)` -- handler may set context.Properties with parsed values
    4. For modules with no handler: raw bytes are still in ModuleFields for downstream access
  - The inode extension area is read from `context.Inode` (ExtendedInode512). Access via a helper that gets the raw extension bytes from the inode's serialized form, or via `context.Properties["InodeExtensionBytes"]` set by a prior stage (InodeLookup).

Both stages:
- Log each handled module to `context.StagesExecuted` as sub-entries (e.g., "ModulePopulator:Security", "ModulePopulator:RAID")
- Are tolerant of missing handlers (log warning but continue -- extensibility pattern)
- Use `using DataWarehouse.SDK.VirtualDiskEngine.Format;` for ModuleRegistry, ModuleId, VdeModule
