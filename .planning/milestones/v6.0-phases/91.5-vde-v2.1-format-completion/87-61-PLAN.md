Create `VdeFilesystemAdapter` class with `[SdkCompatibility("6.0.0")]`. This is the shared storage logic layer that ALL platform mount providers delegate to. It translates filesystem operations into VDE inode/extent/block operations.

**Constructor:**
`VdeFilesystemAdapter(IBlockDevice device, IInodeTable inodeTable, IBlockAllocator allocator, IWriteAheadLog wal, IArcCache cache, MountOptions options)`

**Core filesystem operations (all async, all CancellationToken-aware):**

Lookup:
- `Task<InodeLookupResult> LookupAsync(long parentInode, string name, CancellationToken ct)`
  - Read directory entries from `inodeTable.ReadDirectoryAsync(parentInode)`
  - Find entry matching `name`
  - Return inode number + InodeType, or not-found status
- `InodeLookupResult` readonly struct: `long InodeNumber, InodeType Type, bool Found`

Getattr:
- `Task<InodeAttributes> GetattrAsync(long inodeNumber, CancellationToken ct)`
  - Get inode via `inodeTable.GetInodeAsync`
  - Map to `InodeAttributes` (size, permissions, timestamps, link count, type, blocks)
- `InodeAttributes` readonly struct: `long InodeNumber, InodeType Type, long Size, long AllocatedSize, InodePermissions Permissions, int LinkCount, Guid OwnerId, long CreatedUtc, long ModifiedUtc, long AccessedUtc, long ChangedUtc, int BlockSize`

Readdir:
- `Task<IReadOnlyList<DirectoryEntryInfo>> ReaddirAsync(long directoryInode, CancellationToken ct)`
  - Read directory entries, include "." and ".." synthetic entries
  - Return list of `DirectoryEntryInfo` (name, inode number, type)
- `DirectoryEntryInfo` readonly struct: `string Name, long InodeNumber, InodeType Type`

Read:
- `Task<int> ReadAsync(long inodeNumber, Memory<byte> buffer, long offset, CancellationToken ct)`
  - Get inode, resolve extent tree for the given offset
  - For each extent covering the requested range:
    - Try ARC cache first (`cache.GetAsync`)
    - On miss, read from `device.ReadBlockAsync`, put in cache
    - Copy relevant bytes into buffer
  - Return bytes read (may be less than buffer if past EOF)

Write:
- `Task<int> WriteAsync(long inodeNumber, ReadOnlyMemory<byte> data, long offset, CancellationToken ct)`
  - If `options.ReadOnly`, throw `InvalidOperationException`
  - Get inode, resolve/allocate extents for write range
  - For each block in write range:
    - If partial block: read-modify-write (read existing, overlay, write back)
    - WAL-journal the write
    - Write via `device.WriteBlockAsync`
    - Update cache via `cache.PutAsync`
  - Update inode size if write extends past EOF
  - Update `ModifiedUtc` timestamp
  - Return bytes written

Create:
- `Task<long> CreateAsync(long parentInode, string name, InodeType type, InodePermissions permissions, CancellationToken ct)`
  - Allocate new inode via `inodeTable.AllocateInodeAsync(type)`
  - Set permissions, timestamps, owner
  - Add directory entry to parent via `inodeTable.AddDirectoryEntryAsync`
  - Return new inode number

Unlink:
- `Task UnlinkAsync(long parentInode, string name, CancellationToken ct)`
  - Lookup entry in parent directory
  - Decrement link count on target inode
  - If link count reaches 0: free all extents via `allocator.FreeExtent`, free inode via `inodeTable.FreeInodeAsync`
  - Remove directory entry via `inodeTable.RemoveDirectoryEntryAsync`

Mkdir:
- `Task<long> MkdirAsync(long parentInode, string name, InodePermissions permissions, CancellationToken ct)`
  - Create directory inode, add "." entry (self) and ".." entry (parent)
  - Add directory entry in parent, increment parent link count

Rmdir:
- `Task RmdirAsync(long parentInode, string name, CancellationToken ct)`
  - Verify directory is empty (only "." and "..")
  - Remove directory entry from parent, decrement parent link count
  - Free directory inode

Rename:
- `Task RenameAsync(long oldParentInode, string oldName, long newParentInode, string newName, CancellationToken ct)`
  - Lookup source entry, remove from old parent
  - If destination exists: unlink destination first
  - Add entry in new parent with source inode number

Symlink:
- `Task<long> SymlinkAsync(long parentInode, string name, string target, CancellationToken ct)`
  - Create symlink inode, store target path as inline data in inode extents
  - Add directory entry in parent

Readlink:
- `Task<string> ReadlinkAsync(long inodeNumber, CancellationToken ct)`
  - Read symlink target from inode data

Setattr:
- `Task SetattrAsync(long inodeNumber, InodeAttributes attrs, InodeAttributeMask mask, CancellationToken ct)`
  - Update only fields indicated by mask (size, permissions, timestamps, owner)
  - If size changed (truncate): free excess extents or allocate new ones
- `InodeAttributeMask` flags enum: `None = 0, Size = 1, Permissions = 2, Owner = 4, AccessTime = 8, ModifyTime = 16, ChangeTime = 32`

Flush/Sync:
- `Task FlushAsync(long inodeNumber, CancellationToken ct)` -- flush cached writes for inode
- `Task SyncAsync(CancellationToken ct)` -- flush entire device via `device.FlushAsync`

Statfs:
- `Task<FilesystemStats> StatfsAsync(CancellationToken ct)`
  - Return block size, total blocks, free blocks, inode count from allocator/inodeTable
- `FilesystemStats` readonly struct: `int BlockSize, long TotalBlocks, long FreeBlocks, long AllocatedInodes, long MaxNameLength = 255`

Xattr operations (delegated to inode extended attributes):
- `Task SetXattrAsync(long inodeNumber, string key, byte[] value, CancellationToken ct)`
- `Task<byte[]?> GetXattrAsync(long inodeNumber, string key, CancellationToken ct)`
- `Task<IReadOnlyList<string>> ListXattrAsync(long inodeNumber, CancellationToken ct)`

Thread safety: Use `SemaphoreSlim` with `MaxConcurrentOps` from options to throttle concurrent operations. Each operation acquires the semaphore, performs work, releases.

All operations must handle `OperationCanceledException` correctly and clean up partial state.
