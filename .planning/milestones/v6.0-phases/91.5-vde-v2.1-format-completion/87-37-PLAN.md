Create `ProbabilisticCorruptionRadar` class with `[SdkCompatibility("6.0.0")]`:
- Constructor: `IBlockDevice device, int blockSize, CorruptionRadarConfig config`

TRLR block positions (per separated trailer architecture):
- `IReadOnlyList<long> GenerateTrlrBlockPositions(long dataRegionStartBlock, long dataRegionBlockCount)`:
  - TRLR blocks at positions: startBlock + 255, startBlock + 511, startBlock + 767, ...
  - Every 256th block starting at offset 255 within data region

Reservoir sampling:
- `IReadOnlyList<long> SelectSample(IReadOnlyList<long> trlrPositions, int sampleSize)`:
  - Use reservoir sampling (Algorithm R) for uniform random selection
  - If sampleSize >= trlrPositions.Count, return all positions

Scan:
- `Task<RadarResult> ScanAsync(long dataRegionStartBlock, long dataRegionBlockCount, CancellationToken ct)`:
  1. Generate all TRLR block positions
  2. Select sample via reservoir sampling
  3. For each sampled TRLR block:
     a. Read the 4KB TRLR block
     b. Parse 255 trailer records (16B each): DataBlockTypeTag(4), GenerationNumber(4), XxHash64(8)
     c. For each record: verify DataBlockTypeTag == DATA (0x44415441)
     d. Check for suspect indicators: GenerationNumber == 0, XxHash64 == 0
     e. Optionally verify by reading the corresponding data block and recomputing XxHash64
  4. Count verified, suspect, corrupt records
  5. Estimate corruption rate with confidence interval

Statistical estimation:
- `(double EstimatedRate, double LowerBound, double UpperBound) EstimateCorruptionRate(int corrupt, int sampled, double confidenceLevel)`:
  - Use Wilson score interval for binomial proportion confidence interval
  - Works well for small proportions (corruption is typically rare)

- `RadarResult` readonly struct:
  - `int TrlrBlocksSampled, int TrailerRecordsVerified`
  - `int SuspectRecords, int ConfirmedCorrupt`
  - `double EstimatedCorruptionRate, double ConfidenceLowerBound, double ConfidenceUpperBound`
  - `bool AlertTriggered` -- true if estimated rate > AlertThreshold
  - `IReadOnlyList<long> CorruptBlockNumbers` -- confirmed corrupt data block numbers
  - `TimeSpan Duration`

Continuous monitoring:
- `Task RunContinuousAsync(CancellationToken ct)` -- periodic scan loop with alert callback
- `event Action<RadarResult>? OnAlertTriggered`
