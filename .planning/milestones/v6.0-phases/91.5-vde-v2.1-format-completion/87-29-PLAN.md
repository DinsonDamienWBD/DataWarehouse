<objective>
Implement Semantic Wear-Leveling Allocator (VOPT-41) that segregates writes into AllocationGroups by Expected_TTL hint (Hot/Warm/Cold/Frozen). Per spec: "SWLV activates ONLY when ZNS_AWARE is NOT set. On ZNS devices, epoch-based zone allocation is strictly superior. SWLV segregates writes by Expected_TTL hint. Background Vacuum reclaims entire temperature groups, eliminating write amplification on conventional NVMe."

Purpose: Conventional NVMe SSDs suffer write amplification when hot and cold data are mixed in the same erase blocks. Segregating by temperature class ensures that entire groups of same-TTL data can be reclaimed together without copying still-live data.
Output: Two files in `DataWarehouse.SDK/VirtualDiskEngine/BlockAllocation/`.
</objective>

Create `WearLevelingConfig` class with `[SdkCompatibility("6.0.0")]`:
- `bool Enabled { get; set; } = true` -- master switch
- `bool ZnsAwareActive { get; set; } = false` -- set true at mount if ZNS device detected; disables SWLV
- `TemperatureClass` enum : byte { Hot = 0, Warm = 1, Cold = 2, Frozen = 3 } -- maps to 2-bit SLC_Wear_Hint in SPOL module
- Group allocation ratios (fraction of total AGs reserved per temperature):
  - `double HotGroupRatio { get; set; } = 0.2` -- 20% of AGs for hot data
  - `double WarmGroupRatio { get; set; } = 0.3` -- 30% for warm
  - `double ColdGroupRatio { get; set; } = 0.35` -- 35% for cold
  - `double FrozenGroupRatio { get; set; } = 0.15` -- 15% for frozen/archive
- `TimeSpan VacuumGroupReclaimInterval { get; set; } = TimeSpan.FromMinutes(5)` -- how often vacuum checks temperature groups
- `double GroupReclaimThreshold { get; set; } = 0.9` -- reclaim group when >90% dead blocks

Create `SemanticWearLevelingAllocator` class implementing `IBlockAllocator` with `[SdkCompatibility("6.0.0")]`:
- Constructor: `IBlockAllocator innerAllocator, AllocationGroup[] allGroups, WearLevelingConfig config`
- ZNS gate check: if `config.ZnsAwareActive`, all methods delegate directly to innerAllocator (SWLV disabled)

Temperature group assignment:
- On construction, partition allGroups into 4 temperature bands based on config ratios
- `Dictionary<TemperatureClass, List<AllocationGroup>> _temperatureGroups`
- Round-robin within each temperature band for even distribution

Allocation with TTL hint:
- `long AllocateBlock(CancellationToken ct)`: allocate from Hot group (default, most common)
- `long AllocateBlockWithHint(TemperatureClass hint, CancellationToken ct)`:
  - Select AllocationGroup from appropriate temperature band (round-robin)
  - Allocate from that group
  - If group full, try next group in same band
  - If entire band full, fall back to any available group (with logging)
- `long[] AllocateExtent(int blockCount, CancellationToken ct)`: allocate from Hot group (default)
- `long[] AllocateExtentWithHint(int blockCount, TemperatureClass hint, CancellationToken ct)`:
  - Same temperature-aware routing as AllocateBlockWithHint

IBlockAllocator passthrough:
- `void FreeBlock(long blockNumber)`: delegates to innerAllocator
- `void FreeExtent(long startBlock, int blockCount)`: delegates to innerAllocator
- `long FreeBlockCount`: innerAllocator.FreeBlockCount
- `long TotalBlockCount`: innerAllocator.TotalBlockCount
- `double FragmentationRatio`: weighted average considering temperature segregation
- `bool IsAllocated(long blockNumber)`: innerAllocator.IsAllocated
- `Task PersistAsync(IBlockDevice device, long bitmapStartBlock, CancellationToken ct)`: delegates

Temperature group metrics:
- `WearLevelingStats GetStats()` readonly struct:
  - Per temperature class: `int GroupCount, long FreeBlocks, long TotalBlocks, double Utilization, long WriteCount`
  - `bool SwlvActive` (false if ZNS gate engaged)
  - `double EstimatedWriteAmplificationReduction` (ratio vs uniform allocation)

Group reclamation support:
- `IReadOnlyList<AllocationGroup> GetReclaimCandidates(TemperatureClass tempClass)`:
  - Returns groups in the temperature band where dead block ratio exceeds config threshold
  - Used by Background Vacuum to reclaim entire temperature groups at once
- `TemperatureClass GetGroupTemperature(int allocationGroupId)`: lookup which temperature class a group belongs to
