Create `DeletionProof` readonly struct with `[SdkCompatibility("6.0.0")]`:
- `byte[] ProofHash` (16 bytes, BLAKE3 truncated — matches ExpectedHash field size in extent pointer)
- `long BlockNumber` -- which block was erased
- `long Timestamp` -- UTC nanoseconds since epoch (secure timestamp)
- `int BlockSize` -- size of zeroed block
- Static `DeletionProof Compute(long blockNumber, int blockSize, long timestampNanos)`:
  - Create zero-filled buffer of blockSize
  - Compute BLAKE3(zeros || BitConverter.GetBytes(timestampNanos))
  - Use SHA256 as fallback with comment noting "BLAKE3 preferred; SHA256 used for compatibility"
  - Truncate hash to 16 bytes for ExpectedHash
  - Return new DeletionProof with computed hash
- Static `bool Verify(DeletionProof proof)`:
  - Recompute BLAKE3(zeros[proof.BlockSize] || BitConverter.GetBytes(proof.Timestamp))
  - Compare truncated hash to proof.ProofHash
  - Return true if match
- `byte[] Serialize()` and `static DeletionProof Deserialize(ReadOnlySpan<byte> data)`:
  - Binary format: [ProofHash:16][BlockNumber:8][Timestamp:8][BlockSize:4] = 36 bytes

Create `GdprTombstoneEngine` class with `[SdkCompatibility("6.0.0")]`:
- Constructor: `IBlockDevice device, IWriteAheadLog wal, int blockSize`

Core operation:
- `Task<DeletionProof> TombstoneExtentAsync(long startBlock, int blockCount, CancellationToken ct)`:
  1. Get secure timestamp (DateTimeOffset.UtcNow.ToUnixTimeMilliseconds() * 1_000_000 for nano precision)
  2. WAL-journal "tombstone-start" with startBlock, blockCount
  3. For each block in extent:
     a. Write blockSize zeros to device at block offset
  4. Compute deletion proof: BLAKE3(zeros || timestamp)
  5. WAL-journal "tombstone-complete" with proof hash
  6. Return DeletionProof
  7. Caller is responsible for setting extent flags TOMBSTONE (bit 18) and writing proof hash to ExpectedHash

- `Task FlattenDeltaChainAsync(long inodeNumber, ReadOnlyMemory<byte> extentData, CancellationToken ct)`:
  Per spec: "DELT delta chains MUST be flattened before tombstoning."
  1. Read all delta extents for the inode (extents with Delta_Extent flag, bit 4)
  2. Apply VCDIFF patches sequentially to reconstruct base data
  3. Write reconstructed data to new contiguous extent (or same location)
  4. Clear Delta_Extent flags
  5. Now the data is in flat form, safe to tombstone
  Note: This method reconstructs the chain so that tombstoning overwrites ALL data, not just the latest patch.

- `Task<TombstoneResult> ExecuteGdprDeletionAsync(long inodeNumber, ReadOnlyMemory<byte> inodeData, CancellationToken ct)`:
  Full GDPR deletion workflow:
  1. Check if inode has delta extents — if yes, flatten first
  2. Tombstone all data extents for the inode
  3. Collect all deletion proofs
  4. Return `TombstoneResult` with proofs array

- `TombstoneResult` readonly struct:
  - `DeletionProof[] Proofs, long InodeNumber, int ExtentsTombstoned, long BlocksErased, TimeSpan Duration`

Crash recovery:
- If "tombstone-start" found without "tombstone-complete" in WAL:
  - Re-execute the tombstone (idempotent — zeroing already-zeroed blocks is safe)
  - Compute proof and journal completion

Batch operation for Background Vacuum:
- `Task<int> ExecuteBatchTombstonesAsync(IReadOnlyList<long> inodeNumbers, CancellationToken ct)`:
  - Process multiple inodes sequentially
  - Returns count of successfully tombstoned inodes
