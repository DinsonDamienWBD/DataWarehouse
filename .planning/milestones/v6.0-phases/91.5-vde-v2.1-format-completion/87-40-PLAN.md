Create `ForensicNecromancyRecovery` class with `[SdkCompatibility("6.0.0")]`:
- Constructor: `IBlockDevice device, ForensicRecoveryConfig config`

Phase 1 - TRLR stride scan:
- `Task<TrlrScanResult> ScanForTrailersAsync(CancellationToken ct)`:
  1. Scan entire device at stride 256 blocks (candidate TRLR positions)
  2. For each candidate position, read 4KB block
  3. Check if block's own trailer (last 16B) has BlockTypeTag == TRLR (0x54524C52)
  4. If valid TRLR block: parse 255 trailer records
  5. Each record: DataBlockTypeTag(4B), GenerationNumber(4B), XxHash64(8B)
  6. Record location of every valid TRLR block found
- `TrlrScanResult`: `int TrlrBlocksFound, long TotalTrailerRecords, IReadOnlyList<TrlrBlockInfo> ValidBlocks`
- `TrlrBlockInfo` readonly struct: `long BlockNumber, int ValidRecordCount, uint MinGeneration, uint MaxGeneration`

Phase 2 - Data block verification:
- `Task<RecoveryResult> RecoverDataBlocksAsync(TrlrScanResult scanResult, CancellationToken ct)`:
  1. For each valid TRLR block:
     a. The 255 data blocks preceding this TRLR block are the corresponding data blocks
     b. For each data block: read it, compute XxHash64, compare with TRLR record
     c. Mark as: Verified (checksum matches), Corrupt (checksum mismatch), or Suspect (zero gen/hash)
  2. Collect all verified data blocks with their metadata (generation, checksum)
- `RecoveredBlock` readonly struct: `long PhysicalBlockNumber, long LogicalDataIndex, uint GenerationNumber, ulong Checksum, RecoveryStatus Status`
- `enum RecoveryStatus { Verified, Corrupt, Suspect, TrlrMissing }`

Phase 3 - Metadata block recovery:
- `Task<MetadataRecoveryResult> RecoverMetadataBlocksAsync(CancellationToken ct)`:
  1. Scan all blocks (not just stride 256) looking for known BlockTypeTag values in trailer position
  2. Match tags: SUPB, RMAP, INOD, BTRE, SNAP, etc. from BlockTypeTags
  3. If tag matches a known type: validate trailer checksum
  4. Report recovered metadata blocks by type

Full recovery:
- `Task<ForensicReport> RunFullRecoveryAsync(CancellationToken ct)`:
  1. Phase 1: TRLR stride scan
  2. Phase 2: Data block verification
  3. Phase 3: Metadata block recovery (if enabled)
  4. Generate report

- `ForensicReport` readonly struct:
  - `long TotalBlocksScanned, long DataBlocksRecovered, long DataBlocksCorrupt`
  - `long MetadataBlocksRecovered`
  - `Dictionary<uint, int> MetadataBlocksByType` -- recovered metadata blocks by BlockTypeTag
  - `IReadOnlyList<RecoveredBlock> RecoveredBlocks`
  - `double RecoveryRate` -- proportion of data blocks recovered
  - `TimeSpan Duration`

Export (optional):
- `Task ExportRecoveredBlocksAsync(ForensicReport report, string outputPath, CancellationToken ct)`:
  - Write verified data blocks to files in outputPath, named by logical index
