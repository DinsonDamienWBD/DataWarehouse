2. Create `SpdkDmaAllocator.cs` with `[SdkCompatibility("6.0.0")]`:
   - `sealed class SpdkDmaAllocator : IDisposable`
   - Constructor takes `int blockSize, int poolSize` (number of pre-allocated buffers)
   - On construction, pre-allocates `poolSize` DMA buffers via `spdk_dma_zmalloc(blockSize, blockSize, IntPtr.Zero)`
   - Stores in a `ConcurrentStack<IntPtr>` for lock-free rent/return
   - `IntPtr Rent()` — pops a buffer from pool; if empty, allocates a new one (growth)
   - `void Return(IntPtr buffer)` — pushes back to pool
   - `Dispose()` — frees all buffers via `spdk_dma_free`, clears pool
   - Track total allocated count for diagnostics: `int TotalAllocated`, `int PoolAvailable`

3. Create `SpdkBlockDevice.cs` with `[SdkCompatibility("6.0.0")]`:
   - `sealed class SpdkBlockDevice : IBlockDevice`
   - Purpose: Preamble boot-specific SPDK device. Used when DW Kernel boots from preamble and takes NVMe ownership via vfio-pci.
   - Constructor: `(string pciAddress, int blockSize, long blockCount, SpdkDmaAllocator allocator)`
   - Stores NVMe controller handle (`IntPtr _controller`), namespace handle (`IntPtr _namespace`), qpair handle (`IntPtr _qpair`)
   - `InitializeAsync()` — calls spdk_env_init, spdk_nvme_probe for the specified PCI address, allocates I/O queue pair
   - `ReadBlockAsync` implementation:
     a. Rent DMA buffer from allocator
     b. Submit spdk_nvme_ns_cmd_read with LBA = blockNumber
     c. Poll completions via spdk_nvme_qpair_process_completions in a spin loop with Task.Yield every N iterations
     d. Copy from DMA buffer to user Memory<byte>
     e. Return DMA buffer to allocator
   - `WriteBlockAsync` implementation: similar pattern (copy user data to DMA buffer, submit write, poll)
   - `FlushAsync` — issue NVMe flush command
   - `DisposeAsync` — free qpair, detach controller, dispose allocator
   - `BlockSize` and `BlockCount` properties from constructor
   - XML doc clearly states: "This device is for preamble bare-metal boot scenarios where SPDK takes direct NVMe ownership via vfio-pci. For hosted environments, see the general SPDK device in PhysicalDevice."
