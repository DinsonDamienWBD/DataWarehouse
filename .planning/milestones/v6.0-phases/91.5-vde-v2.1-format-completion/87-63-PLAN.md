<objective>
Implement the Linux FUSE3 low-level mount provider (VOPT-85). Fuse3LowLevelMountProvider uses P/Invoke to libfuse3's low-level inode-based API (fuse_lowlevel_ops), providing zero-copy and splice support.

Purpose: Per AD-51, Linux users must be able to mount VDE files as native directories. The low-level FUSE API is chosen over high-level because it provides inode-level control matching VDE's native inode model -- no path-to-inode translation overhead. Linux kernel 6.9+ FUSE passthrough mode enables near-native performance for unencrypted data.

**Fuse3MountHandle** class with `[SdkCompatibility("6.0.0")]`, `[SupportedOSPlatform("linux")]`:
- Implements `IMountHandle`
- Properties: `MountPoint`, `VdePath`, `IsActive`, `MountedAtUtc`, `Options`

**Fuse3LowLevelMountProvider** class with `[SdkCompatibility("6.0.0")]`, `[SupportedOSPlatform("linux")]`:
- Implements `IVdeMountProvider`
- `SupportedPlatforms => PlatformFlags.Linux`
- `IsAvailable => Fuse3Native.IsAvailable() && RuntimeInformation.IsOSPlatform(OSPlatform.Linux)`
- Track active mounts in `ConcurrentDictionary<string, Fuse3MountHandle>`

**MountAsync implementation:**
1. Validate `mountPoint` is an existing empty directory
2. Create VdeFilesystemAdapter
3. Build `fuse_args` from MountOptions:
   - `-o allow_other` if `options.AllowOther`
   - `-o auto_unmount` if `options.AutoUnmount`
   - `-o writeback_cache` if `options.EnableWriteback`
   - `-o attr_timeout={options.AttrTimeout.TotalSeconds}`
   - `-o entry_timeout={options.EntryTimeout.TotalSeconds}`
   - `-o max_idle_threads={options.MaxConcurrentOps}`
   - Any `PlatformOptions` entries as `-o key=value`
4. Allocate GCHandle for the adapter (passed as userdata to fuse_session_new)
5. Build `fuse_lowlevel_ops` struct, populating callback function pointers:
   - **lookup**: `adapter.LookupAsync(parent, name)` -> `fuse_reply_entry` with entry_param (ino, attr, timeouts) or `fuse_reply_err(ENOENT)`
   - **getattr**: `adapter.GetattrAsync(ino)` -> fill stat struct -> `fuse_reply_attr`
   - **setattr**: `adapter.SetattrAsync(ino, attrs, mask)` -> `fuse_reply_attr`
   - **readlink**: `adapter.ReadlinkAsync(ino)` -> `fuse_reply_readlink`
   - **mkdir**: `adapter.MkdirAsync(parent, name, perms)` -> `fuse_reply_entry`
   - **unlink**: `adapter.UnlinkAsync(parent, name)` -> `fuse_reply_err(0)`
   - **rmdir**: `adapter.RmdirAsync(parent, name)` -> `fuse_reply_err(0)`
   - **symlink**: `adapter.SymlinkAsync(parent, name, target)` -> `fuse_reply_entry`
   - **rename**: `adapter.RenameAsync(oldParent, oldName, newParent, newName)` -> `fuse_reply_err(0)`
   - **open**: `adapter.GetattrAsync(ino)` (validate exists) -> store ino in fi.fh -> `fuse_reply_open`
   - **read**: `adapter.ReadAsync(ino, buffer, offset)` -> `fuse_reply_buf`
   - **write**: `adapter.WriteAsync(ino, data, offset)` -> `fuse_reply_write`
   - **flush**: `adapter.FlushAsync(ino)` -> `fuse_reply_err(0)`
   - **release**: no-op -> `fuse_reply_err(0)` (adapter cleanup happens at unlink/close)
   - **opendir**: validate is directory -> store ino in fi.fh -> `fuse_reply_open`
   - **readdir**: `adapter.ReaddirAsync(ino)` -> `fuse_add_direntry_plus` for each entry -> `fuse_reply_buf`
   - **releasedir**: no-op -> `fuse_reply_err(0)`
   - **statfs**: `adapter.StatfsAsync()` -> fill statvfs -> `fuse_reply_statfs`
   - **create**: `adapter.CreateAsync(parent, name, File, perms)` -> `fuse_reply_create` (entry + fi)
   - **setxattr**: `adapter.SetXattrAsync(ino, name, value)` -> `fuse_reply_err(0)`
   - **getxattr**: `adapter.GetXattrAsync(ino, name)` -> `fuse_reply_buf` or `fuse_reply_xattr`
   - **listxattr**: `adapter.ListXattrAsync(ino)` -> `fuse_reply_buf` or `fuse_reply_xattr`
   - **forget**: no-op (VDE manages inode lifecycle via refcounting in IInodeTable)
6. Call `fuse_session_new` with ops and userdata
7. Call `fuse_session_mount(session, mountPoint)`
8. Start `fuse_session_loop_mt` on a background thread (use `MaxConcurrentOps` for thread config)
9. Return `Fuse3MountHandle`

**Callback helper pattern:**
Each callback is a static method with `[UnmanagedCallersOnly]` attribute (or delegates pinned with GCHandle). Retrieve adapter from userdata via `fuse_req_userdata`. Wrap async adapter calls with `.GetAwaiter().GetResult()` in the callback (FUSE callbacks are synchronous from libfuse's perspective -- the multi-threaded loop provides concurrency).

**InodeAttributes -> stat mapping:**
- `st_ino` = InodeNumber
- `st_mode` = InodeType (S_IFREG=0x8000, S_IFDIR=0x4000, S_IFLNK=0xA000) | permissions
- `st_nlink` = LinkCount
- `st_size` = Size
- `st_blocks` = AllocatedSize / 512
- `st_blksize` = BlockSize
- Timestamps: ticks -> timespec conversion (ticks / TimeSpan.TicksPerSecond for tv_sec, remainder * 100 for tv_nsec)

**Error mapping:**
- FileNotFoundException -> ENOENT
- DirectoryNotEmptyException -> ENOTEMPTY
- UnauthorizedAccessException -> EACCES
- IOException (disk full) -> ENOSPC
- InvalidOperationException (read-only) -> EROFS
- Default -> EIO (5)

**UnmountAsync:**
- `fuse_session_exit`, join loop thread, `fuse_session_unmount`, `fuse_session_destroy`
- Free GCHandle
- adapter.SyncAsync, set IsActive = false

**ListMountsAsync:** return MountInfo from active handles dictionary.
