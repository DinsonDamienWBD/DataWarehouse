Create `Metaslab` class with `[SdkCompatibility("6.0.0")]`:
- Constructor: `int metaslabId, long startBlock, long blockCount, IBlockDevice device, long bitmapBlockOffset, int blockSize`
- Lazy bitmap: `byte[]? _bitmap` is null until first allocation. `EnsureBitmapLoadedAsync(CancellationToken ct)` loads from device on demand.
- `bool IsLoaded => _bitmap != null`
- `long FreeBlockCount { get; }` (maintained without loading bitmap if tracked in directory entry)
- `double Utilization => 1.0 - (double)FreeBlockCount / BlockCount`
- Size-class segregation with three free lists:
  - Small: extents 1-16 blocks (4KB-64KB)
  - Medium: extents 17-256 blocks (64KB-1MB)
  - Large: extents 257+ blocks (1MB+)
  - Each free list: sorted by start block for coalescing, or by size for best-fit
- `long AllocateBlock(CancellationToken ct)`: load bitmap if needed, find first free bit, mark allocated
- `long[] AllocateExtent(int blockCount, CancellationToken ct)`: pick appropriate size class, find contiguous run, mark allocated
- `void FreeBlock(long blockNumber)`: mark free, merge with adjacent free entries in size class list
- `void FreeExtent(long startBlock, int blockCount)`: mark free, coalesce
- `Task PersistAsync(CancellationToken ct)`: write dirty bitmap back to device
- `void Unload()`: release bitmap memory (for memory pressure eviction)
- `MetaslabState State`: enum { Unloaded, Active, Full, Fragmented, EvictionCandidate }

Create `MetaslabDirectory` class with `[SdkCompatibility("6.0.0")]`:
- Per spec: "32 bytes per entry — 128 MB for 4M metaslabs" at exabyte scale
- `MetaslabDirectoryEntry` readonly struct (32 bytes): `int MetaslabId, long StartBlock, long BlockCount, long FreeBlockCount, MetaslabState State, int ZoneId, int RegionId`
- Constructor: `IBlockDevice device, long directoryStartBlock, int blockSize, long totalVdeBlocks`
- `int MetaslabCount { get; }`
- Zone/Region/AG hierarchy:
  - Zone: top-level partition (16 zones for first 16 PB, expandable)
  - Region: mid-level within zone (256 regions per zone)
  - AG: existing AllocationGroup within region
  - Metaslab: smallest unit (~256MB default, tunable)
- `Metaslab GetBestMetaslab(int requestedBlocks)`: O(1) lookup from directory — pick least-utilized metaslab in preferred zone with sufficient free space in the right size class
- `Metaslab GetMetaslabForBlock(long blockNumber)`: O(1) by dividing block number by metaslab size
- `Task LoadDirectoryAsync(CancellationToken ct)`: read directory entries from device
- `Task PersistDirectoryAsync(CancellationToken ct)`: write directory back
- Memory management: LRU eviction of loaded metaslabs when count exceeds `MaxLoadedMetaslabs` (default 64)

Create `MetaslabAllocator` class implementing `IBlockAllocator` with `[SdkCompatibility("6.0.0")]`:
- Constructor: `IBlockDevice device, long bitmapStartBlock, long totalBlocks, int blockSize`

Morph level detection (per spec):
- `MorphLevel` enum: `Level0_FlatBitmap, Level1_ShardedGroups, Level2_MetaslabTree, Level3_HierarchicalMetaslab`
- Auto-detect at construction: totalBlocks * blockSize gives volume size
  - < 128 MB: Level 0 — delegate to existing BitmapAllocator
  - 128 MB - 1 TB: Level 1 — delegate to AllocationGroupDescriptorTable (existing)
  - 1 TB - 1 PB: Level 2 — use MetaslabDirectory with flat metaslabs
  - > 1 PB: Level 3 — use MetaslabDirectory with zone/region hierarchy
- `MorphLevel CurrentLevel { get; }`

IBlockAllocator implementation:
- `long AllocateBlock(CancellationToken ct)`: dispatch to current morph level
  - Level 0-1: delegate to existing allocator
  - Level 2-3: `directory.GetBestMetaslab(1).AllocateBlock(ct)`
- `long[] AllocateExtent(int blockCount, CancellationToken ct)`: dispatch to current level
  - Level 2-3: pick size-class-appropriate metaslab, allocate within it
- `void FreeBlock(long blockNumber)`: dispatch — Level 2-3 routes to `directory.GetMetaslabForBlock(blockNumber)`
- `void FreeExtent(long startBlock, int blockCount)`: dispatch
- `long FreeBlockCount`: sum across all metaslabs (cached in directory entries)
- `long TotalBlockCount`: total VDE blocks
- `double FragmentationRatio`: weighted average across loaded metaslabs
- `bool IsAllocated(long blockNumber)`: dispatch to appropriate metaslab
- `Task PersistAsync(IBlockDevice device, long bitmapStartBlock, CancellationToken ct)`: persist all dirty metaslabs + directory

Online morph promotion (per spec):
- `Task<bool> TryPromoteAsync(CancellationToken ct)`: when VDE resized past tier boundary, promote non-blocking
  - Level 0 -> 1: shard existing bitmap (background)
  - Level 1 -> 2: build metaslab directory lazily
  - Level 2 -> 3: partition into zones
- Promotion is safe: existing allocations remain valid, only indexing layer changes
  </action>
