**MacFuseNative** internal static class with `[SdkCompatibility("6.0.0")]`, `[SupportedOSPlatform("osx")]`:

macFUSE 4.x ships two options: the traditional kext (`osxfuse`) and the newer `fuse-t` (user-space). Both expose a libfuse3-compatible API. The P/Invoke layer must handle both.

**Library loading:**
- Primary: `NativeLibrary.TryLoad("libfuse-t.dylib", ...)` (macFUSE fuse-t, user-space, no kext needed)
- Fallback: `NativeLibrary.TryLoad("/usr/local/lib/libfuse.dylib", ...)` (traditional macFUSE kext path)
- `IsAvailable()`: returns true if either library loads on macOS

**Struct definitions:** Reuse the same struct shapes as Fuse3Native (fuse_lowlevel_ops, fuse_entry_param, fuse_file_info, statvfs) but with macOS-specific `stat` struct layout:

macOS `stat` struct (differs from Linux -- 144 bytes on x86_64, 144 bytes on arm64):
- `dev_t st_dev` (int32), `mode_t st_mode` (uint16), `nlink_t st_nlink` (uint16)
- `ino_t st_ino` (uint64), `uid_t st_uid` (uint32), `gid_t st_gid` (uint32)
- `dev_t st_rdev` (int32)
- `timespec st_atimespec, st_mtimespec, st_ctimespec, st_birthtimespec`
- `off_t st_size` (int64), `blkcnt_t st_blocks` (int64), `blksize_t st_blksize` (int32)
- `uint32_t st_flags, st_gen`

**P/Invoke declarations** -- same function signatures as Fuse3Native but loading from the macFUSE library:
- `fuse_session_new`, `fuse_session_mount`, `fuse_session_loop_mt`, `fuse_session_unmount`, `fuse_session_destroy`, `fuse_session_exit`
- `fuse_reply_entry`, `fuse_reply_attr`, `fuse_reply_readlink`, `fuse_reply_open`, `fuse_reply_buf`, `fuse_reply_write`, `fuse_reply_err`, `fuse_reply_none`, `fuse_reply_statfs`, `fuse_reply_xattr`
- `fuse_req_userdata`, `fuse_add_direntry_plus`

**errno constants** (macOS values differ from Linux for some):
- `ENOENT = 2, EACCES = 13, EEXIST = 17, ENOTDIR = 20, EISDIR = 21, EINVAL = 22, ENOSPC = 28, EROFS = 30, ENOTEMPTY = 66` (note: ENOTEMPTY is 66 on macOS, not 39), `ENOTSUP = 45, ENOATTR = 93`

**MacFuseMountProvider** class with `[SdkCompatibility("6.0.0")]`, `[SupportedOSPlatform("osx")]`:
- Implements `IVdeMountProvider`
- `SupportedPlatforms => PlatformFlags.MacOS`
- `IsAvailable => MacFuseNative.IsAvailable() && RuntimeInformation.IsOSPlatform(OSPlatform.OSX)`
- Track active mounts in `ConcurrentDictionary<string, MacFuseMountHandle>`

**MountAsync implementation:**
1. Validate `mountPoint` is an existing empty directory
2. Create VdeFilesystemAdapter
3. Build `fuse_args` from MountOptions:
   - `-o allow_other` if `options.AllowOther`
   - `-o auto_unmount` if `options.AutoUnmount`
   - `-o attr_timeout={options.AttrTimeout.TotalSeconds}`
   - `-o entry_timeout={options.EntryTimeout.TotalSeconds}`
   - macOS-specific: `-o volname=DWVD` (volume name shown in Finder)
   - macOS-specific: `-o local` (show as local volume, not network)
   - macOS-specific: `-o noappledouble` (suppress ._ resource fork files)
   - macOS-specific: `-o noapplexattr` (suppress com.apple.* xattrs from VDE)
   - Any `PlatformOptions` entries
4. Build `fuse_lowlevel_ops` struct -- same callback pattern as Fuse3LowLevelMountProvider:
   - All callbacks delegate to VdeFilesystemAdapter (lookup, getattr, setattr, readlink, mkdir, unlink, rmdir, symlink, rename, open, read, write, flush, release, opendir, readdir, releasedir, statfs, create, setxattr, getxattr, listxattr, forget)
   - Use macOS stat struct layout for `fuse_reply_attr`
   - **Spotlight xattr integration**: when getxattr is called for:
     - `com.apple.metadata:kMDItemFSName` -> return file name
     - `com.apple.metadata:kMDItemFSSize` -> return file size
     - `com.apple.metadata:kMDItemContentCreationDate` -> return CreatedUtc
     - `com.apple.metadata:kMDItemContentModificationDate` -> return ModifiedUtc
     - `com.apple.metadata:kMDItemContentType` -> return UTI based on file extension
     - `com.apple.metadata:kMDItemContentTypeTree` -> return UTI hierarchy
     - Other `com.apple.metadata:*` keys -> delegate to adapter.GetXattrAsync
     - Non-apple xattrs -> delegate to adapter.GetXattrAsync normally
5. Call `fuse_session_new`, `fuse_session_mount`
6. Start `fuse_session_loop_mt` on background thread
7. Return `MacFuseMountHandle`

**Spotlight metadata helper:**
Create internal static class `SpotlightMetadataHelper` within the file:
- `byte[]? GetSpotlightXattr(string key, InodeAttributes attrs, string? fileName)`:
  - Serializes Spotlight metadata values as binary plist format (simple NSData/NSString/NSDate encoding)
  - For dates: Core Data absolute reference date (2001-01-01) offset encoding
  - For strings: UTF-8 encoded NSString plist
  - Returns null if key is not a recognized Spotlight metadata key

**Error mapping:** Same as Fuse3LowLevelMountProvider but with macOS errno values (ENOTEMPTY=66, ENOATTR=93 instead of ENODATA=61).

**UnmountAsync/ListMountsAsync:** Same pattern as Fuse3LowLevelMountProvider.

**macOS Finder integration notes (in XML doc comments):**
- The `local` mount option ensures the volume appears in Finder sidebar under "Locations"
- `noappledouble` prevents Finder from creating `._` resource fork files inside the VDE
- Spotlight indexes mounted VDE content via the xattr metadata provider -- no separate mdimporter plugin needed for basic metadata
