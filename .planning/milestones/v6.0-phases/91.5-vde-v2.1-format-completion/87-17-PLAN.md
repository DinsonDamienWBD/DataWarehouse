Create `MerkleBatchConfig` class with `[SdkCompatibility("6.0.0")]`:
- `TimeSpan BatchInterval { get; set; } = TimeSpan.FromSeconds(5)` -- epoch-batched update interval per spec
- `int MaxDirtyBlocksPerBatch { get; set; } = 8192` -- cap per batch to bound CPU time
- `bool WalRecoveryEnabled { get; set; } = true` -- rebuild dirty set from WAL after crash
- `int DirtyQueueCapacity { get; set; } = 65536` -- max pending dirty entries before back-pressure

Create `EpochBatchedMerkleUpdater` class with `[SdkCompatibility("6.0.0")]`, implements `IAsyncDisposable`:
- Constructor: `MerkleIntegrityVerifier verifier, IWriteAheadLog wal, MerkleBatchConfig config`
- Dirty queue: `ConcurrentQueue<DirtyBlockEntry>` where `DirtyBlockEntry` is a readonly struct: `long BlockNumber, byte[] BlockHash` (BLAKE3 truncated to 32 bytes)
- `void MarkDirty(long blockNumber, ReadOnlySpan<byte> blockData)`:
  - Compute BLAKE3 hash of blockData (use System.Security.Cryptography if BLAKE3 unavailable, fall back to SHA256 with a comment noting BLAKE3 preferred)
  - Enqueue `DirtyBlockEntry` to ConcurrentQueue
  - This is the ONLY method called on the write hot path -- no locks, no tree traversal
  - If queue exceeds DirtyQueueCapacity, log warning (back-pressure signal)
- `Task RunAsync(CancellationToken ct)`:
  - Background loop: sleep BatchInterval, then drain up to MaxDirtyBlocksPerBatch entries from queue
  - For each batch: call `verifier.UpdateLeafAsync(blockNumber, hash)` for each dirty entry (or equivalent update method)
  - After processing batch: recompute Merkle root via verifier
  - Track metrics: `long TotalBatchesProcessed, long TotalBlocksUpdated, TimeSpan LastBatchDuration`
  - Respects cancellation
- `Task RecoverFromWalAsync(IReadOnlyList<JournalEntry> walEntries, IBlockDevice device, int blockSize, CancellationToken ct)`:
  - Scan WAL entries for BlockWrite operations since last Merkle checkpoint
  - For each BlockWrite: read the current block data from device, compute hash, add to dirty queue
  - Process one full batch to bring Merkle tree up to date
  - This runs at mount time before the background loop starts
- `MerkleBatchStats GetStats()`: returns readonly struct with `TotalBatchesProcessed, TotalBlocksUpdated, int PendingDirtyBlocks, TimeSpan LastBatchDuration, DateTimeOffset LastBatchTime`
- `Task FlushAsync(CancellationToken ct)`: force-process all pending dirty entries immediately (used at unmount)
