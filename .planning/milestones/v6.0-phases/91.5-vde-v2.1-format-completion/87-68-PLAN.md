**WRITE STAGES (IVdeWriteStage):**

1. `InodeResolverStage` -- Write Stage 1 (always runs, ModuleGate = null):
   - Constructor: `IBlockAllocator allocator` (from BlockAllocation namespace)
   - ExecuteAsync: Allocate or lookup inode from allocator. Set `context.Inode` to new ExtendedInode512. Set `context.Inode.Flags` based on active modules in manifest. Write CreatedEpoch, ModifiedEpoch, Size. Copy manifest to inode's ModuleManifest field.
   - StageName = "InodeResolver"

2. `TagIndexerStage` -- Write Stage 3 (ModuleGate = ModuleId.Tags):
   - Constructor: no external deps required (tag data comes from context.Properties)
   - ExecuteAsync: Read tag data from `context.Properties["InlineTags"]` if present. Write to inode's InlineTagArea (176 bytes at offset 320 per spec). Queue RoaringBitmap tag index update via `context.SetProperty("TagIndexUpdate", tagUpdate)`.
   - StageName = "TagIndexer"

3. `DataTransformerStage` -- Write Stage 4 (always runs, ModuleGate = null):
   - Constructor: no external deps (transformation implementations are pluggable via context.Properties)
   - ExecuteAsync: Check `context.IsCompressed` -- if true, compress `context.DataBuffer` using algorithm from `context.Properties["CompressionAlgorithm"]`. Check `context.IsEncrypted` -- if true, encrypt using IV from `context.Properties["EncryptionIV"]`. Check `context.IsDelta` -- if true, compute delta. Update `context.DataBuffer` with transformed data. All actual crypto/compression delegates are injected via Properties to avoid hard SDK dependencies.
   - StageName = "DataTransformer"

4. `ExtentAllocatorStage` -- Write Stage 5 (always runs, ModuleGate = null):
   - Constructor: `IBlockAllocator allocator`
   - ExecuteAsync: Calculate blocks needed from `context.DataBuffer.Length / context.BlockSize`. Request blocks from allocator. Build `InodeExtent` entries. Add to `context.Extents`. Set extent flags (IS_POISON, SPATIOTEMPORAL, DeadlineTier, Compression_Class) from context.Properties. If RAID module active, set `context.SetProperty("RaidParityRequired", true)` for downstream.
   - StageName = "ExtentAllocator"

5. `IntegrityCalculatorStage` -- Write Stage 6 (ModuleGate = ModuleId.Integrity):
   - Constructor: none
   - ExecuteAsync: For each extent in `context.Extents`, compute BLAKE3 hash of the corresponding data slice from `context.DataBuffer`. Store as `context.SetProperty("ExtentHashes", hashes)`. Set TRLR Generation = current epoch. Queue Merkle tree update via property bag.
   - StageName = "IntegrityCalculator"

6. `WalBlockWriterStage` -- Write Stage 7 (always runs, ModuleGate = null):
   - Constructor: `IBlockDevice device, IWriteAheadLog wal`
   - ExecuteAsync: Journal metadata to WAL via `wal`. Write data blocks to device via `device.WriteBlockAsync` for each extent. Write TRLR entries (DataBlockTypeTag + GenerationNumber + XxHash64). Post-write: set `context.SetProperty("WriteComplete", true)`.
   - StageName = "WalBlockWriter"

**READ STAGES (IVdeReadStage):**

7. `InodeLookupStage` -- Read Stage 1 (always runs, ModuleGate = null):
   - Constructor: `IBlockDevice device`
   - ExecuteAsync: Check ARC cache (via `context.TryGetProperty("ArcCache", ...)`) for inode. On miss, read from device via B-tree traversal. Set `context.Inode`. Read ModuleManifest from inode and set `context.Manifest`. Extract inode extension bytes and store as `context.SetProperty("InodeExtensionBytes", extensionBytes)` for ModuleExtractorStage. Set I/O priority from QoS module field if active.
   - StageName = "InodeLookup"

8. `AccessControlStage` -- Read Stage 2 (ModuleGate = ModuleId.Security):
   - Constructor: none
   - ExecuteAsync: Check IS_POISON flag on inode -- if set, set `context.SetProperty("PoisonAlert", true)` and throw. Check EKEY quarantine. Check WLCK epoch restriction. Check MACR macaroon caveats. Check Policy Vault ACL. All checks read from `context.Inode.Flags` and `context.ModuleFields`.
   - StageName = "AccessControl"

9. `ExtentResolverStage` -- Read Stage 3 (always runs, ModuleGate = null):
   - Constructor: `IBlockDevice device`
   - ExecuteAsync: Walk extent tree from `context.Inode`. Build list of `InodeExtent` entries. Handle DELTA chain resolution. Handle SHARED_COW -> physical block resolution. Decode Dict_ID and I/O deadline hints from extent Flags. Store all in `context.Extents`.
   - StageName = "ExtentResolver"

10. `BlockReaderIntegrityStage` -- Read Stage 4 (always runs, ModuleGate = null):
    - Constructor: `IBlockDevice device`
    - ExecuteAsync: For each extent in `context.Extents`, read blocks via `device.ReadBlockAsync`. Verify TRLR XxHash64. If Integrity module active, verify BLAKE3 ExpectedHash. On mismatch: attempt RAID reconstruction if RAID module active, else throw IntegrityException. Verify Generation monotonicity. Assemble `context.DataBuffer` from all read blocks.
    - StageName = "BlockReaderIntegrity"

11. `PostReadUpdatesStage` -- Read Stage 6 (always runs, ModuleGate = null):
    - Constructor: none
    - ExecuteAsync: Set ARC cache promotion hint via `context.SetProperty("ArcPromote", true)`. Increment read counters via `context.SetProperty("HealthSummaryUpdate", update)`. Update heat map entry via property bag.
    - StageName = "PostReadUpdates"

- Where external dependencies don't exist yet (e.g., IBlockAllocator for InodeResolver), accept the interface in the constructor. The interface already exists in `DataWarehouse.SDK.VirtualDiskEngine.BlockAllocation.IBlockAllocator`.

This is NOT a stub -- the stage runs, processes data, produces output. The hash algorithm is the only deferred detail.
