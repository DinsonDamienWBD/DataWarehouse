Create `FeatureActivationConfig` class with `[SdkCompatibility("6.0.0")]` in ModuleManagement namespace:
- `FeatureRolloutPolicy` record:
  - `int ModuleBitPosition` -- which ActiveModules bit (0-63) this policy controls
  - `double RolloutPercentage` -- 0.0 to 1.0, fraction of new inodes that get this module
  - `string ExperimentName` -- human-readable A/B test name
  - `bool Enabled { get; set; } = true`
  - `DateTimeOffset? StartTime` -- when rollout began (null = not started)
  - `DateTimeOffset? EndTime` -- when to stop rollout (null = indefinite)
- `List<FeatureRolloutPolicy> Policies { get; set; } = new()`
- `int HashSeed { get; set; } = 42` -- seed for deterministic group assignment

Create `ProgressiveFeatureActivation` class with `[SdkCompatibility("6.0.0")]`:
- Constructor: `FeatureActivationConfig config`

Group assignment (deterministic, reproducible):
- `int GetExperimentGroup(long inodeNumber, string experimentName)`:
  - Hash: combine inodeNumber + experimentName + HashSeed via XxHash64
  - Return (int)(hash % 100) to get group 0-99

Module activation decision:
- `ulong ComputeActiveModulesForNewInode(long inodeNumber, ulong volumeModuleManifest)`:
  1. Start with base modules (volume defaults or zero)
  2. For each enabled policy:
     a. Check volume ModuleManifest has the module bit set (module available on volume)
     b. Compute experiment group for this inode + experiment
     c. If group < (RolloutPercentage * 100): set the module bit
     d. Else: leave bit unset
  3. Return final ActiveModules bitmap

- `bool ShouldActivateModule(long inodeNumber, FeatureRolloutPolicy policy)`:
  - Shorthand: returns true if inode falls within rollout percentage

Monitoring:
- `Task<RolloutStats> GetRolloutStatsAsync(int moduleBitPosition, IEnumerable<(long InodeNumber, ulong ActiveModules)> inodeSample)`:
  1. Count inodes with module bit set vs total
  2. Return actual activation rate
- `RolloutStats` readonly struct:
  - `string ExperimentName, int ModuleBitPosition`
  - `double TargetPercentage, double ActualPercentage`
  - `long ActivatedCount, long TotalCount`

Rollout control:
- `void SetRolloutPercentage(string experimentName, double newPercentage)`:
  - Updates policy (affects only NEW inodes, not existing)
- `void PauseRollout(string experimentName)` -- set Enabled = false
- `void CompleteRollout(string experimentName)` -- set to 100%, mark EndTime

Rollback:
- `IReadOnlyList<long> IdentifyRollbackCandidates(string experimentName, IEnumerable<(long InodeNumber, ulong ActiveModules)> inodes)`:
  - Returns inode numbers that have the experiment's module bit set (for manual rollback)
