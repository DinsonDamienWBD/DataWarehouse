**IMemoryAllocator** interface:
- `IMemoryOwner<byte> Allocate(int byteCount)` -- allocate buffer of at least byteCount
- `int Alignment { get; }` -- alignment guarantee of allocated buffers (e.g., 4096)
- Extends `IDisposable` for cleanup of pooled resources

**AlignedMemoryAllocator** sealed class implementing `IMemoryAllocator`:
- Constructor: `int alignment = 4096`
- `Alignment` property returns configured alignment
- `Allocate(int byteCount)`: uses `NativeMemory.AlignedAlloc((nuint)byteCount, (nuint)alignment)` and returns a new `AlignedMemoryOwner` (from 87-56). If AlignedMemoryOwner is in the same namespace, reference it directly. If its constructor takes `(int byteCount, int alignment)`, pass through.
- `Dispose()`: no-op (each IMemoryOwner disposes independently)
- Thread-safe (stateless allocator)

**AlignedMemoryPool** sealed class implementing `IMemoryAllocator`:
- Constructor: `int bufferSize, int alignment = 4096, int maxPooled = 256`
- Maintains a `ConcurrentBag<AlignedMemoryOwner>` of pre-allocated buffers
- `Allocate(int byteCount)`:
  - If `byteCount != bufferSize`, throw `ArgumentException` ("Pool only supports fixed-size buffers of {bufferSize} bytes")
  - Try dequeue from pool. If available, return wrapped in `PooledBufferOwner` (private nested class)
  - If pool empty, allocate new via `NativeMemory.AlignedAlloc`
  - `PooledBufferOwner` implements `IMemoryOwner<byte>`. On `Dispose()`, returns buffer to pool instead of freeing (if pool count < maxPooled; otherwise frees)
- `Dispose()`: frees all pooled buffers via `NativeMemory.AlignedFree`
- Thread-safe via `ConcurrentBag`
- `int PooledCount { get; }` -- current number of pooled buffers (diagnostic)
- `int TotalAllocated { get; }` -- total buffers created (Interlocked counter)

**RawPartitionNativeMethods** internal static class:
- Windows P/Invoke (kernel32.dll):
  - `CreateFileW` for `\\.\PhysicalDriveN` or `\\.\HarddiskVolumeN` paths
  - `DeviceIoControl` with `IOCTL_DISK_GET_DRIVE_GEOMETRY_EX` (0x000700A0) to get sector size and disk size
  - `DISK_GEOMETRY_EX` struct: Cylinders (long), MediaType (int), TracksPerCylinder (uint), SectorsPerTrack (uint), BytesPerSector (uint), DiskSize (long)
- Linux P/Invoke (libc):
  - `open` for `/dev/nvme0n1`, `/dev/sda`, etc. with `O_RDWR | O_DIRECT`
  - `ioctl` with `BLKSSZGET` (0x1268) to get sector size
  - `ioctl` with `BLKGETSIZE64` (0x80081272) to get device size in bytes
- macOS P/Invoke (libc):
  - `open` for `/dev/rdiskN` with `O_RDWR`
  - `ioctl` with `DKIOCGETBLOCKSIZE` to get sector size
  - `ioctl` with `DKIOCGETBLOCKCOUNT` to get block count
  - `fcntl` with `F_NOCACHE = 48` for cache bypass

- `static (int sectorSize, long totalBytes) GetPartitionGeometry(string devicePath)`:
  - Platform-dispatched: detect OS, call appropriate ioctls
  - Returns sector size and total partition/device size
  - Throws `PlatformNotSupportedException` on unsupported OS

**RawPartitionBlockDevice** sealed class implementing `IBatchBlockDevice`:
- Constructor: `string devicePath, int blockSize, BlockDeviceOptions? options = null`
  - `devicePath` examples: `/dev/nvme0n1p1`, `\\.\PhysicalDrive0`, `/dev/rdisk2`
  - Calls `GetPartitionGeometry(devicePath)` to detect sector size and total size
  - `BlockCount = totalBytes / blockSize`
  - **DWVD magic validation (fail-closed)**: Read first `blockSize` bytes. Check for `VdeConstants.MagicBytes` (0x44575644) at expected superblock location. If magic not found AND `options?.AllowNonDwvd != true`, throw `InvalidOperationException("Partition does not contain DWVD signature. Use BlockDeviceOptions.AllowNonDwvd = true to override (DANGEROUS).")`. This prevents accidental overwrites of non-DWVD partitions.
  - For `createNew` equivalent: write a minimal superblock header with magic bytes to block 0 (caller responsibility to format properly afterward)
  - Opens device handle:
    - Windows: `File.OpenHandle(devicePath, FileMode.Open, FileAccess.ReadWrite, FileShare.None, FileOptions.Asynchronous | (FileOptions)0x20000000 | FileOptions.WriteThrough)`
    - Linux/macOS: `File.OpenHandle` with `FileOptions.Asynchronous | FileOptions.WriteThrough` (O_DIRECT applied separately if available)
- `BlockSize`, `BlockCount`, `MaxBatchSize = 64`
- `string DevicePath { get; }` -- the raw device path
- `int SectorSize { get; }` -- detected physical sector size
- `ReadBlockAsync` / `WriteBlockAsync`: via `RandomAccess.ReadAsync`/`WriteAsync` (same pattern as DirectFileBlockDevice, device handles work the same way)
- `ReadBatchAsync` / `WriteBatchAsync`: parallel `Task.WhenAll` over individual operations
- `FlushAsync`: `RandomAccess.FlushToDisk`
- `DisposeAsync`: dispose file handle

Add `AllowNonDwvd` property to `BlockDeviceOptions` (from 87-56): `bool AllowNonDwvd { get; set; } = false` -- must be explicitly set to bypass DWVD magic check on raw partitions.

IMPORTANT: Raw partition access requires elevated privileges (root/Administrator). The constructor should catch `UnauthorizedAccessException` and wrap it with a helpful message: "Raw partition access requires elevated privileges (root on Linux/macOS, Administrator on Windows)."
