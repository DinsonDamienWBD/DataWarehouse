- Constructor: `IntegrityCacheConfig config`

Cache entry:
- `CacheEntry` readonly struct:
  - `long StartBlock` -- extent start block (cache key)
  - `int BlockCount` -- extent size
  - `uint VerifiedGeneration` -- GenerationNumber when last verified
  - `bool IsVerified` -- true if integrity check passed
  - `DateTimeOffset VerifiedAt` -- timestamp of verification
  - `byte[]? ExpectedHash` -- cached BLAKE3 hash (16B) for quick comparison

Thread-safe storage:
- `ConcurrentDictionary<long, CacheEntry> _cache` -- keyed by StartBlock
- `ConcurrentQueue<long> _lruQueue` -- approximate LRU ordering for eviction

Core operations:
- `bool TryGetVerified(long startBlock, uint currentGeneration, out CacheEntry entry)`:
  1. Lookup by startBlock
  2. If found: check generation match (currentGeneration == entry.VerifiedGeneration)
  3. If found: check TTL not expired
  4. If valid: return true + entry (cache hit)
  5. If stale/expired: remove entry, return false (cache miss)

- `void RecordVerification(long startBlock, int blockCount, uint generation, bool isVerified, byte[]? expectedHash)`:
  1. Create CacheEntry with current timestamp
  2. Insert or update in _cache
  3. Add to LRU queue
  4. If _cache.Count > MaxEntries: run eviction

- `void InvalidateExtent(long startBlock)`:
  - Remove specific entry (called on write)

- `void InvalidateByGeneration(uint olderThanGeneration)`:
  - Bulk invalidate all entries with VerifiedGeneration < threshold

LRU eviction:
- `void EvictOldest()`:
  1. Dequeue EvictionBatchSize entries from _lruQueue
  2. Remove from _cache if still present
  3. Entries recently re-verified may be re-added to queue (approximate LRU)

Statistics:
- `IntegrityCacheStats GetStats()`:
  - `IntegrityCacheStats` readonly struct: `int TotalEntries, long Hits, long Misses, long Evictions, double HitRate`
- Thread-safe counters using Interlocked for hits/misses/evictions

- `void Clear()` -- flush all entries
