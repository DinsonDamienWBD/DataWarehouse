First, create `DataWarehouse.SDK/VirtualDiskEngine/Retention/` directory if it does not exist.

Create `RetentionEpochMapper` class with `[SdkCompatibility("6.0.0")]`:
- Maps time-based retention policies to epoch numbers
- `RetentionPolicy` record: `string Name, TimeSpan Duration, bool AutoDelete`
- `Dictionary<string, RetentionPolicy> _policies`
- `void RegisterPolicy(RetentionPolicy policy)`: add named retention policy
- `long ComputeExpirationEpoch(string policyName, long currentEpoch, TimeSpan epochDuration)`:
  - Given current epoch and epoch duration, compute which epoch corresponds to `now - policy.Duration`
  - Returns the epoch number at which data under this policy expires
- `IReadOnlyList<RetentionPolicyExpiration> GetExpiredPolicies(long currentEpoch, TimeSpan epochDuration)`:
  - Returns all policies where `ComputeExpirationEpoch < currentEpoch`
  - `RetentionPolicyExpiration` readonly struct: `string PolicyName, long ExpirationEpoch, TimeSpan OverdueBy`
- `long GetOldestRetentionEpoch(long currentEpoch, TimeSpan epochDuration)`:
  - Returns the minimum expiration epoch across all registered policies
  - This is the safe epoch to advance OldestActiveEpoch to

Create `EpochGatedLazyDeletion` class with `[SdkCompatibility("6.0.0")]`, implements `IAsyncDisposable`:
- Constructor: `IBlockDevice device, IBlockAllocator allocator, RetentionEpochMapper retentionMapper, int blockSize`
- `long OldestActiveEpoch { get; private set; }` -- read from superblock at mount
- `long CurrentEpoch { get; }` -- current global epoch

Epoch advancement (the core O(1) bulk delete):
- `Task<EpochAdvanceResult> AdvanceOldestEpochAsync(long newOldestEpoch, CancellationToken ct)`:
  1. Validate: newOldestEpoch > current OldestActiveEpoch
  2. Validate: newOldestEpoch <= CurrentEpoch (cannot delete future data)
  3. Write newOldestEpoch to superblock (single block write — the O(1) operation)
  4. Update in-memory OldestActiveEpoch
  5. Return EpochAdvanceResult with estimated blocks to reclaim
  6. Actual block reclamation happens lazily in Background Vacuum

- `EpochAdvanceResult` readonly struct: `long PreviousOldestEpoch, long NewOldestEpoch, long EstimatedBlocksToReclaim, long EstimatedInodesAffected`

Lazy vacuum scan:
- `Task<LazyReclaimResult> VacuumScanAsync(int maxBlocksPerCycle, CancellationToken ct)`:
  1. Scan TRLR blocks sequentially (or from last scan position)
  2. For each data block: read GenerationNumber from TRLR
  3. If GenerationNumber maps to an epoch < OldestActiveEpoch:
     - Block is logically deleted — free it via allocator
  4. Track scan position for incremental processing
  5. Return reclaim stats

- `LazyReclaimResult` readonly struct: `long BlocksScanned, long BlocksReclaimed, long BytesReclaimed, double ScanProgress, TimeSpan Duration`

- `long _lastScanPosition` -- persisted to enable incremental scanning across vacuum cycles

Retention-driven auto-advancement:
- `Task<EpochAdvanceResult?> AutoAdvanceFromRetentionAsync(TimeSpan epochDuration, CancellationToken ct)`:
  1. Query retentionMapper.GetOldestRetentionEpoch(currentEpoch, epochDuration)
  2. If computed epoch > current OldestActiveEpoch: advance
  3. If no advancement needed: return null
  - This enables automatic time-series expiration driven by retention policies

Continuous background mode:
- `Task RunAsync(TimeSpan vacuumInterval, TimeSpan epochDuration, CancellationToken ct)`:
  - Background loop: auto-advance from retention, then run vacuum scan
  - Respects cancellation

Statistics:
- `EpochDeletionStats GetStats()` readonly struct:
  - `long OldestActiveEpoch, long CurrentEpoch, long TotalBlocksReclaimed, long TotalEpochsAdvanced`
  - `double VacuumScanProgress` (0.0 to 1.0, fraction of TRLR scanned since last epoch advance)
  - `DateTimeOffset LastAdvanceTime, DateTimeOffset LastVacuumTime`
