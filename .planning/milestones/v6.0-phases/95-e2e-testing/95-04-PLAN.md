- `public static async Task<(int passed, int failed, IReadOnlyList<string> failures)> RunAllAsync(CancellationToken ct = default)`
- Each test in a named tuple array, foreach with try/catch, Trace.TraceInformation for PASS/FAIL
- All test helpers as private static async methods

First, create a LOCAL `InMemoryPhysicalBlockDevice` class (internal, inside same file or as nested class) implementing `IPhysicalBlockDevice`. Do NOT reference the one in UltimateRAID plugin (SDK cannot reference plugins). Model it after the `InMemoryBlockDevice` in FederationIntegrationTests but extended with:
- `PhysicalDeviceInfo DeviceInfo` (generate synthetic serial/model/capacity)
- `bool IsOnline` with `SetOnline(bool)` control method
- `long PhysicalSectorSize => 4096`
- `long LogicalSectorSize => 512`
- `TrimAsync` (no-op, returns completed)
- `ReadScatterAsync` / `WriteGatherAsync` (loop over operations, delegate to ReadBlockAsync/WriteBlockAsync)
- `GetHealthAsync` returning default healthy PhysicalDeviceHealth
- Use `ConcurrentDictionary<long, byte[]>` for sparse block storage
- Throw `IOException("Device offline")` from read/write/flush when `!IsOnline`

Implement these 8 tests:

1. **TestRaid5SingleDeviceFailureRecovery**: Create 4-device RAID 5 CompoundBlockDevice (stripe=8). Write known pattern to 100 blocks. Take device[2] offline via SetOnline(false). Read all 100 blocks — verify all data correct (parity reconstruction). Bring device back online. Verify reads still correct.

2. **TestRaid6DualDeviceFailureRecovery**: Create 6-device RAID 6 CompoundBlockDevice (stripe=8). Write known pattern to 200 blocks. Take device[1] and device[4] offline simultaneously. Read all 200 blocks — verify all data correct (double-parity reconstruction). Assert array is still readable despite 2 failures.

3. **TestMirrorReadFailover**: Create 2-device mirrored CompoundBlockDevice. Write known pattern to 50 blocks. Take device[0] offline. Read all 50 blocks — verify correct data served from device[1]. Take device[0] online, device[1] offline. Read again — verify served from device[0].

4. **TestStripedMirrorPartialFailure**: Create 4-device RAID 10 CompoundBlockDevice. Write known pattern to 100 blocks. Take one device from each mirror pair offline (device[0] and device[3]). Read all 100 blocks — verify data correct from surviving mirrors.

5. **TestHotSpareRebuildAfterFailure**: Create 4-device RAID 5 CompoundBlockDevice with 1 hot spare (5 devices total). Write known pattern to 100 blocks. Take device[2] offline. Create HotSpareManager with AutoRebuild=true, MaxSpares=1. Trigger rebuild by calling appropriate manager method. After rebuild, verify all data reads back correctly. Verify the spare device now contains the rebuilt data (read from spare device directly and compare).

6. **TestSimultaneousWriteDuringDegradedMode**: Create 4-device RAID 5 CompoundBlockDevice. Write initial pattern to 50 blocks. Take device[1] offline (degraded mode). Write NEW pattern to blocks 50-99 while degraded. Bring device[1] back online. Read ALL 100 blocks — verify blocks 0-49 have original pattern, blocks 50-99 have new pattern. Proves write correctness during degraded operation.

7. **TestZeroOverheadSingleDeviceBaseline**: Create single-device "RAID 0" CompoundBlockDevice (1 device, stripe=1). Time 1000 block writes and 1000 block reads via CompoundBlockDevice. Time same operations directly on the underlying InMemoryPhysicalBlockDevice. Assert CompoundBlockDevice overhead is less than 20% of direct device time (generous threshold for in-memory). This proves the abstraction adds minimal overhead.

8. **TestDataIntegrityAcrossAllRaidLevels**: Parameterized across [Striped, Mirrored, Parity, DoubleParity, StripedMirror]. For each: create appropriate device array, write 50 blocks with RAID-level-specific pattern (e.g., byte = raidLevel ^ blockIndex), read back all 50, verify byte-exact match. Proves data fidelity across every supported RAID level.

Helper methods needed:
- `CreateDevices(int count, int blockSize = 4096, long blockCount = 10_000)` — returns `List<InMemoryPhysicalBlockDevice>`
- `WritePattern(IBlockDevice device, long startBlock, int count, byte seed)` — writes deterministic pattern
- `VerifyPattern(IBlockDevice device, long startBlock, int count, byte seed)` — reads and verifies, throws on mismatch
- `AssertEqual(bool condition, string message)` — throws InvalidOperationException on failure

Use deterministic patterns: for block N with seed S, fill with `(byte)((N * 7 + S) & 0xFF)` repeated to fill block.

All InMemoryPhysicalBlockDevice instances use blockSize=4096 and blockCount=10_000 unless test requires otherwise.
