## Known Limitations

1. **Stryker mutation testing:** Source-analysis tests (~50% of suite) read `.cs` files directly and cannot detect IL mutations. Full mutation testing requires runtime-exercising tests. CI gate configured at 95% applies only to runtime-testable code.

2. **dotCover runtime coverage:** 0% expected because source-analysis tests validate code patterns via string/regex matching, not execution. Finding coverage is 100% (all 11,128 findings have corresponding test assertions).

3. **Coyote VDE tests:** Coyote's systematic scheduler cannot control OS file I/O operations. VDE file tests use standard xUnit; managed-code concurrency uses Coyote.

4. **dotCover tool version:** JetBrains dotCover 2025.3.3 is incompatible with .NET 10.0-preview ("Snapshot container not initialized"). Coverlet used as fallback for CI coverage.

5. **Soak test duration:** CI runs 2-minute soak (conditional pass). Full 24-72hr soak requires manual execution on dedicated hardware.

6. **Equivalent mutants:** Some Stryker mutations produce equivalent behavior (e.g., changing `>=` to `>` where boundary is never hit). These are expected and documented in Stryker HTML reports.
